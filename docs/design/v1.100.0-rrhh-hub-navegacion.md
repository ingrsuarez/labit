```

  Modo: mejora + reestructuración de navegación
  Pantalla/feature: Hub RRHH en /rrhh; panel analítico movido a /rrhh/resumen

    - admin/section.blade.php → patrón de cards (referencia visual obligatoria)
```

### Bloques de sección (×3 máximo visibles)

Cada bloque corresponde a una de las secciones actuales de `AdminSectionController` (`personal`, `ausencias`, `liquidaciones`).

#### Encabezado de bloque

| Elemento | Clases / contenido |
|----------|-------------------|
| Título de sección | `text-lg font-semibold text-gray-900` — texto: **Personal** / **Ausencias** / **Liquidaciones** |
| Descripción | `text-sm text-gray-500 mt-0.5` — misma `description` que hoy en el controller |
| Separador opcional | `border-b border-gray-200 pb-3 mb-4` bajo el encabezado del bloque |
| Ancla HTML | `id="personal"`, `id="ausencias"`, `id="liquidaciones"` en el `<section>` para deep links y redirects desde `admin.section.*` |

#### Grid de cards (por bloque)

**Reutilizar exactamente** el markup de `resources/views/admin/section.blade.php`:

```blade

    @foreach ($section['items'] as $item)
        {{-- card idéntica a admin.section --}}

```

#### Orden de secciones en pantalla

1. Personal  
2. Ausencias  
3. Liquidaciones  

Solo se renderizan las secciones para las que el usuario tenga permiso de sección.

### Estados de la pantalla

| Estado | Qué mostrar |
|--------|-------------|
| Usuario con 1+ permisos de sección | Hub con solo los bloques permitidos |
| Usuario sin ningún permiso RRHH | 403 (no debería ver el ítem de sidebar) |
| Sección con 0 ítems visibles (todos filtrados por `permission` en ítems) | **Ocultar el bloque completo** (no mostrar título vacío) |
| Usuario solo con `liquidaciones.section` | Un bloque «Liquidaciones» con sus cards |

### Flujo de acciones

1. Usuario hace clic en **Recursos Humanos** en el sidebar → llega a `/rrhh`.
2. Ve los bloques según sus permisos.
3. Hace clic en una card → navega directo a la ruta destino (empleados, vacaciones, payroll, etc.).
4. (Opcional) Admin/contador hace clic en **Ver resumen** → `/rrhh/resumen`.

### Visibilidad del ítem «Recursos Humanos»

Mostrar si el usuario cumple **cualquiera** de:

- `hasAnyRole(['admin', 'contador'])`, **o**
- `@can('personal.section')`, **o**
- `@can('ausencias.section')`, **o**
- `@can('liquidaciones.section')`

```blade
@if (
    auth()->user()?->hasAnyRole(['admin', 'contador'])
    || auth()->user()?->can('personal.section')
    || auth()->user()?->can('ausencias.section')
    || auth()->user()?->can('liquidaciones.section')

    <a href="{{ route('rrhh.index') }}" …>Recursos Humanos</a>
@endif
```

## Rutas legacy `admin.section.*`

| Ruta actual | Comportamiento propuesto |
|-------------|------------------------|
| `GET admin/personal` | Redirect 302 → `route('rrhh.index')#personal` |
| `GET admin/ausencias` | Redirect 302 → `route('rrhh.index')#ausencias` |
| `GET admin/liquidaciones` | Redirect 302 → `route('rrhh.index')#liquidaciones` |

Mantener middleware de permiso existente en cada ruta antes del redirect.

Las vistas `admin.section` pueden quedar sin uso directo; el Dev puede deprecarlas o convertirlas en partial — **no** eliminar en v1.100.0 si no hay tests que lo exijan.

## Permisos y autorización

| Recurso | Regla |
|---------|--------|
| `GET /rrhh` | Usuario autenticado con al menos uno: `personal.section`, `ausencias.section`, `liquidaciones.section`, o rol `admin`/`contador` |
| Cada bloque en la vista | `@can('personal.section')` etc. envolviendo el `<section>` |
| Cada card con `'permission' => '...'` en el array | `@can($item['permission'])` en el loop (hoy **no** se filtra en `admin.section` — **corregir** en el hub) |
| `GET /rrhh/resumen` | Solo `admin` o `contador` (igual que hoy `RrhhController@index`) |

## Datos y backend (orientación al Dev, sin código)

1. **Extraer** los tres arrays de sección de `AdminSectionController` a un método estático o servicio compartido, p. ej. `RrhhNavigation::sections(): array`, retornando:

```php
[
    [
        'key' => 'personal',
        'permission' => 'personal.section',
        'title' => 'Personal',
        'description' => '...',
        'items' => [ /* mismos ítems actuales */ ],
    ],
    // ausencias, liquidaciones
]
```

2. `RrhhController@index` — solo arma `$sections` filtradas y devuelve vista hub (sin queries de KPI).
3. `RrhhController@resumen` — mueve la lógica actual de `index` (queries + compact).
4. Filtrar ítems: si `isset($item['permission'])`, incluir solo si `auth()->user()->can(...)`.

## Qué NO cambiar

- Rutas destino de cada card (empleados, payroll, vacation, etc.).
- Textos, descripciones e íconos SVG path de cada ítem (copiar de `AdminSectionController` tal cual).
- Estilo visual de cards (zinc/gris, igual que Compras).
- Contenido funcional del panel resumen (solo cambia URL y títulos).
- Permisos de sección existentes en la base (`personal.section`, etc.).

## Qué NO incluir en v1.100.0

- Reordenar cards por uso frecuente.
- Badges de contadores en cards (ej. solicitudes pendientes).
