## Pantalla 1 — Catálogo de percepciones (`/purchase-perceptions`)

### Propósito
Administrar los tipos de percepciones recibidas en facturas de compra por empresa
(ej: IIBB Neuquén 1,5%, IVA RG 4597, IIBB CABA). Cada tipo define: nombre,
jurisdicción, alícuota y la cuenta contable del anticipo sufrido.

### Layout general
Sigue el patrón estándar del proyecto: layout admin, sidebar Compras activo,
header con breadcrumb + botón "+ Nueva percepción", tabla filtrable.

### Secciones

#### Cabecera
- Título: **"Percepciones de compra"**
- Breadcrumb: `Compras > Percepciones`
- Botón top-right: `[+ Nueva percepción]` — `bg-indigo-600 hover:bg-indigo-700`

#### Filtros
- Buscador texto: filtra por nombre o jurisdicción (Alpine `x-model` client-side)
- Filtro estado: select `Activas | Inactivas | Todas` (default: Activas)
- Comportamiento: filtrado en tiempo real (Alpine, sin submit)

#### Tabla de catálogo

Columnas:

| Columna | Tipo | Notas |
|---|---|---|
| Nombre | texto | ej: "IIBB Neuquén" |
| Jurisdicción | texto / "—" | ej: "Neuquén", "Nacional" |
| Alícuota | porcentaje | ej: "1,50%" — si es 0: "—" |
| Cuenta contable | código + nombre | ej: "1.1.05 — Anticipo IIBB Nqn" |
| Estado | badge | Activa (verde) / Inactiva (gris) |
| Acciones | botones | Editar (lápiz), toggle Activa/Inactiva, Eliminar |

- Eliminar: solo si la percepción no tiene líneas históricas en FC/NC
- Ordenamiento: por `sort_order` ascendente
- Paginación: 20 por página

#### Estado vacío
Mensaje: "No hay percepciones configuradas para esta empresa."  
CTA: botón `[+ Agregar la primera]`

#### Formulario create/edit

**Formato preferido: modal** (pocos campos, evita salir de la tabla).  
Título del modal: "Nueva percepción" / "Editar percepción".

Campos:

| Campo | Tipo | Validación |
|---|---|---|
| Nombre | `input text` | requerido, max:100 |
| Jurisdicción | `input text` | opcional, ej: "Neuquén", "CABA" |
| Alícuota % | `input number` step 0.01 | requerido, min:0 — puede ser 0 (alícuota libre en FC) |
| Cuenta contable | select Tom Select | requerido, filtrado a cuentas 1.1.xx (activos corrientes) |
| Orden visualización | `input number` | default: último + 10 |
| Activa | checkbox | default: checked |

Botones del modal: `[Cancelar]` (gris) + `[Guardar]` (verde)

### Estados de la pantalla

| Estado | Qué mostrar |
|---|---|
| Sin datos | Empty state + CTA |
| Con datos | Tabla normal |
| Guardando | Botón "Guardar" deshabilitado + spinner |
| Error | Flash `bg-red-50 border-red-400` arriba del formulario |

## Pantalla 2 — Bloque "Percepciones" en Factura de Compra

### Propósito
Reemplazar el campo `percepciones` (input numérico suelto) por un bloque Alpine.js
multi-línea donde el usuario selecciona el tipo de percepción del catálogo e ingresa
el monto. El sistema almacena snapshots para el asiento contable.

### Ubicación en el formulario de FC

**Card separada**, debajo de la card "Remitos asociados" y encima de la grilla de ítems.

*Justificación:* el bloque tiene sub-tabla y sub-total propio; mezclarlo en el
`grid-cols-4` de datos generales haría el formulario muy denso. Una card propia
mantiene la lógica visual aislada y el Tab flow limpio.

### Layout del bloque

```
┌───────────────────────────────────────────────────────────────────────────────┐
│  Percepciones                                                                 │
├───────────────────────────────────────────────────────────────────────────────┤
│  Tipo de percepción              │  Alícuota  │  Monto          │             │
│  [IIBB Neuquén             ▾]    │   1,50 %   │  [$ 150,00   ]  │  [✕]       │
│  [IVA RG 4597              ▾]    │    —       │  [$ 50,00    ]  │  [✕]       │
├───────────────────────────────────────────────────────────────────────────────┤
│  + Agregar percepción                        Total percepciones: $ 200,00    │
└───────────────────────────────────────────────────────────────────────────────┘
```

**Columnas de la sub-tabla:**

| Columna | Ancho | Comportamiento |
|---|---|---|
| Tipo de percepción | flex (resto) | `<select>` Tom Select o nativo. Al seleccionar → auto-rellena Alícuota con `rate` del catálogo |
| Alícuota | ~90px | Solo lectura — `<span x-text>`. Muestra "1,50%" o "—" si rate = 0 |
| Monto | ~140px | `input[type=number]` step=0.01 min=0, editable |
| Eliminar | ~40px | Botón `×` `text-red-500 hover:text-red-700` |

**Footer del bloque:**
- Izquierda: botón `[+ Agregar percepción]` — texto link, color `text-indigo-600`, icono `bi-plus`
- Derecha: "Total percepciones: $ X,XX" — `font-medium text-gray-700`

**Estado vacío (0 líneas):** solo footer con botón agregar + "Total percepciones: $0,00".

**Si no hay tipos en el catálogo de la empresa:**  
Mostrar mensaje inline: `"No hay percepciones configuradas."` + link `[Configurar →]` que apunta a `/purchase-perceptions/create`. El botón "Agregar" no aparece.

**Integración con el total general de la FC:**  
La variable Alpine `perceptionsTotal` (suma de montos de las líneas) reemplaza la
variable anterior `percepciones`. Se incluye en el cálculo de `grandTotal`.

**Inputs ocultos enviados por línea (igual que en v1.63.1):**
```
perceptions[i][purchase_perception_id]
perceptions[i][name_snapshot]
perceptions[i][jurisdiction_snapshot]
perceptions[i][rate_snapshot]
perceptions[i][accounting_account_id]
perceptions[i][amount]
```

### Vista show de FC — ajuste de la sección "Percepciones"

**Antes (legacy):** una sola fila `Percepciones: $200,00`

**Después (si hay líneas en la pivote):** mini-lista detallada:
```
Percepciones:
  • IIBB Neuquén (1,50%)     $  150,00
  • IVA RG 4597               $   50,00
  ─────────────────────────
  Total:                      $  200,00
```

**Retrocompatibilidad:** si `$invoice->perceptions->isEmpty()` pero
`$invoice->percepciones > 0` → mostrar el número suelto como antes (FCs legado).

### Componentes Blade/Alpine a usar
- Card: `bg-white rounded-xl shadow-sm border border-gray-200 p-5 mb-5`
- Sub-tabla Alpine: `<template x-for>` — mismo patrón que la tabla de ítems
- Input monto: `input[type=number] step="0.01" min="0"`
- Select tipo: nativo `<select>` (Tom Select si el catálogo es largo > 10 tipos)
- Alícuota: `<span x-text="perception.rate_snapshot ? perception.rate_snapshot + '%' : '—'"></span>`

## Pantalla 3 — Saldos de percepciones (`/purchase-perceptions/balances`)

### Propósito
Consulta por período: cuánto se cargó como anticipos en FCs (y NCs después de v1.63.1)
vs. el saldo real en la cuenta contable (Libro Mayor). Permite saber cuánto hay
disponible para imputar en una DDJJ de impuesto.

### Layout general
Página de consulta (sin acciones de modificación): filtros de período + tabla por percepción.

### Secciones

#### Cabecera
- Título: **"Saldos de percepciones sufridas"**
- Breadcrumb: `Compras > Percepciones > Saldos`
- Sin botones de acción (solo consulta)

#### Filtros de período

```
┌──────────────────────────────────────────────────────────────────────┐
│  Desde: [input date]    Hasta: [input date]    [Consultar]           │
└──────────────────────────────────────────────────────────────────────┘
```

- Preset inicial: primer y último día del mes en curso
- `method="GET"` para que los filtros sean linkables
- Empresa activa: automática (usa la empresa activa del contexto, no se elige)

#### Tabla de saldos

| Columna | Descripción |
|---|---|
| Percepción | Nombre del tipo |
| Jurisdicción | Texto o "—" |
| Cuenta contable | `código — nombre` (ej: "1.1.05 — Anticipo IIBB Nqn") |
| Anticipos cargados | Suma de `PurchaseInvoicePerception.amount` en el período |
| Saldo en cuenta (Libro Mayor) | Saldo neto de la cuenta en el período (desde `journal_entry_lines`) |
| Diferencia | `anticipos - saldo_libro_mayor` — badge verde "OK" si = 0, amarillo "Revisar" si ≠ 0 |

**Fila de totales:** `<tfoot>` con suma de "Anticipos cargados" y "Saldo en cuenta".

**Tooltip en "Saldo en cuenta (Libro Mayor)":**  
`"Calculado desde asientos contables del período seleccionado."`

#### Badges de diferencia
- `bg-green-100 text-green-800` → "OK" (diferencia = 0)
- `bg-yellow-100 text-yellow-800` → "Revisar" (diferencia ≠ 0)

#### Estados de la pantalla

| Estado | Qué mostrar |
|---|---|
| Sin percepciones configuradas | "No hay percepciones configuradas. [Configurar →]" |
| Sin transacciones en el período | "No se encontraron percepciones en el período seleccionado." |
| Con datos | Tabla con fila de totales |

### Componentes Blade a usar
- Form filtros: `form method="GET"` con `input[type=date]` y botón `[Consultar]`
- Tabla: patrón `overflow-x-auto + divide-y divide-gray-200`
- Badges: `bg-green-100/text-green-800` y `bg-yellow-100/text-yellow-800`

## Notas para el programador

- El campo `percepciones` (decimal) en `purchase_invoices` se mantiene como **caché** —
  se recalcula desde la pivote en `PurchaseInvoice::recalculate()`.
- Para las FCs existentes sin líneas de percepción: el cache `percepciones` es el
  valor que tenían; no es necesario migrar retroactivamente.
- El asiento contable (`AccountingEntryService::fromPurchaseInvoice`) debe debitar
  cada percepción en su `accounting_account_id` del snapshot (en lugar de usar
  una cuenta hardcoded).
- Ver `docs/design/v1.63.0-percepciones-compra.md` (este archivo) desde el prompt
  `v1.63.0-percepciones-compra.md` para contexto visual.
