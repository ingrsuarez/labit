# v1.54.1 — Hotfix lote/vencimiento en remito: ocultar inputs cuando no trackea lote

**VERSIÓN:** v1.54.1
**SLUG:** hotfix-lote-vencimiento-remito-xshow
**DEPENDENCIAS:** Ninguna
**ESTIMACIÓN:** 20min
**PRIORIDAD:** urgente

---

## Descripción

Al crear un insumo nuevo desde el modal de remitos con "¿Controla lote/vencimiento?"
destildado, los campos Lote y Vencimiento siguen visibles (solo deshabilitados) y el
servidor los valida como requeridos, bloqueando el guardado del remito.

El fix reemplaza `:disabled="!item._tracks_lot"` por `x-show="item._tracks_lot"` en los
inputs de Lote y Vencimiento, de modo que:

- `_tracks_lot = false` → inputs ocultos, celda muestra `—`, valores limpiados
- `_tracks_lot = true`  → inputs visibles y completables (sin cambios funcionales)

También corrige `createSupply()` para limpiar explícitamente `lot_number` y
`expiration_date` cuando el nuevo insumo no trackea lote.

## PASO 1 — Fix en delivery-notes/create.blade.php

Archivo: `resources/views/delivery-notes/create.blade.php`

### 1.1 — Celda Lote: reemplazar :disabled por x-show

Localizar el `<td>` de Lote en la tabla de ítems (dentro del `<template x-for>`).

**ANTES:**
```html
<td class="px-3 py-2">
    <input type="text" :name="'items[' + index + '][lot_number]'"
           x-model="item.lot_number"
           @keydown.enter.prevent="$dispatch('add-remito-item')"
           placeholder="—"
           :disabled="!item._tracks_lot"
           :class="item._tracks_lot ? '' : 'bg-gray-50 text-gray-300'"
           class="w-24 rounded border-gray-300 text-xs focus:border-zinc-500 focus:ring-zinc-500">
</td>
```

**DESPUÉS:**
```html
<td class="px-3 py-2">
    <template x-if="item._tracks_lot">
        <input type="text" :name="'items[' + index + '][lot_number]'"
               x-model="item.lot_number"
               @keydown.enter.prevent="$dispatch('add-remito-item')"
               placeholder="Lote"
               class="w-24 rounded border-gray-300 text-xs focus:border-zinc-500 focus:ring-zinc-500">
    </template>
    <template x-if="!item._tracks_lot">
        <span class="text-gray-300 text-xs">—</span>
    </template>
</td>
```

### 1.2 — Celda Vencimiento: reemplazar :disabled por x-show


### 1.3 — Corregir createSupply(): limpiar lote/vencimiento cuando no trackea

Localizar la función `createSupply()` dentro de `deliveryForm()`. Dentro del bloque
donde se actualizan los datos del item después de crear el insumo:

```js
if (this.newSupplyForRow !== null && this.items[this.newSupplyForRow]) {
    this.items[this.newSupplyForRow].supply_id = data.id;
    this.items[this.newSupplyForRow]._supply_name = data.name;
    this.items[this.newSupplyForRow]._supply_code = data.code;
    this.items[this.newSupplyForRow]._tracks_lot = !!data.tracks_lot;
    // ...
}
```

Agregar la limpieza explícita de lot_number y expiration_date cuando no trackea:

```js
if (this.newSupplyForRow !== null && this.items[this.newSupplyForRow]) {
    this.items[this.newSupplyForRow].supply_id = data.id;
    this.items[this.newSupplyForRow]._supply_name = data.name;
    this.items[this.newSupplyForRow]._supply_code = data.code;
    this.items[this.newSupplyForRow]._tracks_lot = !!data.tracks_lot;
    if (!data.tracks_lot) {
        this.items[this.newSupplyForRow].lot_number = '';
        this.items[this.newSupplyForRow].expiration_date = '';
    }
    // ... (resto del nextTick sin cambios)
}
```

**Verificación PASO 1:**
- Crear insumo "guantes" sin tildar lote → fila muestra `—` en Lote y Vencimiento, se puede guardar el remito
- Crear insumo "reagente X" con lote tildado → fila muestra inputs de Lote y Vencimiento completables
- Seleccionar insumo existente sin tracks_lot → `—` en Lote y Vencimiento
- Seleccionar insumo existente con tracks_lot → inputs visibles
