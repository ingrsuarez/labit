# v1.45.1 — Alta protocolo veterinario: destrabar agregar prácticas / buscador

**VERSIÓN:** v1.45.1  
**SLUG:** fix-ux-alta-protocolo-vet-practicas  
**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 1–2h  
**PRIORIDAD:** alta

---

## Descripción

**Reporte de usuario:** al crear un protocolo veterinario nuevo no puede agregar prácticas (determinaciones).

**Análisis previo (PM):** en `resources/views/vet/admissions/create.blade.php` el campo “Agregar determinación” está **deshabilitado** hasta que `customerId` (veterinaria) tiene valor (`:disabled="!customerId"`). Si el usuario no elige primero la veterinaria, la UI parece “sin opción” de cargar prácticas. Además, la búsqueda usa URL relativa `./search-tests`, sensible al path base del sitio.

**Objetivo:** que quede claro cómo agregar prácticas y que la búsqueda sea fiable; corregir cualquier bug de red/ruta confirmado en verificación.

---

## PASO 0 — Contexto a leer

- `resources/views/vet/admissions/create.blade.php` — Alpine `vetAdmissionForm()`, input determinaciones, `fetch` a búsqueda
- `app/Http/Controllers/VetAdmissionController.php` — `searchTests()`, validación `customer_id`
- `routes/web.php` — grupo `prefix('vet')`, ruta nombrada `vet.admissions.searchTests`
- (Opcional comparación) `resources/views/lab/admissions/create.blade.php` — patrón de buscador + `route()` en fetch si aplica

---

## PASO 1 — Verificación y causa raíz

1. Reproducir en navegador: entrar a **Nuevo protocolo veterinario** sin seleccionar veterinaria → confirmar que el input está deshabilitado.
2. Seleccionar veterinaria → confirmar que el buscador habilita y `GET` a búsqueda devuelve JSON (Network tab).
3. Si `./search-tests` falla en el entorno (404, CORS, path incorrecto con subcarpeta `public/`), documentar y pasar a PASO 2.

---

## PASO 2 — Cambios de implementación (mínimos)

1. **URL de búsqueda:** reemplazar el `fetch` con URL relativa `./search-tests?...` por la ruta nombrada de Laravel, p. ej. `{{ route('vet.admissions.searchTests') }}` + query string, para que funcione con cualquier `APP_URL` / subdirectorio.
2. **UX “no puedo agregar prácticas”:**
   - En la sección **Determinaciones**, texto corto visible siempre: indicar que **primero hay que elegir la veterinaria** para buscar y agregar prácticas del nomenclador veterinario.
   - Opcional (preferible si mejora percepción): en lugar de `disabled` en el input, mantener el campo enfocable y, si no hay `customerId`, al `@focus` o al intentar buscar mostrar mensaje claro (sin llamar al servidor hasta tener `customer_id`). No regresar a tabla con checkboxes.
3. Si `veterinary_nbu_value` es 0: el mensaje ya existe (“Configurá el valor NBU…”); no debe impedir agregar líneas (precio $0), solo aclarar precios.
