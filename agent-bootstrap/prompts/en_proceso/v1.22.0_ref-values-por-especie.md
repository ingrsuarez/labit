## Descripción

Crea el sistema de valores de referencia por especie para determinaciones veterinarias.
Cada test veterinario puede tener rangos de referencia distintos por especie (Canino, Felino, Equino).

Se implementa con una tabla nueva `test_species_references` que es independiente del sistema
existente de `reference_categories` + `test_reference_values` (usado para normativas de aguas como CAA, AE, etc.).

La UI se agrega como sección en la página de valores de referencia de cada test, visible solo
para tests con categoría `veterinario`.

## PASO 2 — Modelo TestSpeciesReference

Crear `app/Models/TestSpeciesReference.php`:

```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class TestSpeciesReference extends Model
{
    use HasFactory;

    protected $fillable = [
        'test_id',
        'species_id',
        'low',
        'high',
        'other_reference',
    ];

    public function test()
    {
        return $this->belongsTo(Test::class);
    }

    public function species()
    {
        return $this->belongsTo(Species::class);
    }

    /**
     * Obtiene el rango formateado como string para mostrar en protocolos/PDFs
     */
    public function getFormattedRangeAttribute(): string
    {
        if ($this->other_reference) {
            if ($this->low || $this->high) {
                return ($this->low ?? '') . ' - ' . ($this->high ?? '') . ' | ' . $this->other_reference;
            }
            return $this->other_reference;
        }

        if ($this->low && $this->high) {
            return $this->low . ' - ' . $this->high;
        }

        if ($this->low) {
            return '>= ' . $this->low;
        }

        if ($this->high) {
            return '<= ' . $this->high;
        }

        return '';
    }
}
```

**Verificación:** Modelo se instancia sin errores en tinker.

### 4.2 — Método `storeSpeciesReference()`

```php
public function storeSpeciesReference(Request $request, Test $test)
{
    $validated = $request->validate([
        'species_id' => 'required|exists:species,id',
        'low' => 'nullable|string|max:50',
        'high' => 'nullable|string|max:50',
        'other_reference' => 'nullable|string|max:500',
    ]);

    TestSpeciesReference::updateOrCreate(
        ['test_id' => $test->id, 'species_id' => $validated['species_id']],
        [
            'low' => $validated['low'],
            'high' => $validated['high'],
            'other_reference' => $validated['other_reference'],
        ]
    );

    return back()->with('success', 'Valor de referencia por especie guardado.');
}
```

### 4.3 — Método `destroySpeciesReference()`

```php
public function destroySpeciesReference(Test $test, TestSpeciesReference $speciesReference)
{
    $speciesReference->delete();
    return back()->with('success', 'Valor de referencia por especie eliminado.');
}
```

## Notas / Consideraciones

- Este sistema es **independiente** de `reference_categories` + `test_reference_values`. No se mezclan conceptos de normativas de aguas con especies animales.
- `updateOrCreate` se usa en `storeSpeciesReference` para que guardar sea idempotente: si ya existe el registro para esa especie, lo actualiza. Si no existe, lo crea.
- El método `Test::getReferenceForSpecies($speciesId)` se usará en v1.24.0 al crear el protocolo veterinario para resolver automáticamente el valor de referencia según la especie del animal.
- Los campos `low`, `high` y `other_reference` son strings (igual que en la tabla `tests`) para soportar tanto rangos numéricos como textuales.
- La vista `reference-values.blade.php` ya existe y se usa para normativas. La sección de especies se agrega **debajo**, no reemplaza nada.
