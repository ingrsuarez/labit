## Descripción

Al crear o editar un cliente, el usuario puede ingresar un CUIT y consultar el padrón
de AFIP (`ws_sr_padron_a5`) para obtener automáticamente la razón social, condición de IVA,
domicilio fiscal, actividad económica y estado del CUIT.

Esto previene errores de facturación electrónica: si un cliente es Responsable Inscripto
y se le carga como Monotributista por error, AFIP rechaza la factura. Con la consulta
al padrón, la condición de IVA viene verificada desde la fuente y se bloquea para
evitar edición accidental.

El web service a usar es `ws_sr_padron_a5` (Padrón Alcance 5), que requiere autenticación
WSAA con el mismo certificado que ya se usa para WSFEv1.

2. Implementar método `consultarPadron(string $cuit): array`:

   - Hacer llamada SOAP `getPersona` con el CUIT.
   - Parsear la respuesta y devolver array normalizado:
     ```php
     [
         'success' => true,
         'razon_social' => '...', // persona.razonSocial o persona.apellido + persona.nombre
         'condicion_iva' => '...', // mapear desde persona.impuestos (imp 30/32) + categoría
         'condicion_iva_id' => int, // ID numérico AFIP para CondicionIVAReceptorId
         'domicilio' => [
             'direccion' => '...',
             'localidad' => '...',
             'provincia' => '...',  // mapear código de provincia AFIP a nombre
             'cod_postal' => '...',
         ],
         'actividad_principal' => '...', // descripción de la actividad
         'estado_cuit' => '...', // ACTIVO / INACTIVO
     ]
     ```
   - En caso de error, devolver `['success' => false, 'error' => '...']`.

3. Mapeo de condición IVA desde datos de AFIP:
   - Si tiene impuesto 30 (IVA) con estado "AC" → "Responsable Inscripto"
   - Si tiene impuesto 20 (Monotributo) con estado "AC" → "Monotributista"
   - Si tiene impuesto 30 con categoría exento → "Exento"
   - Fallback → "Consumidor Final"

4. Mapeo de provincias AFIP (código numérico → nombre):
   - 0 → "CABA", 1 → "Buenos Aires", 2 → "Catamarca", 3 → "Córdoba", etc.

## Notas / Consideraciones

- **WSDL de ws_sr_padron_a5**: la URL puede variar. Verificar en la documentación actual de AFIP. En homologación podría ser `https://awshomo.afip.gov.ar/sr-padron/webservices/personaServiceA5?WSDL`.
- **Rate limiting de AFIP**: no hacer consultas automáticas (en blur o en keyup). Solo con clic explícito del usuario.
- **Mapeo de condición IVA**: AFIP no devuelve directamente "Responsable Inscripto" como string — hay que inferirlo desde los impuestos inscriptos. El impuesto 30 = IVA, el 20 = Monotributo. Verificar con datos reales.
- **CUIT de personas físicas vs jurídicas**: la respuesta tiene estructura diferente (`persona.razonSocial` para jurídicas, `persona.apellido` + `persona.nombre` para físicas). Manejar ambos.
- **Certificado AFIP**: el mismo certificado y key que se usan para WSFEv1 sirven para ws_sr_padron_a5, pero el servicio debe estar habilitado en el portal de AFIP (Administración de Relaciones de Clave Fiscal). Si no está habilitado, la consulta va a fallar con error de autorización.
- **Documento de diseño UI**: `docs/designs/DISEÑO_v1.19.0-consulta-padron-afip.md` — leer antes de implementar los formularios.
