# v2.1.1 — Fix "Aplicar a todos" no persiste valores de referencia en muestras

**VERSIÓN:** v2.1.1  
**SLUG:** fix-aplicar-todos-ref-values  
**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 30min  
**PRIORIDAD:** crítica (bug reportado por cliente)

---

## Descripción

En la pantalla de carga de resultados de muestras (`/sample/{id}/load-results`),
el botón "Aplicar a todos" de la sección padre actualiza los `<select>` de valores
de referencia de las subdeterminaciones via JavaScript, pero **no guarda los cambios**.
El usuario ve el toast "X valores de referencia actualizados según..." pero al volver
a la página los valores están como antes.

### Causa raíz

1. **"Aplicar a todos" solo modifica el DOM** — La función `applyParentCategory()` en
   el `<script>` de `load-results.blade.php` actualiza los selects y hidden fields
   en el cliente, pero no hace submit del formulario. El usuario necesita clickear
   "Guardar Todo" después, lo cual no es obvio.

2. **Bug en `saveResults` con operador `??`** — En `SampleController::saveResults()`,
   el operador `??` impide actualizar campos a vacío/null porque cae al valor anterior:
   ```php
   'reference_value' => $data['reference_value'] ?? $determination->reference_value,
   ```
   Si el valor llega como `null` (middleware `ConvertEmptyStringsToNull`), se mantiene
   el valor viejo. Debería usar `array_key_exists` para distinguir "no enviado" de "vacío".

## PASO 1 — Fix: "Aplicar a todos" auto-guarda el formulario

En `resources/views/sample/load-results.blade.php`, modificar la función
`applyParentCategory()` para que después de actualizar los selects, haga submit
automático del formulario.

Al final de la función `applyParentCategory`, después del bloque que muestra
la notificación toast, agregar el submit del formulario:

```javascript
if (updatedCount > 0) {
    // ... (toast notification existente) ...
    
    // Auto-guardar: submit del formulario después de actualizar los selects
    setTimeout(function() {
        document.getElementById('resultsForm').submit();
    }, 500);
}
```

El `setTimeout` de 500ms da tiempo para que el usuario vea el toast antes del
reload de página. El submit envía todo el formulario (resultados + valores de
referencia actualizados + estados) al mismo endpoint `saveResults`.

**Verificación:** 
1. Seleccionar una normativa en el dropdown del padre
2. Click "Aplicar a todos"
3. Se ve el toast Y la página se recarga con "Resultados guardados correctamente"
4. Los valores de referencia persisten al volver a entrar

## PASO 2 — Fix: operador `??` en saveResults

En `app/Http/Controllers/SampleController.php`, método `saveResults()`,
reemplazar el bloque de construcción de `$updateData` (líneas ~334-341).

**Antes:**
```php
$updateData = [
    'result' => $data['result'] ?? $determination->result,
    'reference_value' => $data['reference_value'] ?? $determination->reference_value,
    'reference_category_id' => $data['reference_category_id'] ?? $determination->reference_category_id,
    'method' => $data['method'] ?? $determination->method,
    'observations' => $data['observations'] ?? $determination->observations,
    'status' => $data['status'] ?? $determination->status,
];
```

**Después:**
```php
$updateData = [
    'result' => array_key_exists('result', $data) ? $data['result'] : $determination->result,
    'reference_value' => array_key_exists('reference_value', $data) ? $data['reference_value'] : $determination->reference_value,
    'reference_category_id' => array_key_exists('reference_category_id', $data) ? $data['reference_category_id'] : $determination->reference_category_id,
    'method' => array_key_exists('method', $data) ? $data['method'] : $determination->method,
    'observations' => array_key_exists('observations', $data) ? $data['observations'] : $determination->observations,
    'status' => array_key_exists('status', $data) ? $data['status'] : $determination->status,
];
```

**Diferencia:** `array_key_exists` devuelve `true` si la key existe en el array,
incluso si su valor es `null`. Esto permite:
- Enviar `null` → el campo se limpia (se pone en NULL en la DB)
- No enviar el campo → se mantiene el valor anterior

**Verificación:**
1. En un protocolo, cargar un resultado con valor de referencia
2. Borrar el valor de referencia (dejar vacío)
3. Guardar
4. El valor de referencia debería estar vacío, no volver al anterior

## Notas / Consideraciones

- El auto-submit después de "Aplicar a todos" envía TODO el formulario, no solo los valores de referencia. Esto significa que si el usuario cambió otros campos (resultados, métodos, etc.) antes de aplicar la normativa, esos cambios también se guardan. Esto es el comportamiento esperado.
- El `setTimeout` de 500ms es para que el usuario vea el toast. Si se siente lento, se puede reducir a 300ms o eliminar y hacer submit inmediato.
- El fix del `??` también aplica a los campos `result`, `method` y `observations` — ahora todos se pueden limpiar correctamente.
