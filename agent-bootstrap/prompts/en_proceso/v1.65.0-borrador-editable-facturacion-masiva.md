# v1.65.0 — Borrador editable y líneas extras en facturación masiva

**VERSIÓN:** v1.65.0
**SLUG:** borrador-editable-facturacion-masiva
**DEPENDENCIAS:** ninguna (refactor sobre código existente)
**ESTIMACIÓN:** 4-6 horas
**PRIORIDAD:** media — fix UX del flujo de facturación masiva (caso pedido por el cliente)

---

## Descripción

Cambiar el flujo de `/billing/batch-preview` para que el botón "Crear factura" deje de
generar la factura final + AFIP en un paso, y en cambio cree un **borrador editable**
(`status = 'pendiente'` sin CAE) y redirija al `/sales-invoices/{id}/edit` existente
donde el usuario puede:

- Editar/quitar items individuales
- Agregar líneas libres (descripción + cantidad + precio + IVA) — útil para toma de muestra,
  flete, descuentos, otros conceptos no asociados a determinaciones de protocolos
- Cambiar customer, PV, fecha, notas, percepciones
- Apretar "Enviar a AFIP" (PV electrónico) o "Confirmar" (PV no electrónico)

**No requiere migración**, ni modelos nuevos, ni catálogos nuevos. Es refactor + ajustes
de UI sobre infraestructura existente.

## PASO 1 — Refactor de `BillingController::batchInvoice`

**Objetivo:** dejar de llamar a AFIP en línea. Siempre crear como borrador y redirigir
al edit.

En `app/Http/Controllers/BillingController.php`, método `batchInvoice` (línea 178+):

**Quitar** todo el bloque desde:

```php
if ($isElectronic) {
    $invoice->load(['pointOfSale', 'customer', 'items']);
    try {
        $afip = new AfipService;
        $result = $afip->createVoucher($invoice);
        ...
    }

return redirect()->route('sales-invoices.show', $invoice)
    ->with('success', "Factura creada con {$protocols->count()} protocolos.");
```

**Reemplazar** por:

```php

        .($isElectronic ? 'enviar a AFIP.' : 'confirmar.'));
```

El `invoice_number` queda como estaba: `'PENDIENTE-AFIP'` para electrónicas, `null` para
no electrónicas. El usuario lo asigna desde el edit (el form ya tiene el campo).
