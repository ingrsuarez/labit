## Descripción

Crea la infraestructura para vincular protocolos de laboratorio con facturas de venta y da
visibilidad sobre qué protocolos están facturados y cuáles no. Incluye:

1. **Tabla pivot polimórfica** `invoice_protocols` que liga `sales_invoices` con los 3 tipos
   de protocolo (`Admission`, `Sample`, `VetAdmission`).
2. **Vista "Protocolos sin facturar"** en ventas: tabla completa con filtros por módulo,
   obra social, rango de fechas y tipo de cliente.
3. **Resumen en sidebar del lab**: badge con contador de protocolos sin facturar.
4. **Botón "Facturar"** en la vista show de protocolo clínico (particular) y veterinario,
   que redirige al formulario de factura pre-llenado con los datos del protocolo.
5. **Regla de negocio**: toda factura a particular sale obligatoriamente de la empresa
   IPAC Laboratorios SAS (CUIT `30-71922759-3`).

**Contexto de negocio:**
- Los particulares se facturan en el momento (desde el protocolo).
- Las obras sociales se facturan a fin de mes (batch — va en v1.31.1).
- Las muestras de aguas se facturan al entregar informes (batch — va en v1.31.1).
- Los veterinarios pueden facturarse al dueño (inmediato) o a la veterinaria a fin de mes.

**Modelo de datos actual:**
- `SalesInvoice` tiene un campo `admission_id` (nullable, sin FK real) que enlaza débilmente
  con `Admission`. No hay enlace con `Sample` ni `VetAdmission`.
- `SalesInvoiceItem` tiene `test_id` que apunta a `Test`.
- Los protocolos no tienen campo `sales_invoice_id` ni estado de facturación.
- `Admission.isParticular()` retorna true si `insuranceRelation->type === 'particular'`.
- `Customer.type` es un array JSON (`['particular', 'veterinario', 'aguas', ...]`).

public function invoiceProtocols()
{
    return $this->morphMany(InvoiceProtocol::class, 'protocol');
}

public function salesInvoices()
{
    return $this->morphToMany(SalesInvoice::class, 'protocol', 'invoice_protocols')
        ->withPivot('amount')
        ->withTimestamps();
}

public function isInvoiced(): bool
{
    return $this->invoiceProtocols()->exists();
}

public function scopeUninvoiced($query)
{
    return $query->whereDoesntHave('invoiceProtocols');
}

public function scopeInvoiced($query)
{
    return $query->whereHas('invoiceProtocols');
}

## PASO 4 — Controller `BillingController` para vista de protocolos sin facturar

### 4.1 — Crear `app/Http/Controllers/BillingController.php`

```php
namespace App\Http\Controllers;

use App\Models\Admission;
use App\Models\Sample;
use App\Models\VetAdmission;
use App\Models\Insurance;
use App\Models\Customer;
use Illuminate\Http\Request;

class BillingController extends Controller
{
    public function uninvoiced(Request $request)
    {
        $this->authorize('sales-invoices.index');

        $module = $request->get('module', 'all');
        $dateFrom = $request->get('date_from');
        $dateTo = $request->get('date_to');
        $insuranceId = $request->get('insurance_id');
        $customerId = $request->get('customer_id');

        $admissions = collect();
        $samples = collect();
        $vetAdmissions = collect();

        if (in_array($module, ['all', 'clinico'])) {
            $q = Admission::uninvoiced()
                ->with(['patient', 'insuranceRelation', 'admissionTests.test', 'labBranch']);
            if ($dateFrom) $q->whereDate('date', '>=', $dateFrom);
            if ($dateTo) $q->whereDate('date', '<=', $dateTo);
            if ($insuranceId) $q->where('insurance', $insuranceId);
            $admissions = $q->orderBy('date', 'desc')->get();
        }

        if (in_array($module, ['all', 'aguas'])) {
            $q = Sample::uninvoiced()
                ->with(['customer', 'determinations', 'labBranch']);
            if ($dateFrom) $q->whereDate('entry_date', '>=', $dateFrom);
            if ($dateTo) $q->whereDate('entry_date', '<=', $dateTo);
            if ($customerId) $q->where('customer_id', $customerId);
            $samples = $q->orderBy('entry_date', 'desc')->get();
        }

        if (in_array($module, ['all', 'veterinario'])) {
            $q = VetAdmission::uninvoiced()
                ->with(['customer', 'species', 'labBranch']);
            if ($dateFrom) $q->whereDate('date', '>=', $dateFrom);
            if ($dateTo) $q->whereDate('date', '<=', $dateTo);
            if ($customerId) $q->where('customer_id', $customerId);
            $vetAdmissions = $q->orderBy('date', 'desc')->get();
        }

        $insurances = Insurance::where('type', '!=', 'nomenclador')
            ->orderByRaw("CASE WHEN type = 'particular' THEN 0 ELSE 1 END")
            ->orderBy('name')->get();
        $customers = Customer::where('status', 'activo')->orderBy('name')->get();

        $totals = [
            'clinico' => $admissions->sum(fn ($a) => $a->total_patient ?: $a->patient_price ?: 0),
            'aguas' => $samples->sum(fn ($s) => $s->determinations->sum('price')),
            'veterinario' => $vetAdmissions->sum('total_price'),
        ];

        return view('billing.uninvoiced', compact(
            'admissions', 'samples', 'vetAdmissions',
            'insurances', 'customers', 'module', 'totals'
        ));
    }

    public function summary()
    {
        return response()->json([
            'clinico' => Admission::uninvoiced()->count(),
            'aguas' => Sample::uninvoiced()->count(),
            'veterinario' => VetAdmission::uninvoiced()->count(),
        ]);
    }
}
```

## PASO 7 — Botón "Facturar" en show de protocolo + pre-llenado

### 7.1 — Método `createFromProtocol` en `SalesInvoiceController`

Agregar un método que recibe el tipo de protocolo y su ID, y redirige al formulario de
creación de factura con datos pre-llenados:

```php
public function createFromProtocol(Request $request)
{
    $this->authorize('sales-invoices.create');

    $type = $request->get('protocol_type');
    $id = $request->get('protocol_id');

    $items = [];
    $customerId = null;
    $voucherType = 'B';

    if ($type === 'admission') {
        $protocol = Admission::with(['patient', 'admissionTests.test', 'insuranceRelation'])->findOrFail($id);
        $customerId = $protocol->patient_id;

        foreach ($protocol->admissionTests as $at) {
            $items[] = [
                'description' => $at->test?->name ?? 'Determinación',
                'test_id' => $at->test_id,
                'quantity' => 1,
                'unit_price' => $at->paid_by_patient ? ($at->copago ?: $at->price) : $at->price,
                'iva_rate' => 21,
            ];
        }
    } elseif ($type === 'vet_admission') {
        $protocol = VetAdmission::with(['customer', 'tests.test'])->findOrFail($id);
        $customerId = $protocol->customer_id;

        foreach ($protocol->tests as $vt) {
            $items[] = [
                'description' => $vt->test?->name ?? 'Determinación',
                'test_id' => $vt->test_id,
                'quantity' => 1,
                'unit_price' => $vt->price,
                'iva_rate' => 21,
            ];
        }
    }

    // Determinar tipo de factura por condición IVA del cliente
    if ($customerId) {
        $customer = Customer::find($customerId);
        if ($customer && in_array($customer->iva_condition, ['IVA Responsable Inscripto', 'Responsable Inscripto'])) {
            $voucherType = 'A';
        }
    }

    // Forzar empresa IPAC SAS para particulares
    $forceCompanyId = null;
    if ($type === 'admission' && isset($protocol) && $protocol->isParticular()) {
        $forceCompanyId = ipac_sas_company_id();
    }

    $customers = Customer::where('status', 'activo')->orderBy('name')->get();
    $pointsOfSale = PointOfSale::where('is_active', true)
        ->where('company_id', $forceCompanyId ?? active_company_id())
        ->orderBy('code')->get();

    return view('sales-invoices.create', [
        'customers' => $customers,
        'pointsOfSale' => $pointsOfSale,
        'quote' => null,
        'selectedCustomerId' => $customerId,
        'prefillItems' => $items,
        'prefillVoucherType' => $voucherType,
        'protocolType' => $type,
        'protocolId' => $id,
        'protocolNumber' => $protocol->protocol_number ?? null,
        'forceCompanyId' => $forceCompanyId,
    ]);
}
```

## Notas / Consideraciones

- La tabla `invoice_protocols` es polimórfica para soportar los 3 tipos de protocolo con una sola tabla.
  Usa `protocol_type` (class name completo) y `protocol_id`.
- El campo `admission_id` existente en `sales_invoices` se mantiene por compatibilidad pero la fuente
  de verdad para el vínculo protocolo↔factura es ahora `invoice_protocols`.
- `ipac_sas_company_id()` usa cache estática para no hacer query repetidos. Si la empresa no existe,
  fallback a `active_company_id()`.
- La regla "particulares siempre IPAC SAS" implica que al facturar, los puntos de venta mostrados
  son los de IPAC SAS, no los de la empresa activa del usuario. Esto puede requerir que la
  recepcionista esté asociada a la empresa IPAC SAS (ver `company_user`).
- Los protocolos de obras sociales NO muestran botón "Facturar" en esta versión. La facturación
  batch de OS va en v1.31.1.
- En producción, habilitar permisos para recepcionistas con tinker (ver PASO 8).
- Para que los puntos de venta de IPAC SAS estén disponibles, el usuario debe estar asociado
  a esa empresa en `company_user`. Documentar esto.
