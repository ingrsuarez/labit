# v1.82.0 — PayrollPayment: pago de haberes agrupado + asiento contable automático

**VERSIÓN:** v1.82.0  
**SLUG:** payroll-payment-pago-haberes-asiento  
**DEPENDENCIAS:** v2.3.0 (RRHH multi-empresa), v3.0.0 (infra contable base), v3.3.0 (AccountingEntryService)  
**ESTIMACIÓN:** 1 día  
**PRIORIDAD:** alta

---

## Descripción

Introduce el modelo `PayrollPayment` que agrupa N liquidaciones de sueldo (`Payroll`)
en un único pago de haberes. Esto resuelve la realidad operativa donde el banco registra
**un débito por lote de empleados** (ej: uno para empleados del mismo banco, otro para
transferencias interbancarias, otro para liquidaciones finales).

Al confirmar un `PayrollPayment`:
1. Todas las liquidaciones del lote se marcan como `pagado`
2. Se genera el asiento contable automático: **Db 2.1.07 Sueldos a Pagar / Cr cuenta bancaria**
3. El `PayrollPayment` implementa `reconciledMovements()` (igual que `CollectionReceipt`),
   lo que lo hace reconciliable con movimientos bancarios en v1.83.0

**Regla central:** un `Payroll` solo puede pertenecer a un `PayrollPayment` a la vez.
No hay splits. Múltiples `PayrollPayment` pueden existir para el mismo período (mes/año).

### 1.2 — Modelo PayrollPayment

Crear `app/Models/PayrollPayment.php`:

```php
class PayrollPayment extends Model
{
    // fillable: todos los campos de la tabla
    
    // Relaciones
    public function payrolls(): HasMany          // payrolls where payroll_payment_id = this->id
    public function bankAccount(): BelongsTo
    public function company(): BelongsTo
    public function creator(): BelongsTo         // created_by → users
    public function confirmer(): BelongsTo        // confirmed_by → users
    
    // Para conciliación bancaria (igual que CollectionReceipt)
    public function reconciledMovements(): MorphMany
    {
        return $this->morphMany(BankMovement::class, 'reconciled', 'reconciled_type', 'reconciled_id');
    }
    
    public function isConfirmado(): bool
    {
        return $this->status === 'confirmado';
    }
    
    // Recalcular total y employee_count desde las liquidaciones vinculadas
    public function recalculate(): void
    {
        $this->total = $this->payrolls()->sum('neto_a_cobrar');
        $this->employee_count = $this->payrolls()->count();
        $this->save();
    }
}
```

Agregar en `Payroll`:

```php
public function payrollPayment(): BelongsTo
{
    return $this->belongsTo(PayrollPayment::class);
}
```

## PASO 2 — AccountingEntryService: método fromPayrollPayment

En `app/Services/AccountingEntryService.php` agregar el método:

```php
public function fromPayrollPayment(PayrollPayment $payment): ?JournalEntry
```

**Asiento:**

| Cuenta | Db | Cr | Descripción línea |
|---|---|---|---|
| `2.1.07` — Sueldos a Pagar | `$payment->total` | — | "Pago haberes {$payment->period_label}" |
| cuenta bancaria (`bank_account.accounting_account_id`) | — | `$payment->total` | "Pago haberes {$payment->period_label}" |

**Precondiciones (iguales al patrón existente):**
- Si la cuenta `2.1.07` no existe → log warning, retornar null, operación continúa
- Si `bank_account_id` es null o la cuenta bancaria no tiene `accounting_account_id` → log warning, retornar null
- El asiento no se genera si ya existe un `JournalEntry` con `source_type = PayrollPayment::class` y `source_id = $payment->id` (idempotencia)

## PASO 3 — PayrollPaymentController

Crear `app/Http/Controllers/PayrollPaymentController.php`.

### Rutas a agregar en `routes/web.php`

```php
Route::prefix('rrhh/pagos-haberes')->name('payroll-payments.')->group(function () {
    Route::get('/', [PayrollPaymentController::class, 'index'])->name('index');
    Route::get('/create', [PayrollPaymentController::class, 'create'])->name('create');
    Route::post('/', [PayrollPaymentController::class, 'store'])->name('store');
    Route::get('/{payrollPayment}', [PayrollPaymentController::class, 'show'])->name('show');
    Route::delete('/{payrollPayment}', [PayrollPaymentController::class, 'destroy'])->name('destroy');
    Route::post('/{payrollPayment}/confirm', [PayrollPaymentController::class, 'confirm'])->name('confirm');
});
```

Todas las rutas requieren middleware `auth` y permiso `payroll-payments.manage`.

### Métodos del controlador

**`index()`**
- Listar `PayrollPayment` de la empresa activa, ordenados por `year DESC, month DESC, id DESC`
- Pasar a la vista: `payrollPayments` paginado (15/página), total de importes, badge de estado

**`create()`**
- Recibe parámetros opcionales `?year=&month=` para preseleccionar período
- Carga `PayrollPayment` borrador ya existentes del período (para que el usuario sepa cuántos lotes ya se crearon)
- Carga `Payroll` en estado `liquidado` sin `payroll_payment_id` asignado para el período seleccionado
- Carga `BankAccount` de la empresa activa
- Si no hay período seleccionado, preseleccionar el mes actual

**`store(Request $request)`**

Validar:
- `year`: required integer
- `month`: required integer min:1 max:12
- `bank_account_id`: nullable, exists en bank_accounts
- `payment_date`: nullable, date
- `notes`: nullable, string, max:500
- `payroll_ids`: required, array, min:1
- `payroll_ids.*`: exists en payrolls, verificar que cada uno pertenece a la empresa activa, tiene status `liquidado`, y no tiene `payroll_payment_id` ya asignado

Lógica:
1. Crear `PayrollPayment` con status `borrador`
2. Asignar `payroll_payment_id` en cada `Payroll` seleccionado
3. Llamar `$payment->recalculate()` para calcular total y employee_count
4. Redirect a `show`

**`show(PayrollPayment $payrollPayment)`**
- Verificar `company_id === active_company_id()`, abort 403 si no
- Cargar relaciones: `payrolls.employee`, `bankAccount`, `creator`, `confirmer`
- Cargar `JournalEntry` si existe (`source_type = PayrollPayment::class, source_id = $id`)
- Cargar `reconciledMovements` para mostrar si ya fue conciliado con el banco

**`confirm(PayrollPayment $payrollPayment)`**
- Verificar que el pago está en `borrador` y tiene al menos una liquidación vinculada
- En transacción DB:
  1. Actualizar `PayrollPayment` → status `confirmado`, `confirmed_at = now()`, `confirmed_by = auth()->id()`
  2. Actualizar `payrolls` del lote → `status = 'pagado'`, `paid_at = now()`
  3. Llamar `AccountingEntryService::fromPayrollPayment($payrollPayment)`
- Redirect a `show` con mensaje de éxito

**`destroy(PayrollPayment $payrollPayment)`**
- Solo si está en `borrador`
- Desasignar `payroll_payment_id` en las liquidaciones vinculadas (set null)
- Eliminar el `PayrollPayment`
- Redirect a `index`
