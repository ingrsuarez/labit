# v1.70.0 — Sección Pacientes en el lab clínico (lista + sidebar)

**Tipo:** minor — nueva funcionalidad (punto de entrada que faltaba)
**Prioridad:** alta — bug de UX en producción, el cliente no puede editar pacientes
**Dependencias:** ninguna (formulario de edición ya existe, permisos ya están en seeder)

---

## Qué hace

Agrega "Pacientes" al sidebar del laboratorio y crea una página moderna de listado
(`/lab/patients`) con búsqueda, paginación y botones Editar + Nueva Admisión.

El formulario de edición (`/patient/edit?current_patient=DNI`) ya existe y funciona.
El problema es que el sidebar del nuevo layout (`lab/partials/sidebar.blade.php`)
no tiene ningún link a Pacientes, haciendo la función completamente inalcanzable
desde la UI actual.

Los permisos `patients.index` y `patients.edit` ya existen en el seeder y están
asignados a los roles `recepcion-lab` y `bioquimico`.

## PASO 1 — Crear `LabPatientController`

Crear `app/Http/Controllers/LabPatientController.php`:

```php
<?php

namespace App\Http\Controllers;

use App\Models\Insurance;
use App\Models\Patient;
use Illuminate\Http\Request;

class LabPatientController extends Controller
{
    public function index(Request $request)
    {
        $this->authorize('patients.index');

        $query = Patient::with('insuranceRelation')
            ->orderBy('lastName')
            ->orderBy('name');

        if ($request->filled('search')) {
            $search = $request->search;
            $query->where(function ($q) use ($search) {
                $q->where('patientId', 'like', "%{$search}%")
                  ->orWhereRaw('lower(name) LIKE ?', ['%'.strtolower($search).'%'])
                  ->orWhereRaw('lower(lastName) LIKE ?', ['%'.strtolower($search).'%']);
            });
        }

        if ($request->filled('insurance')) {
            $query->where('insurance', $request->insurance);
        }

        $patients = $query->paginate(20)->withQueryString();

        $insurances = Insurance::where('type', '!=', 'nomenclador')
            ->orderByRaw("CASE WHEN type = 'particular' THEN 0 ELSE 1 END")
            ->orderBy('name')
            ->get();

        return view('lab.patients.index', compact('patients', 'insurances'));
    }
}
```

## PASO 3 — Crear vista `lab/patients/index.blade.php`

Crear `resources/views/lab/patients/index.blade.php`.

Usar la misma estructura visual que `lab/admissions/index.blade.php`:
- Header con título "Pacientes" y descripción "Gestione los pacientes del laboratorio"
- Botón "+ Nuevo Paciente" (`@can('patients.create')`) → `route('patient.index')`
- Barra de filtros con:
  - Input búsqueda: "Buscar por nombre, apellido o DNI..."  (`name="search"`)
  - Select obra social (`name="insurance"`)
  - Botón Filtrar + link Limpiar (si hay filtros activos)
- Tabla con columnas: DNI | Nombre y Apellido | Obra Social | Teléfono | Email | Acciones
- Por cada fila (`@foreach $patients as $patient`):
  - DNI: `{{ $patient->patientId }}`
  - Nombre: `{{ $patient->full_name }}`
  - Obra social: `{{ strtoupper($patient->insuranceRelation?->name ?? '—') }}`
  - Teléfono: `{{ $patient->phone ?? '—' }}`
  - Email: `{{ $patient->email ?? '—' }}`
  - Acciones:
    ```blade
    @can('patients.edit')
    <a href="{{ route('patient.edit', ['current_patient' => $patient->patientId]) }}"
       class="text-teal-600 hover:text-teal-800 text-sm font-medium">
        Editar
    </a>
    @endcan
    @can('lab-admissions.create')
    <a href="{{ route('lab.admissions.create', ['patient_id' => $patient->id]) }}"
       class="text-blue-600 hover:text-blue-800 text-sm ml-3"
       title="Nueva admisión para este paciente">
        + Admisión
    </a>
    @endcan
    ```
- Paginación al pie: `{{ $patients->links() }}`
- Estado vacío si no hay resultados (mismo patrón que admissions/index)
