## Descripción

Reemplaza el selector de determinaciones veterinarias (tabla con checkboxes y filtrado client-side)
por un buscador tipo dropdown con Enter para agregar, idéntico al del laboratorio clínico.
Agrega el campo `owner_email` que falta para poder enviar resultados por email al dueño del animal.

**Problema actual:** La admisión veterinaria usa una tabla con todos los tests renderizados en el HTML
y checkboxes para seleccionar. No soporta Enter, no tiene dropdown de sugerencias, y escala mal
si hay muchos tests. El lab clínico tiene un buscador asíncrono con dropdown y Enter que es mucho más ágil.

**Referencia del patrón correcto:** `resources/views/lab/admissions/create.blade.php` líneas 190-230 (HTML)
y 468-521 (JS: `searchTests`, `selectFirstTest`, `addTest`).

### 2.1 — Agregar método `searchTests`

Crear un endpoint de búsqueda que filtre tests con categoría `veterinario`:

```php
public function searchTests(Request $request)
{
    $query = $request->get('q', '');
    if (strlen($query) < 2) {
        return response()->json([]);
    }

    $tests = Test::where(function ($q) use ($query) {
            $q->where('code', 'like', "%{$query}%")
              ->orWhere('name', 'like', "%{$query}%");
        })
        ->whereJsonContains('categories', 'veterinario')
        ->whereNull('parent_id') // Solo padres y standalone, no hijos
        ->where('is_active', true)
        ->limit(20)
        ->get(['id', 'code', 'name', 'price', 'tracks_lot']);

    return response()->json($tests);
}
```

**Nota:** Verificar si los tests veterinarios usan `parent_id` o la tabla pivot `test_parents`.
Si usan pivot, el filtro para excluir hijos sería diferente:
`->whereDoesntHave('parents')` en vez de `->whereNull('parent_id')`.
Revisar el modelo `Test` para confirmar. El objetivo es que el buscador devuelva solo padres
y standalone, porque al agregarlos, `store()` ya expande los hijos con `getAllChildren()`.

### 3.2 — Alpine.js actualizado

Reemplazar `vetAdmissionForm()` completo:

```js
function vetAdmissionForm() {
    return {
        customerId: '{{ old("customer_id", "") }}',
        veterinarianId: '{{ old("veterinarian_id", "") }}',
        veterinarians: [],
        testSearch: '',
        filteredTests: [],
        showTestDropdown: false,
        testsData: [],
        totalPrice: 0,

        init() {
            if (this.customerId) this.loadVeterinarians();
        },

        async loadVeterinarians() {
            this.veterinarians = [];
            this.veterinarianId = '';
            if (!this.customerId) return;
            try {
                const res = await fetch(`{{ url('vet/customer') }}/${this.customerId}/veterinarians`);
                this.veterinarians = await res.json();
            } catch (e) { console.error(e); }
        },

        async searchTests() {
            if (this.testSearch.length < 2) {
                this.filteredTests = [];
                return;
            }
            try {
                const response = await fetch(`{{ route('vet.admissions.searchTests') }}?q=${encodeURIComponent(this.testSearch)}`);
                const tests = await response.json();
                this.filteredTests = tests.filter(t => !this.testsData.find(td => td.test_id === t.id));
                this.showTestDropdown = true;
            } catch (error) {
                console.error('Error buscando tests:', error);
            }
        },

        selectFirstTest() {
            if (this.filteredTests.length > 0) {
                this.addTest(this.filteredTests[0]);
            }
        },

        addTest(test) {
            if (this.testsData.find(t => t.test_id === test.id)) {
                this.testSearch = '';
                this.filteredTests = [];
                this.showTestDropdown = false;
                return;
            }

            this.testsData.push({
                test_id: test.id,
                code: test.code,
                name: test.name,
                price: parseFloat(test.price || 0),
            });

            this.testSearch = '';
            this.filteredTests = [];
            this.showTestDropdown = false;
            this.totalPrice = this.testsData.reduce((sum, t) => sum + t.price, 0);

            // Re-enfocar el buscador para seguir agregando
            this.$nextTick(() => {
                this.$refs.testSearchInput.focus();
            });
        },

        removeTest(index) {
            this.testsData.splice(index, 1);
            this.totalPrice = this.testsData.reduce((sum, t) => sum + t.price, 0);
        },
    }
}
```

**Nota:** Ya no se necesita `selectedTests` (array de checkbox values) ni `toggleTest()`.
El array `$tests` ya no se pasa desde el controller al Blade (se remueve de `create()`).

**Verificación:** En el formulario de nueva admisión vet, escribir "hemo" → aparece dropdown → Enter → se agrega Hemograma → el foco vuelve al buscador → escribir siguiente determinación.

## Notas / Consideraciones

- El patrón replicado es idéntico al de `lab/admissions/create.blade.php` adaptado al estilo
  amber del módulo veterinario.
- A diferencia del lab clínico, la búsqueda veterinaria NO depende de obra social — busca
  directamente todos los tests con categoría `veterinario`.
- Los hijos de tests padre se siguen expandiendo automáticamente en `store()` con `getAllChildren()` —
  el buscador solo muestra padres y standalone.
- El `owner_email` es nullable porque no siempre se tiene el email del dueño al momento de la admisión.
- El envío de resultados por email (`sendEmail` en `VetAdmissionController`) debería usar
  `owner_email` como destinatario. Verificar si ya lo hace o si usa otro campo.
