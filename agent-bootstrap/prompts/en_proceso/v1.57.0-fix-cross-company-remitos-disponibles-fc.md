# v1.57.0 — Fix cross-company: remitos visibles en FC sin importar empresa activa

**VERSIÓN:** v1.57.0
**SLUG:** fix-cross-company-remitos-disponibles-fc
**DEPENDENCIAS:** v2.0.0 (multi-empresa)
**ESTIMACIÓN:** 15min
**PRIORIDAD:** urgente

---

## Descripción

Al crear una Factura de Compra, el endpoint `availableDeliveryNotes` filtra los remitos
por `company_id = empresa_activa`. Si el usuario creó el remito mientras tenía activa
la empresa A y ahora está en empresa B, el remito no aparece en el selector.

El fix reemplaza el filtro por empresa activa con un filtro por **todas las empresas
del usuario autenticado**, de modo que los remitos de cualquier empresa a la que pertenezca
el usuario sean visibles al crear una FC.

**Root cause:** línea ~123 en `PurchaseInvoiceController::availableDeliveryNotes()`:
```php
->where('company_id', $companyId)   // ← solo empresa activa
```

**Fix:**
```php
->whereIn('company_id', auth()->user()->companies->pluck('id'))   // ← todas las empresas del usuario
```

## PASO 1 — Corregir availableDeliveryNotes() en PurchaseInvoiceController

Archivo: `app/Http/Controllers/PurchaseInvoiceController.php`

Localizar el método `availableDeliveryNotes()`. La consulta de remitos actualmente filtra
por `company_id`:

```php
$notes = DeliveryNote::query()
    ->where('company_id', $companyId)
    ->where('supplier_id', $supplierId)
    ->where('status', 'aceptado')
    ->orderByDesc('date')
    ->get(['id', 'remito_number', 'date', 'lab_branch_id']);
```

Cambiar **únicamente** la línea `->where('company_id', $companyId)` por un `whereIn`
con todas las empresas del usuario:

```php
$userCompanyIds = auth()->user()->companies->pluck('id');

$notes = DeliveryNote::query()
    ->whereIn('company_id', $userCompanyIds)
    ->where('supplier_id', $supplierId)
    ->where('status', 'aceptado')
    ->orderByDesc('date')
    ->get(['id', 'remito_number', 'date', 'lab_branch_id']);
```

La variable `$companyId` sigue usándose más arriba para la lógica del `$excludePi` —
no eliminarla, solo dejar de usarla en la consulta de remitos.

**Verificación PASO 1:**
- Usuario con dos empresas (ej: IPAC SAS y Distribuidora Uruguay)
- Crear un remito en empresa A con proveedor X
- Cambiar a empresa B
- Abrir Nueva FC con proveedor X
- El remito de empresa A aparece en el selector "Agregar remito"

## PASO 2 — Verificar que la relación `companies` existe en User

Confirmar que `auth()->user()->companies` está eager-loaded o cargada antes de usar
`->pluck('id')`. Si la relación puede no estar cargada en este contexto, usar:

```php
$userCompanyIds = auth()->user()->load('companies')->companies->pluck('id');
```

O alternativamente, si la relación ya viene cargada en el middleware de selección de
empresa activa, usar directamente `auth()->user()->companies->pluck('id')`.

Verificar en el modelo `User.php` cómo se define la relación y si hay un scope
o middleware que la pre-carga. Ajustar según corresponda para no duplicar queries.
