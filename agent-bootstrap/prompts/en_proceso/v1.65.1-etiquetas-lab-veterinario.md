**VERSIÓN:** v1.65.1
**SLUG:** etiquetas-lab-veterinario
**DEPENDENCIAS:** ninguna (infraestructura Zebra, BarcodeFormatService y modal genérico ya existen)
**ESTIMACIÓN:** 30min–1h
**PRIORIDAD:** hotfix — gap funcional detectado; el lab clínico tiene etiquetas desde v1.18.0 pero el veterinario no

---

## Descripción

Agrega impresión de etiquetas Zebra para tubos en protocolos veterinarios (`VetAdmission`).
Una etiqueta por material distinto, exactamente como funciona en lab clínico (v1.18.0 + v1.18.1 + v1.48.5).

Reutiliza toda la infraestructura existente:
- `public/js/zebra-label-print.js` y componente Alpine `zebraPrintModal()` — sin modificar
- `App\Services\BarcodeFormatService::forLabel()` — para formato extendido `protocol_number^material`
- Vista fallback HTML — réplica de `lab/admissions/label.blade.php`

El campo `customer_name` de la etiqueta será el **nombre y apellido del dueño** (`owner_name` del `VetAdmission`).

## PASO 2 — Métodos en `VetAdmissionController`

En `app/Http/Controllers/VetAdmissionController.php`, agregar 3 métodos replicando la estructura exacta de `LabAdmissionController` pero adaptada a `VetAdmission`:

### 2.1 — Método privado `groupByMaterial(VetAdmission $vetAdmission): array`

Replicar la lógica de `LabAdmissionController::groupByMaterial()` adaptada:

- Usar `$vetAdmission->vetTests` en vez de `$admission->admissionTests`
- Cada `VetAdmissionTest` tiene relación `test()` → `Test` (con `materialRelation` y `material_abbreviation`)
- Cada `VetAdmissionTest` tiene relación `test.parentTests` para filtrar hijos (evitar etiquetas duplicadas)
- Obtener `$parentTestIds` de los `test_id` de los vetTests para filtrar hijos
- Para cada vetTest que tenga material y no sea hijo de otro test en el protocolo, agrupar por `materialRelation->id`
- El `customer_name` en cada label debe ser: `$vetAdmission->owner_name` (nombre y apellido del dueño)
- El `sample_type` debe ser: `'VETERINARIO'`
- El `entry_date` debe ser: `$vetAdmission->date->format('d/m/Y')`
- El `branch_name` debe venir de: `$vetAdmission->labBranch->name ?? 'VETERINARIO'`
- Fallback si no hay materiales: generar una etiqueta con `material = '?'` y `material_name = 'Sin material'`

### 2.2 — `labelData(VetAdmission $vetAdmission)`

```php
public function labelData(VetAdmission $vetAdmission)
{
    $this->authorize('vet-labels.print');

    $vetAdmission->load(['vetTests.test.materialRelation', 'vetTests.test.parentTests', 'labBranch']);

    $labels = $this->groupByMaterial($vetAdmission);

    return response()->json([
        'labels' => $labels,
        'total_labels' => count($labels),
    ]);
}
```

### 2.3 — `printLabel(VetAdmission $vetAdmission)`

Réplica de `LabAdmissionController::printLabel()` adaptada:

```php
public function printLabel(VetAdmission $vetAdmission)
{
    $this->authorize('vet-labels.print');

    $vetAdmission->load(['vetTests.test.materialRelation', 'vetTests.test.parentTests', 'labBranch']);

    $labels = $this->groupByMaterial($vetAdmission);

    $barcode = new \Picqer\Barcode\BarcodeGeneratorSVG;

    foreach ($labels as &$label) {
        $content = \App\Services\BarcodeFormatService::forLabel(
            $vetAdmission->protocol_number,
            $label['material'] ?? null
        );

        $label['barcode_content'] = $content;
        $label['barcode_svg'] = $barcode->getBarcode(
            $content,
            $barcode::TYPE_CODE_128,
            2,
            60
        );
    }
    unset($label);

    return view('vet.admissions.label', [
        'vetAdmission' => $vetAdmission,
        'labels' => $labels,
    ]);
}
```

**Verificación:** Los métodos existen. `labelData` devuelve JSON con array `labels`. `printLabel` renderiza HTML.

## Notas / Consideraciones

- El campo `customer_name` en el JSON se llama así (no `owner_name`) para mantener compatibilidad con `generateZPL()` que espera ese nombre de campo. Es un detalle de interfaz, no de dominio.
- Los materiales se obtienen del accessor `material_abbreviation` del modelo `Test` via la relación `vetTest.test.materialRelation`.
- Si un protocolo vet tiene tests con distintos materiales (ej: hemograma=EDTA + bioquímica=SUE), se generan N etiquetas (una por material).
- Si no hay materiales asignados a los tests, se genera una etiqueta fallback con material `?`.
- El botón de etiqueta NO se agrega al index de vet admissions en esta versión (solo al show). Si se necesita en el index, planificar como v1.65.2.
- No se modifica el JS de Zebra ni el modal genérico — la infraestructura es 100% reutilizable.
