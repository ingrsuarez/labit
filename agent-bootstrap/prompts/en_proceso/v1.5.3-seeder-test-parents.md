## Descripción

Crear un seeder que configure las relaciones padre-hijo en la tabla `test_parents`
para las prácticas del laboratorio clínico, basado en el protocolo ejemplo
(`docs/Protocolo-ejemplo.pdf`).

Las prácticas ya existen en la tabla `tests`. El seeder solo crea las relaciones
en `test_parents` (parent_test_id, child_test_id, order).

**Búsqueda por nombre con LIKE** — cada práctica se busca con
`Test::where('name', 'LIKE', '%nombre%')->first()`. Si no encuentra o encuentra
múltiples, loguea warning y salta esa relación sin romper.

## PASO 1 — Crear el seeder TestParentChildSeeder

Crear `database/seeders/TestParentChildSeeder.php` con esta lógica:

1. Definir un array asociativo `$hierarchy` donde la key es el nombre del padre
   y el value es un array ordenado de nombres de hijas:

```php
$hierarchy = [
    'hemograma completo' => [
        'globulos rojos',
        'hemoglobina',
        'hematocrito',
        'vcm',
        'hcm',
        'chcm',
        'rdw-cv',
        'globulos blancos',
    ],
    'formula leucocitaria' => [
        'neutrofilos segmentados',
        'eosinofilos',
        'basofilos',
        'linfocitos',
        'monocitos',
    ],
    'hepatograma' => [
        'bilirrubina total',
        'bilirrubina directa',
        'bilirrubina indirecta',
        'got - transaminasa glutamico oxalacetica',
        'gpt - transaminasa glutamico piruvica',  // o "gtp"
        'fosfatasa alcalina',
        'colesterol total',
        'proteinas totales',
    ],
    'orina completa' => [
        'color',
        'aspecto',
        'densidad',
        'ph',
        'glucosa',
        'cetonas',
        'proteinas',
        'bilirrubina',
        'urobilina',
        'hemoglobina',
        'nitritos',
        'celulas epiteliales',
        'leucocitos',
        'eritrocitos',
    ],
    'drogas x 2' => [
        'cocaina',
        'marihuana',
    ],
];
```

2. Para cada padre en `$hierarchy`:
   - Buscar el test padre: `Test::where('name', 'LIKE', "%{$parentName}%")->first()`
   - Si no lo encuentra → `$this->command->warn("Padre no encontrado: {$parentName}")` y continuar
   - Si encuentra múltiples → `$this->command->warn("Múltiples matches para padre: {$parentName}")` y continuar
   - Para cada hija (con índice como order):
     - Buscar: `Test::where('name', 'LIKE', "%{$childName}%")->first()`
     - Si no la encuentra → warn y continuar
     - Verificar que la relación no exista ya: `DB::table('test_parents')->where(...)->exists()`
     - Si no existe → insertar en `test_parents` con `order` = posición en el array
     - Si ya existe → `$this->command->info("Relación ya existe: ...")`

3. Al final, mostrar resumen:
   ```
   $this->command->info("Relaciones creadas: {$created}");
   $this->command->info("Ya existentes: {$skipped}");
   $this->command->warn("No encontradas: {$notFound}");
   ```

> ⚠️ IMPORTANTE: Para hijas que pueden tener nombres ambiguos en la DB (como "color",
> "aspecto", "ph" que son muy genéricos), el LIKE puede matchear múltiples registros.
> En ese caso usar una búsqueda más específica o agregar contexto (ej: buscar por
> `work_sheet` si corresponde). Revisar los datos reales de la tabla `tests` antes
> de ejecutar y ajustar los nombres de búsqueda si es necesario.

**Verificación:** El seeder corre sin errores y muestra el resumen con 37 relaciones creadas.

## Notas / Consideraciones

- Los nombres en el array del seeder están en minúscula porque el `TestController::store()` guarda con `strtolower()`. Si en producción hay nombres con capitalización diferente, el LIKE los matchea igual (MySQL es case-insensitive por defecto en collation `utf8mb4_unicode_ci`).
- Si alguna hija tiene un nombre muy genérico (ej: "color", "ph"), puede matchear prácticas de otros contextos. En ese caso, agregar lógica adicional de filtro (por `work_sheet`, `material`, o verificar que no tenga hijos propios).
- El campo legacy `parent` en la tabla `tests` NO se modifica — se usa exclusivamente la tabla pivote `test_parents`.
- El seeder NO se agrega al `DatabaseSeeder.php` porque es una corrida puntual de datos, no para ambientes nuevos.
