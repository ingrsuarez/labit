# v1.73.0 — Estado "enviado" en protocolos de muestras

**Tipo:** minor — nueva funcionalidad
**Prioridad:** alta — base para v1.74.0 (envío masivo)
**Dependencias:** ninguna

---

## Qué hace

Agrega el estado `enviado` al ciclo de vida de los protocolos de aguas/alimentos (`Sample`).
Cuando un protocolo validado se envía por email individual (`sendEmail()`) o se descarga
como PDF (`downloadPdf()`), el protocolo pasa automáticamente al estado `enviado`.

El estado se almacena en una nueva columna `sent_at` (timestamp nullable) en la tabla
`samples`. La lógica de display en `calculated_status` prioriza `enviado` cuando el
protocolo está validado y `sent_at` no es null.

## PASO 2 — Actualizar el modelo `Sample`

### 2.1 — Agregar `sent_at` a `$fillable` y `$casts`

En `app/Models/Sample.php`:

Agregar `'sent_at'` al array `$fillable`.

Agregar al array `$casts`:
```php
'sent_at' => 'datetime',
```

### 2.2 — Modificar `getCalculatedStatusAttribute()`

Agregar al inicio del método, antes de cualquier otra lógica:

```php
// Protocolo enviado (ya validado y enviado por email o descargado)
if ($this->sent_at !== null && $this->validation_status === 'validated') {
    return 'enviado';
}
```

### 2.3 — Agregar 'enviado' a `getStatusLabelAttribute()`

```php
'enviado' => 'Enviado',
```

### 2.4 — Agregar 'enviado' a `getStatusColorAttribute()`

```php
'enviado' => 'sky',
```

### 2.5 — Agregar helper `isSent()`

```php
public function isSent(): bool
{
    return $this->sent_at !== null;
}
```

## PASO 3 — Marcar como `enviado` en `SampleController`

### 3.1 — En `sendEmail()`

Después de la línea `$sample->logAudit(...)` y antes del `return back()`, agregar:

```php
// Marcar como enviado si está validado
if ($sample->isValidated()) {
    $sample->update(['sent_at' => now()]);
}
```

### 3.2 — En `downloadPdf()`

Después de la línea `$sample->logAudit('pdf_generated', ...)` y antes del `return $pdf->download(...)`, agregar:

```php
// Marcar como enviado si está validado
if ($sample->isValidated()) {
    $sample->update(['sent_at' => now()]);
}
```
