## Descripción

Cuando se carga una factura de compra **sin remito asociado**, los ítems que tengan `supply_id` generan
automáticamente un movimiento de entrada al stock. Los ítems de servicios (alquiler, luz, gas) se cargan
con `supply_id = null` y solo impactan contablemente, sin tocar inventario.

Además, si al cargar la factura el insumo no existe, se puede crear al vuelo desde un modal sin salir del
formulario de factura.

## PASO 1 — Entrada de stock en `PurchaseInvoiceController::store()`

Modificar `store()` para que, después de crear la factura y sus ítems, genere movimientos de stock
cuando **no hay remito asociado**:

```php
// Después de crear ítems y llamar recalculate()...

// Generar entrada de stock solo si NO tiene remito asociado
if (!$invoice->delivery_note_id) {
    foreach ($invoice->items()->whereNotNull('supply_id')->get() as $item) {
        $supply = $item->supply;
        if (!$supply) continue;

        $previousStock = $supply->stock;
        $newStock = $previousStock + $item->quantity;

        StockMovement::create([
            'supply_id'      => $supply->id,
            'type'           => 'entrada',
            'quantity'        => $item->quantity,
            'previous_stock' => $previousStock,
            'new_stock'      => $newStock,
            'reason'         => 'compra',
            'reference_type' => PurchaseInvoice::class,
            'reference_id'   => $invoice->id,
            'lot_number'     => $item->lot_number ?? null,
            'expiration_date' => $item->expiration_date ?? null,
            'notes'          => "Factura de compra {$invoice->full_number}",
            'user_id'        => auth()->id(),
        ]);

        $supply->update([
            'stock'      => $newStock,
            'last_price' => $item->unit_price,
        ]);
    }
}
```

**Verificación:** Crear una factura de compra sin remito con 2 ítems con insumo y 1 sin insumo (servicio). Solo los 2 con insumo generan movimiento. El tercero no toca stock.

## PASO 5 — Modal "Crear insumo" inline

Agregar un modal en el formulario de factura de compra que permita crear un insumo sin salir de la página:

### 5.1 — Botón de trigger

Junto al buscador de insumos en cada fila, un botón "+" que abre el modal:

```html

        class="p-2 text-teal-600 hover:text-teal-800" title="Crear insumo nuevo">
    <i class="bi bi-plus-circle"></i>

```

### 5.2 — Modal con campos mínimos

Campos del modal:
- **Nombre** (required)
- **Categoría** (select de `SupplyCategory::active()`)
- **Unidad** (select con las mismas opciones que el form de insumos: unidad, litro, kg, caja, etc.)
- **Stock mínimo** (number, default 0)
- **Marca** (optional)
- **¿Controla lote/vencimiento?** (checkbox `tracks_lot`)

### 5.3 — Guardar por AJAX

Endpoint: `POST /supplies` (el `SupplyController::store()` existente ya funciona).

Al guardar exitosamente:
1. Cerrar el modal
2. Setear el `supply_id` de la fila que disparó el modal al nuevo insumo
3. Mostrar el nombre del insumo creado en la fila
4. Si `tracks_lot` = true, mostrar los campos de lote/vencimiento

**Importante:** El formulario de factura usa `<form>` principal. El modal debe usar `fetch()` / AJAX
para no interferir con el submit del formulario padre. El endpoint `store()` del `SupplyController`
ya devuelve redirect — necesita responder JSON cuando la request es AJAX (`$request->expectsJson()`
o header `X-Requested-With`). Modificar `SupplyController::store()` para retornar JSON si es AJAX:

```php
if ($request->ajax() || $request->wantsJson()) {
    return response()->json([
        'id'         => $supply->id,
        'code'       => $supply->code,
        'name'       => $supply->name,
        'unit'       => $supply->unit,
        'tracks_lot' => $supply->tracks_lot,
    ], 201);
}
```

**Verificación:** Desde el formulario de factura, abrir modal, crear insumo, verificar que se selecciona automáticamente en la fila y aparece en la base de datos.

## PASO 7 — Reversión de stock en `destroy()`

En `PurchaseInvoiceController::destroy()`, antes de eliminar la factura, revertir los movimientos
de stock que generó (solo si no tiene remito):

```php
if (!$invoice->delivery_note_id) {
    $movements = StockMovement::where('reference_type', PurchaseInvoice::class)
        ->where('reference_id', $invoice->id)
        ->get();

    foreach ($movements as $movement) {
        $supply = $movement->supply;
        if ($supply) {
            $supply->decrement('stock', $movement->quantity);
        }
        $movement->delete();
    }
}
```

**Verificación:** Crear factura sin remito (genera stock), eliminarla, verificar que el stock volvió al valor anterior y los movimientos se borraron.

## Notas / Consideraciones

- El patrón de entrada de stock replica lo que hace `DeliveryNoteController::accept()` pero con referencia a `PurchaseInvoice` en vez de `DeliveryNote`.
- El `update()` de factura NO recalcula stock — complejidad innecesaria. Si se necesita corregir, se elimina y se vuelve a crear.
- Los campos de lote son opcionales en facturas (a diferencia de remitos donde son obligatorios si `tracks_lot`) porque puede haber facturas cargadas después del hecho.
- El modal de crear insumo usa los mismos campos y validaciones que el formulario completo de insumos, pero con los campos mínimos necesarios para una carga rápida.
