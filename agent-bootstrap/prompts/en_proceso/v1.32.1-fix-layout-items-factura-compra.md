# v1.32.1 — Fix layout ítems factura de compra

**VERSIÓN:** v1.32.1

**DEPENDENCIAS:** v1.32.0
**ESTIMACIÓN:** 30min
**PRIORIDAD:** alta

---

## Descripción

Corregir el layout de la columna "Descripción / Insumo" en los ítems de facturas de compra.
Actualmente, el badge del código de insumo (INS-XXXXX), el campo Lote y Fecha de Vencimiento
se apilan verticalmente debajo del input de descripción, rompiendo la alineación horizontal
de la fila y haciendo que la celda crezca demasiado en altura.

El fix debe:
1. Integrar el badge del código de insumo inline con el input de descripción (como chip a la derecha o dentro del input)
2. Poner Descripción, Lote y Fecha de Vencimiento en una sola fila horizontal
3. Mantener la alineación consistente con el resto de las columnas (Cantidad, Precio, Tasa IVA, etc.)

**Importante:** El template está duplicado entre create y edit. Aplicar el fix en ambos archivos.

## PASO 1 — Reestructurar la columna Descripción/Insumo en create.blade.php

Archivo: `resources/views/purchase-invoices/create.blade.php`

El `<td>` de descripción (línea ~213) actualmente tiene esta estructura vertical:

```
<td>
  [hidden inputs]
  [input text búsqueda]          ← línea propia, w-full
  [badge INS-XXXXX]              ← línea propia, mt-1 (PROBLEMA)
  [dropdown resultados]          ← absolute, ok
  [lote + fecha vencimiento]     ← línea propia, mt-1 (PROBLEMA)
</td>
```

Cambiar a esta estructura:

```
<td>
  [hidden inputs]
  <div class="flex items-start gap-2">
    <div class="flex-1 min-w-0 relative">
      <div class="relative">
        [input text búsqueda]   ← sin cambios en funcionalidad
        [badge INS-XXXXX]       ← MOVER: posición absolute, top-1/2 right-2, transform -translate-y-1/2
                                   (chip flotante dentro del input, a la derecha)
                                   Cuando hay badge, agregar padding-right al input para que el texto no tape el badge
      </div>
      [dropdown resultados]     ← absolute, sin cambios
    </div>
    [lote]                      ← solo si item.supply_id && getSupplyTracksLot()
    [fecha vencimiento]         ← solo si item.supply_id && getSupplyTracksLot()
  </div>
</td>
```

Detalles del badge inline:
- Sacarlo del `<template x-if>` con `div mt-1` y ponerlo como `absolute` dentro del wrapper del input
- Usar `right-2 top-1/2 -translate-y-1/2` para centrarlo verticalmente a la derecha del input
- Agregar `pr-20` (o similar) al input cuando hay supply_id para que el texto no se tape con el badge
- Usar `:class="{ 'pr-20': item.supply_id }"` en el input para padding condicional

Detalles de Lote y Fecha en fila:
- Sacar el bloque de Lote + Fecha del `<div class="flex gap-2 mt-1">` actual
- Ponerlos como hijos directos del flex container principal, cada uno con `w-32` o similar
- Mantener `text-xs` y estilos actuales
- La condición `x-if="item.supply_id && getSupplyTracksLot(item.supply_id)"` sigue aplicando

**Verificación:** Al seleccionar un insumo con tracking de lote, los campos Descripción + badge + Lote + Fecha deben estar en una sola fila horizontal. Sin insumo, solo se ve el input de descripción ocupando todo el ancho.

## Notas / Consideraciones

- El template de ítems está DUPLICADO entre create y edit — cualquier cambio va en ambos
- El badge usa `<template x-if>` que en Alpine.js puede tener problemas dentro de tablas (foster-parenting) — al moverlo a absolute dentro de un div wrapper, este problema se evita
- Los anchos de Lote (`w-32`) y Fecha (`w-36`) son sugeridos — ajustar según se vea bien en contexto con las demás columnas
- El input de búsqueda tiene `placeholder="Buscar insumo o escribir descripción..."` que es largo — considerar acortarlo si el input se reduce
- Referencia visual: las imágenes del estado actual están en `assets/` del workspace
