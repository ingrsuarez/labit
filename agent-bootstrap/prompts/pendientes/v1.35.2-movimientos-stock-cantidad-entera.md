# v1.35.2 — Ajuste de stock: cantidad siempre entera

**VERSIÓN:** v1.35.2  
**SLUG:** movimientos-stock-cantidad-entera  
**DEPENDENCIAS:** — (módulo movimientos de stock existente)  
**ESTIMACIÓN:** 30–45 min  
**PRIORIDAD:** media

---

## Problema

En `stock-movements/create` el campo **Cantidad** usa `step="0.01"` y validación `numeric|min:0.01`,
permitiendo decimales (ej. 1.01). En la operatoria del laboratorio las **unidades de insumo son siempre
enteras** (unidades, cajas, etc.).

---

## Alcance

1. **Vista** `resources/views/stock-movements/create.blade.php`
   - Input cantidad: `type="number"` con `step="1"`, `min` acorde al tipo (ver abajo).
   - Placeholder entero (ej. `0` o `1`).
   - Opcional coherente UX: en el `<option>` del select y en “Stock actual”, mostrar cantidades **sin
     decimales artificiales** si el negocio es entero:
     - `number_format($sup->stock, 0, ',', '.')` cuando el stock sea conceptualmente entero, **o**
     - mantener 2 decimales solo si existen stocks históricos con fracción (poco probable); criterio:
       **mostrar como entero** redondeando o truncando en display — documentar en commit la opción elegida
       (recomendación: `round` o cast a int solo en **texto**, no alterar BD en esta versión).

2. **Validación** `StockMovementController::store`
   - `quantity` → `integer`.
   - **Entrada** y **salida:** `min:1`.
   - **Ajuste** (establece stock absoluto): `min:0` (permitir dejar depósito en cero).
   - Mensajes de error claros: “La cantidad debe ser un número entero”.

   Ejemplo de regla condicional (pseudo):

   ```php
   'quantity' => [
       'required',
       'integer',
       $request->input('type') === 'ajuste' ? 'min:0' : 'min:1',
   ],
   ```

3. **Alpine `stockForm()`** (si se agrega `min` dinámico según `type`): sincronizar con las mismas reglas
   (`entrada`/`salida` → min 1; `ajuste` → min 0).

4. **Tests** (si el proyecto tiene tests de HTTP para stock): request con `quantity` = 1.5 → 422;
   `quantity` = 2 → OK.

5. **Fuera de alcance:** cambiar el tipo de columna `supplies.stock` o `stock_movements.quantity` en BD
   (pueden seguir siendo `decimal`); solo se fuerza **entrada de usuario entera**.
