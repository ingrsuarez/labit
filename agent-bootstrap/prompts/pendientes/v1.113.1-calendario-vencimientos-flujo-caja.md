todos los pagos a realizar para ordenar el flujo de caja.

(sueldos, gastos fijos) y obligaciones manuales (e-cheqs emitidos, planes ARCA, ganancias,
cuotas de equipos).

## PASO 2 — Servicio `CashFlowCalendarService`

Crear `app/Services/CashFlowCalendarService.php`.

Método principal:

```php
public function eventsForRange(int $companyId, Carbon $from, Carbon $to): Collection
```

Cada evento es un DTO/array normalizado:

```php
[
    'id' => string,           // único estable ej. "fc:123", "iva:2026-06", "manual:45"
    'date' => 'Y-m-d',
    'category' => string,     // slug para color/filtro
    'category_label' => string,
    'title' => string,
    'amount' => float,
    'confidence' => 'confirmed|estimated|manual',
    'source_type' => ?string,  // morph o null
    'source_id' => ?int,
    'url' => ?string,          // link al origen
    'meta' => array,
]
```

### Proveedores de eventos (métodos privados)

**A) FC compra** — `PurchaseInvoice` con `due_date` not null, saldo > 0, status no pagada/anulada, fecha en rango.

**B) IVA mensual** — para cada mes en rango:
- Fecha = `{iva_due_day}` del mes (ajustar si día > último día del mes → último día)
- Monto: `TaxReturn` confirmada impuesto IVA mismo período → `max(balance, 0)`; si no hay → estimación IVA FV − IVA FC del período (sumar `iva_21+iva_10_5+iva_27` neto NC incluidas, mismo criterio `FinancialDashboardService` / facturas con CAE)
- `confidence`: confirmed si DDJJ, else estimated

**C) Form 931** — para cada mes en rango:
- Fecha = `{form931_due_day}`
- Monto = `Form931DeclarationService::latestConfirmedTotal($companyId)` (mismo monto proyectado en meses futuros hasta nueva DDJJ)
- `confidence`: estimated (proyección desde última confirmada)

**D) Sueldos** — cada mes en rango:
- Fecha = 5.º día hábil (`BusinessDayCalculator` usando feriados)
- Monto = suma `neto_a_cobrar` de última liquidación **cerrada** de la empresa (documentar si es la del mes anterior o la más reciente — usar **última cerrada**)
- `confidence`: estimated

**E) Gastos fijos** — suppliers `is_fixed_expense = true`:
- Última FC del proveedor (por `issue_date` desc)
- Proyectar en mismos días del mes que `due_date` de esa FC (o `issue_date` si due_date null — pero FC sin due_date no entra en proyección automática según PM; **usar day-of-month de due_date de última FC; si no tiene due_date, day-of-month de issue_date**)
- Monto = `total` última FC
- Proyectar meses dentro del rango visible
- `confidence`: estimated

**F) Manuales** — `CashFlowObligation` con `due_date` en rango.
