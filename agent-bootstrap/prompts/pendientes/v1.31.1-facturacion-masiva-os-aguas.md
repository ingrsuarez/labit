# v1.31.1 — Facturación masiva por obra social, aguas y veterinarias

**VERSIÓN:** v1.31.1

**DEPENDENCIAS:** v1.31.0  
**ESTIMACIÓN:** 3h  
**PRIORIDAD:** alta

---

## Descripción

Agrega facturación masiva (batch) a la vista "Protocolos sin facturar" creada en v1.31.0.
Permite seleccionar múltiples protocolos y generar una sola factura que agrupa todos los
protocolos como ítems. Cubre 3 flujos de facturación mensual:

1. **Obras sociales (lab clínico):** Una factura por OS por mes, con todas las determinaciones
   de todos los protocolos del período como ítems. Tipo de factura (A/B/C) determinado por
   la condición IVA de la obra social (customer asociado).
2. **Muestras de aguas/alimentos:** Factura al cliente cuando los protocolos están validados
   y entregados. Tipo determinado por condición IVA del cliente.
3. **Veterinarias con cuenta corriente:** Factura mensual a la veterinaria referente con todos
   los protocolos del mes. Similar al flujo de obras sociales.

## PASO 1 — Método `batchInvoice` en `BillingController`

Agregar método que recibe la selección y crea la factura masiva:

```php
public function batchInvoice(Request $request)
{
    $this->authorize('sales-invoices.create');

    $request->validate([
        'protocol_type' => 'required|in:admission,sample,vet_admission',
        'protocol_ids' => 'required|array|min:1',
        'protocol_ids.*' => 'integer',
        'customer_id' => 'required|exists:customers,id',
        'point_of_sale_id' => 'required|exists:points_of_sale,id',
        'voucher_type' => 'required|in:A,B,C',
        'issue_date' => 'required|date',
        'notes' => 'nullable|string',
    ]);

    $protocolClass = match($request->protocol_type) {
        'admission' => \App\Models\Admission::class,
        'sample' => \App\Models\Sample::class,
        'vet_admission' => \App\Models\VetAdmission::class,
    };

    $protocols = $protocolClass::whereIn('id', $request->protocol_ids)
        ->uninvoiced()
        ->get();

    if ($protocols->isEmpty()) {
        return back()->with('error', 'No se encontraron protocolos válidos para facturar.');
    }

    // Construir ítems según tipo de protocolo
    $items = [];
    foreach ($protocols as $protocol) {
        if ($request->protocol_type === 'admission') {
            $protocol->load('admissionTests.test');
            foreach ($protocol->admissionTests as $at) {
                $items[] = [
                    'description' => $protocol->protocol_number . ' — ' . ($at->test?->name ?? 'Determinación'),
                    'test_id' => $at->test_id,
                    'quantity' => 1,
                    'unit_price' => $at->price,
                    'iva_rate' => 21,
                ];
            }
        } elseif ($request->protocol_type === 'sample') {
            $protocol->load('determinations');
            foreach ($protocol->determinations as $det) {
                $items[] = [
                    'description' => $protocol->protocol_number . ' — ' . ($det->test?->name ?? 'Determinación'),
                    'test_id' => $det->test_id ?? null,
                    'quantity' => 1,
                    'unit_price' => $det->price ?? 0,
                    'iva_rate' => 21,
                ];
            }
        } elseif ($request->protocol_type === 'vet_admission') {
            $protocol->load('tests.test');
            foreach ($protocol->tests as $vt) {
                $items[] = [
                    'description' => $protocol->protocol_number . ' — ' . ($vt->test?->name ?? 'Determinación'),
                    'test_id' => $vt->test_id,
                    'quantity' => 1,
                    'unit_price' => $vt->price,
                    'iva_rate' => 21,
                ];
            }
        }
    }

    if (empty($items)) {
        return back()->with('error', 'Los protocolos seleccionados no tienen determinaciones.');
    }

    // Crear la factura
    $pointOfSale = \App\Models\PointOfSale::findOrFail($request->point_of_sale_id);
    $isElectronic = $pointOfSale->is_electronic;

    $invoice = \App\Models\SalesInvoice::create([
        'company_id' => $pointOfSale->company_id,
        'invoice_number' => $isElectronic ? 'PENDIENTE-AFIP' : null,
        'voucher_type' => $request->voucher_type,
        'point_of_sale_id' => $request->point_of_sale_id,
        'customer_id' => $request->customer_id,
        'issue_date' => $request->issue_date,
        'notes' => $request->notes,
        'status' => 'pendiente',
        'amount_collected' => 0,
        'is_electronic' => $isElectronic,
        'created_by' => auth()->id(),
    ]);

    $sortOrder = 0;
    foreach ($items as $itemData) {
        $ivaAmount = round($itemData['quantity'] * $itemData['unit_price'] * $itemData['iva_rate'] / 100, 2);
        $total = $itemData['quantity'] * $itemData['unit_price'] + $ivaAmount;

        $invoice->items()->create([
            'description' => $itemData['description'],
            'test_id' => $itemData['test_id'] ?? null,
            'quantity' => $itemData['quantity'],
            'unit_price' => $itemData['unit_price'],
            'iva_rate' => $itemData['iva_rate'],
            'iva_amount' => $ivaAmount,
            'total' => $total,
            'sort_order' => $sortOrder++,
        ]);
    }

    $invoice->recalculate();

    // Registrar vínculo en invoice_protocols
    foreach ($protocols as $protocol) {
        \App\Models\InvoiceProtocol::create([
            'sales_invoice_id' => $invoice->id,
            'protocol_type' => $protocolClass,
            'protocol_id' => $protocol->id,
            'amount' => 0, // se puede calcular por protocolo si se necesita
        ]);
    }

    // Si es electrónica, autorizar con AFIP
    if ($isElectronic) {
        $invoice->load(['pointOfSale', 'customer', 'items']);
        try {
            $afip = new \App\Services\AfipService();
            $result = $afip->createVoucher($invoice);
            // Actualizar con datos de AFIP (CAE, número)
            $invoice->update([
                'cae' => $result['cae'] ?? null,
                'cae_expiration' => $result['cae_expiration'] ?? null,
                'afip_voucher_number' => $result['voucher_number'] ?? null,
                'invoice_number' => $result['full_number'] ?? $invoice->invoice_number,
                'afip_result' => $result['result'] ?? null,
                'afip_response' => $result['response'] ?? null,
            ]);
        } catch (\Exception $e) {
            // La factura se creó pero falló AFIP — se puede reintentar
            return redirect()->route('sales-invoices.show', $invoice)
                ->with('warning', 'Factura creada pero AFIP devolvió error: ' . $e->getMessage());
        }
    }

    return redirect()->route('sales-invoices.show', $invoice)
        ->with('success', "Factura creada con {$protocols->count()} protocolos.");
}
```

**Verificación:** POST a `/billing/batch-invoice` crea una factura con N protocolos vinculados.

## PASO 2 — Método de previsualización

Agregar método que muestra un resumen antes de confirmar la facturación masiva:

```php
public function batchPreview(Request $request)
{
    $this->authorize('sales-invoices.create');

    $request->validate([
        'protocol_type' => 'required|in:admission,sample,vet_admission',
        'protocol_ids' => 'required|array|min:1',
    ]);

    $protocolClass = match($request->protocol_type) {
        'admission' => \App\Models\Admission::class,
        'sample' => \App\Models\Sample::class,
        'vet_admission' => \App\Models\VetAdmission::class,
    };

    $protocols = $protocolClass::whereIn('id', $request->protocol_ids)
        ->uninvoiced()
        ->get();

    // Calcular totales
    $total = 0;
    $itemCount = 0;
    foreach ($protocols as $p) {
        if ($request->protocol_type === 'admission') {
            $p->load('admissionTests');
            $total += $p->admissionTests->sum('price');
            $itemCount += $p->admissionTests->count();
        } elseif ($request->protocol_type === 'sample') {
            $p->load('determinations');
            $total += $p->determinations->sum('price');
            $itemCount += $p->determinations->count();
        } else {
            $p->load('tests');
            $total += $p->tests->sum('price');
            $itemCount += $p->tests->count();
        }
    }

    // Determinar cliente y tipo de factura
    // Para admissions: el cliente es la obra social (insurance → customer asociado)
    // Para samples/vet: el customer_id del protocolo
    $customerId = null;
    $customerName = '';
    $voucherType = 'B';

    if ($request->protocol_type === 'admission' && $protocols->isNotEmpty()) {
        $insurance = $protocols->first()->insuranceRelation;
        // La obra social es el customer al que se factura
        // Buscar customer por nombre de OS o usar el campo customer de la OS
        $customerName = $insurance?->name ?? 'Sin obra social';
        // El voucherType depende de la condición IVA de la OS
    } else {
        $customer = $protocols->first()?->customer;
        $customerId = $customer?->id;
        $customerName = $customer?->name ?? '';
        if ($customer && in_array($customer->iva_condition, ['IVA Responsable Inscripto', 'Responsable Inscripto'])) {
            $voucherType = 'A';
        }
    }

    $pointsOfSale = \App\Models\PointOfSale::where('is_active', true)
        ->where('company_id', active_company_id())
        ->orderBy('code')->get();

    $customers = \App\Models\Customer::where('status', 'activo')->orderBy('name')->get();

    return view('billing.batch-preview', compact(
        'protocols', 'total', 'itemCount', 'customerId', 'customerName',
        'voucherType', 'pointsOfSale', 'customers'
    ))->with([
        'protocolType' => $request->protocol_type,
        'protocolIds' => $request->protocol_ids,
    ]);
}
```

**Verificación:** Preview muestra resumen con cantidad de protocolos, ítems, total estimado.

### 4.2 — JavaScript Alpine

```js
function batchBilling() {
    return {
        selectedIds: [],
        protocolType: '{{ $module }}',

        toggleAll(event) {
            if (event.target.checked) {
                this.selectedIds = [...document.querySelectorAll('[data-protocol-id]')]
                    .map(el => el.dataset.protocolId);
            } else {
                this.selectedIds = [];
            }
        },

        submitBatch() {
            const form = document.createElement('form');
            form.method = 'POST';
            form.action = '{{ route("billing.batch-preview") }}';

            const csrf = document.createElement('input');
            csrf.type = 'hidden';
            csrf.name = '_token';
            csrf.value = '{{ csrf_token() }}';
            form.appendChild(csrf);

            const typeInput = document.createElement('input');
            typeInput.type = 'hidden';
            typeInput.name = 'protocol_type';
            typeInput.value = this.protocolType;
            form.appendChild(typeInput);

            this.selectedIds.forEach(id => {
                const input = document.createElement('input');
                input.type = 'hidden';
                input.name = 'protocol_ids[]';
                input.value = id;
                form.appendChild(input);
            });

            document.body.appendChild(form);
            form.submit();
        }
    }
}
```

## Notas / Consideraciones

- La facturación masiva NO es automática — siempre requiere intervención del usuario
  (seleccionar protocolos → preview → confirmar).
- Los ítems de la factura incluyen el número de protocolo en la descripción (ej:
  "C2603290001 — Hemograma") para trazabilidad en la factura impresa y en AFIP.
- Si la factura es electrónica y AFIP falla, la factura queda creada localmente
  con estado "pendiente" y sin CAE. Se puede reintentar manualmente.
- Para obras sociales: la obra social (`Insurance`) puede no tener un `Customer` asociado
  directamente. En ese caso, el usuario deberá seleccionar/crear el cliente manualmente
  en el preview. A futuro se puede vincular OS con Customer automáticamente.
- La empresa emisora no se fuerza en facturación masiva (a diferencia de particulares en v1.31.0).
  Se usa la empresa activa del usuario. Esto permite facturar desde cualquiera de las dos empresas.
- La alícuota de IVA se fija en 21% por defecto. Si en el futuro hay servicios exentos
  o con otra alícuota, se deberá parametrizar por determinación.
