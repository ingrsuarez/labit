# v1.32.0 — Buscador inteligente de insumos en facturas de compra

**VERSIÓN:** v1.32.0  
**SLUG:** buscador-insumos-facturas-compra  
**DEPENDENCIAS:** ninguna (usa infraestructura existente de v1.27.0 y v1.28.0)

**PRIORIDAD:** media

## Descripción

Reemplaza la UI actual de carga de ítems en facturas de compra (`purchase-invoices/create`)
por un flujo más intuitivo con buscador tipo combobox. El patrón a seguir es el que ya existe
en admisiones de laboratorio (`lab/admissions/create`) para buscar determinaciones.

**Cambios principales:**

1. Unificar las columnas "Descripción" e "Insumo" en una sola columna con input buscador
   que autocompleta contra la tabla `supplies`.
2. Al seleccionar un insumo, se rellena automáticamente la descripción y el `supply_id`.
3. Navegación con Tab fluida: al seleccionar insumo → foco pasa a Cantidad → Precio Unit. → siguiente.
4. Si el insumo no existe, mostrar opción "Crear insumo nuevo" en el dropdown que abre el
   modal existente. Al cerrar el modal, volver al formulario conservando los ítems ya cargados.
5. Permitir también texto libre (sin vincular a un insumo) para ítems genéricos.

## PASO 1 — Endpoint de búsqueda de insumos

Agregar un endpoint en `SupplyController` (o crear una ruta dedicada) para buscar insumos
por texto. Este endpoint será consumido por el buscador vía fetch.

```php
// En routes/web.php (dentro del grupo autenticado)
Route::get('supplies/search', [SupplyController::class, 'search'])->name('supplies.search');
```

```php
// En SupplyController.php
public function search(Request $request)
{
    $q = $request->get('q', '');
    if (strlen($q) < 2) {
        return response()->json([]);
    }

    $supplies = Supply::active()
        ->where(function ($query) use ($q) {
            $query->where('name', 'like', "%{$q}%")
                  ->orWhere('code', 'like', "%{$q}%");
        })
        ->orderBy('name')
        ->limit(15)
        ->get(['id', 'code', 'name', 'tracks_lot']);

    return response()->json($supplies);
}
```

**Verificación:** `GET /supplies/search?q=guan` devuelve JSON con insumos que matchean.

### 2.3 — Componente Alpine `supplySearch()`

Agregar esta función fuera de `invoiceForm()`, al nivel del `<script>`:

```javascript
function supplySearch(item, index) {
    return {
        searchText: item.description || '',
        results: [],
        showResults: false,
        loading: false,
        highlightedIndex: -1,

        async doSearch() {
            item.description = this.searchText;
            if (this.searchText.length < 2) {
                this.results = [];
                this.showResults = false;
                return;
            }
            this.loading = true;
            try {
                const resp = await fetch(`{{ route('supplies.search') }}?q=${encodeURIComponent(this.searchText)}`);
                this.results = await resp.json();
                this.showResults = true;
                this.highlightedIndex = -1;
            } catch (e) {
                this.results = [];
            } finally {
                this.loading = false;
            }
        },

        selectSupply(supply) {
            item.supply_id = supply.id;
            item.description = supply.name;
            item._supply_code = supply.code;
            this.searchText = supply.name;
            this.showResults = false;
            // Foco a Cantidad
            this.$nextTick(() => {
                const qtyInput = document.querySelector(`#item-qty-${index}`);
                if (qtyInput) qtyInput.focus();
            });
        },

        unlinkSupply() {
            item.supply_id = '';
            item._supply_code = '';
        },

        highlightNext() {
            const max = this.results.length; // +1 por "Crear nuevo" no, simplificamos
            if (this.highlightedIndex < max - 1) this.highlightedIndex++;
        },

        highlightPrev() {
            if (this.highlightedIndex > 0) this.highlightedIndex--;
        },

        selectHighlighted() {
            if (this.highlightedIndex >= 0 && this.highlightedIndex < this.results.length) {
                this.selectSupply(this.results[this.highlightedIndex]);
            }
        },

        onTab(event) {
            if (this.showResults && this.highlightedIndex >= 0) {
                event.preventDefault();
                this.selectHighlighted();
            } else {
                this.showResults = false;
                item.description = this.searchText;
            }
        },

        openCreateModal() {
            this.showResults = false;
            // Prellenar nombre del nuevo insumo con el texto buscado
            const form = Alpine.$data(this.$root.closest('[x-data]'));
            // Acceder al invoiceForm padre
            const parent = document.querySelector('[x-data="invoiceForm()"]').__x.$data;
            parent.newSupply.name = this.searchText;
            parent.newSupplyForRow = index;
            parent.showNewSupplyModal = true;
        },

        init() {
            // Si el item ya tiene supply, mostrar el nombre
            if (item.supply_id && !item.description) {
                const s = Alpine.$data(this.$root.closest('[x-data="invoiceForm()"]'))
                    ?.supplies?.find(s => s.id == item.supply_id);
                if (s) {
                    this.searchText = s.name.replace(/^INS-\d+ - /, '');
                    item._supply_code = s.name.split(' - ')[0];
                }
            }
        }
    }
}
```

**Nota:** La función `openCreateModal()` accede al scope de `invoiceForm()` para abrir
el modal existente. Puede necesitarse ajustar el acceso al Alpine data según la estructura
del DOM. La alternativa más robusta es usar `$dispatch` con un evento custom.

**Verificación:** Escribir en el input busca insumos, seleccionar rellena supply_id y
descripción, Tab pasa a Cantidad.

## Notas / Consideraciones

- El patrón de búsqueda es similar al de `lab/admissions/create.blade.php` pero usa
  un endpoint dedicado (`supplies/search`) en vez de filtrar un array precargado, porque
  la cantidad de insumos puede crecer significativamente.
- Se mantiene el array `supplies` en `invoiceForm()` exclusivamente para la función
  `getSupplyTracksLot()` y para ítems que vienen precargados de delivery notes.
- La columna unificada "Descripción / Insumo" permite tanto texto libre como insumos
  vinculados. Esto es importante porque las facturas de compra pueden tener ítems que
  no corresponden a insumos del sistema (ej: "Gastos de envío", "Bonificación").
- La navegación con Tab sigue el orden natural: Descripción → Cantidad → Precio Unit.
  → Tasa IVA → (siguiente fila). No se fuerza ningún Tab trap.
