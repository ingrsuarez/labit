## Descripción

El cliente necesita dos cosas:

### 1. Jerarquía de 3 niveles (padre → sub-padre → hijo)

Actualmente solo hay 2 niveles (padre → hijo). Se necesita soportar:

```
HEMOGRAMA (padre)
├── Glóbulos Rojos (hijo)
├── Hemoglobina (hijo)
└── FORMULA LEUCOCITARIA (sub-padre)
    ├── Linfocitos (hijo)
    └── Neutrófilos (hijo)
```

Un sub-padre es un test que es **hijo** de un padre y a su vez **padre** de otros tests.
No se requieren más de 3 niveles.

### 2. Orden fijo configurable

Cada determinación tiene un orden fijo que determina su posición en los informes:
- **Orden global** (`tests.sort_order`): define la posición de padres y standalone entre sí
- **Orden interno** (`test_parents.order`): define la posición de hijos dentro de su padre
  (ya existe en la tabla pivot pero NO se usa en los PDFs actualmente)

Ejemplo:

```
sort_order 10 → HEMOGRAMA (padre)
                  order 1 → Glóbulos Rojos
                  order 2 → Hemoglobina
                  order 3 → FÓRMULA LEUCOCITARIA (sub-padre)
                              order 1 → Linfocitos
                              order 2 → Neutrófilos
sort_order 20 → GLUCEMIA (standalone)
sort_order 30 → HEPATOGRAMA (padre)
                  order 1 → GOT
                  order 2 → GPT
```

public function getAllChildren(bool $withRelations = true): \Illuminate\Support\Collection
{
    $query = $withRelations
        ? $this->childTests()->with(['children', 'childTests', 'referenceValues'])
        : $this->childTests();

    $pivotChildren = $query->orderBy('test_parents.order')->get();

}

## PASO 5 — Reescribir lógica de ordenamiento en PDFs

Este es el cambio más importante. Ambos PDFs (muestras y admisiones) tienen un
bloque PHP que agrupa y ordena las determinaciones. Hay que reescribirlos para:

1. Respetar `tests.sort_order` para el orden global
2. Respetar `test_parents.order` para el orden dentro de un padre
3. Soportar 3 niveles: padre → sub-padre → hijo

### 5.1 — Lógica común (extraer como concepto)

La lógica de agrupación es idéntica en ambos PDFs. Describir el algoritmo
una sola vez:

```php
@php
    // 1. Filtrar determinaciones validadas
    $validatedItems = $collection->where('is_validated', true);

    // 2. Construir mapa de test_id → item para acceso rápido
    $itemsByTestId = [];
    foreach ($validatedItems as $item) {
        $itemsByTestId[$item->test_id] = $item;
    }

    // 3. Identificar padres, sub-padres e hijos
    // Un test es "raíz" si no tiene padres en esta colección
    // Un test es "sub-padre" si tiene padres Y tiene hijos en esta colección
    $testIds = $validatedItems->pluck('test_id')->toArray();
    $parentMap = []; // test_id => [child_test_ids ordenados]
    $childOf = [];   // test_id => parent_test_id
    $isSubParent = []; // test_id => true

    foreach ($validatedItems as $item) {
        $test = $item->test;
        if (!$test) continue;

        $children = $test->childTests()
            ->whereIn('tests.id', $testIds)
            ->orderBy('test_parents.order')
            ->pluck('tests.id')
            ->toArray();

        if (!empty($children)) {
            $parentMap[$test->id] = $children;
            foreach ($children as $childId) {
                $childOf[$childId] = $test->id;
            }
        }
    }

    // Detectar sub-padres: tests que son hijos Y padres
    foreach ($parentMap as $testId => $children) {
        if (isset($childOf[$testId])) {
            $isSubParent[$testId] = true;
        }
    }

    // 4. Identificar raíces (padres de nivel 1 y standalone)
    // Raíz = tiene hijos en la colección Y no es hijo de nadie en la colección
    // Standalone = no tiene hijos ni es hijo de nadie
    $roots = [];
    foreach ($validatedItems as $item) {
        $tid = $item->test_id;
        if (!isset($childOf[$tid])) {
            $roots[] = $tid;
        }
    }

    // 5. Ordenar raíces por sort_order del test
    usort($roots, function ($a, $b) use ($itemsByTestId) {
        $sortA = $itemsByTestId[$a]->test->sort_order ?? 0;
        $sortB = $itemsByTestId[$b]->test->sort_order ?? 0;
        return $sortA <=> $sortB;
    });

    // 6. Construir lista ordenada con niveles de indentación
    $orderedItems = collect();

    $addWithChildren = function ($testId, $level) use (
        &$addWithChildren, $parentMap, $isSubParent, $itemsByTestId, &$orderedItems
    ) {
        if (!isset($itemsByTestId[$testId])) return;

        $isParent = isset($parentMap[$testId]);
        $isSub = isset($isSubParent[$testId]);

        $orderedItems->push([
            'item' => $itemsByTestId[$testId],
            'level' => $level,
            'isParent' => $isParent,
            'isSubParent' => $isSub,
            'isChild' => $level > 0,
        ]);

        if ($isParent) {
            foreach ($parentMap[$testId] as $childId) {
                $addWithChildren($childId, $level + 1);
            }
        }
    };

    foreach ($roots as $rootId) {
        $addWithChildren($rootId, 0);
    }
@endphp
```

## PASO 6 — Actualizar auto-agregado de hijos

### 6.1 — SampleController::store() y addDetermination()

El `getAllChildren()` ya debería traer nietos (hijos de sub-padres) si el
sub-padre también fue agregado. Verificar que al agregar HEMOGRAMA:
1. Se agrega HEMOGRAMA
2. Se agrega FORMULA LEUCOCITARIA (hijo directo)
3. Se agregan Linfocitos, Neutrófilos (hijos de FORMULA LEUCOCITARIA)

Si `getAllChildren()` solo trae 1 nivel de profundidad, agregar recursión:

```php
public function getAllChildren(bool $withRelations = true): \Illuminate\Support\Collection
{
    $query = $withRelations
        ? $this->childTests()->with(['children', 'childTests', 'referenceValues'])
        : $this->childTests();

    $directChildren = $query->orderBy('test_parents.order')->get();

    $allDescendants = collect();
    foreach ($directChildren as $child) {
        $allDescendants->push($child);
        // Recursión: agregar hijos del hijo (nietos)
        $grandchildren = $child->getAllChildren($withRelations);
        $allDescendants = $allDescendants->merge($grandchildren);
    }

    // Legacy children que no estén ya incluidos
    $legacyChildren = $withRelations
        ? $this->children()->with(['children', 'childTests', 'referenceValues'])->get()
        : $this->children()->get();

    $allDescendants = $allDescendants->merge(
        $legacyChildren->filter(fn ($c) => !$allDescendants->contains('id', $c->id))
    );

    return $allDescendants->unique('id');
}
```
