# v1.104.0 — Email en protocolos de muestras: UI + permisos + paridad clínico/vet

**VERSIÓN:** v1.104.0  
**SLUG:** email-protocolos-muestras-ui  
**DEPENDENCIAS:** v1.73.0 (`sent_at`), v1.74.0 (backend masivo — ya en código)  
**ESTIMACIÓN:** 2–3 h  
**PRIORIDAD:** alta

---

## Descripción

El backend de envío por email en muestras (aguas/alimentos/hielo) ya existe (`sendEmail`, `batchEmail`, `SampleResultMail`, `SampleBatchMail`), pero la **UI está incompleta**: falta el botón/modal en el detalle del protocolo y el listado masivo no respeta permisos ni es tan visible como en lab clínico.

Esta versión completa la experiencia según el diseño en `docs/designs/DISEÑO_v1.104.0-email-protocolos-muestras-ui.md`: envío individual desde `sample/show`, pulido del listado `sample/index` (banner de selección, `@can`, tooltips), y alinear la regla de envío individual con clínico/vet (≥1 determinación validada).

## PASO 1 — Alinear regla de envío individual en backend

En `SampleController::sendEmail`, reemplazar la condición `! $sample->isValidated()` por la misma regla que lab/vet:

```php
$validatedCount = $sample->determinations()->where('is_validated', true)->count();
if ($validatedCount === 0) {
    return back()->with('error', 'Debe validar al menos una determinación para enviar el informe.');
}
```

Mantener `authorize('samples-reports.send')`, `sent_at` al enviar, y `logAudit`.

**Verificación:** Protocolo con validación parcial (no 100%) puede enviarse si tiene ≥1 determinación validada.

## PASO 2 — Botón y modal de email en `sample/show.blade.php`

Seguir `DISEÑO_v1.104.0-email-protocolos-muestras-ui.md` sección 1.

1. Calcular `$validatedCount` en el header (o reutilizar lógica existente de PDF).
2. Dentro de `@can('samples-reports.send')` y `@if($validatedCount > 0)`:
   - Botón **Enviar por Email** (`bg-teal-600`) después de Descargar PDF.
   - Alpine `showEmailModal` en el contenedor del header (`x-data="{ showEmailModal: false }"`).
3. Modal con form POST a `route('sample.sendEmail', $sample)`:
   - Campo email (default `customer.email`)
   - Chip atajo si hay email de cliente
   - Textarea mensaje opcional
   - Nota de reenvío si `$sample->isSent()`
4. Focus ring: `focus:ring-teal-500 focus:border-teal-500`

**Verificación:** En `/sample/{id}` con determinaciones validadas aparece el botón; sin permiso `samples-reports.send` no aparece.

## PASO 4 — Tests Feature

Ampliar `tests/Feature/SampleBatchEmailTest.php` y crear o extender tests para envío individual, por ejemplo `SampleSendEmailTest.php`:

- Usuario con `samples-reports.send` + protocolo con 1 det. validada → POST `sample.sendEmail` → 302 + `sent_at` seteado.
- Usuario sin permiso → 403.
- Protocolo sin ninguna validada → error flash.
- Invitado → 401/redirect login.

Usar factories/seeders existentes del proyecto para `Sample`, `Customer`, roles.

```bash
php artisan test --filter=SampleBatchEmail
php artisan test --filter=SampleSendEmail
```

**Verificación:** Tests en verde.
