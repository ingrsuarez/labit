## Descripción

Genera los archivos TXT en formato posicional de longitud fija según el diseño de registros de AFIP
(RG 4597/5133) para importar en Portal IVA. Cubre los 4 archivos principales:
Ventas Cabecera, Ventas Alícuotas, Compras Cabecera y Compras Alícuotas.

El contador selecciona un período mensual, previsualiza los totales, y descarga un ZIP con los 4 archivos
listos para subir a Portal IVA de AFIP.

**Referencia oficial de formatos:** https://www.afip.gob.ar/iva/documentos/libro-iva-digital-diseno-registros.pdf

### 1.2 — Helpers de formato posicional

```php
// Formatear importe: 13 enteros + 2 decimales sin punto
private function formatAmount(float $amount, int $length = 15): string
{
    $cents = (int) round(abs($amount) * 100);
    return str_pad((string) $cents, $length, '0', STR_PAD_LEFT);
}

// Formatear texto alfanumérico con relleno de espacios
private function formatText(string $text, int $length): string
{
    return str_pad(mb_substr($text, 0, $length), $length, ' ', STR_PAD_RIGHT);
}

// Formatear numérico con ceros a izquierda
private function formatNumeric(string $value, int $length): string
{
    return str_pad($value, $length, '0', STR_PAD_LEFT);
}

// Formatear fecha como AAAAMMDD
private function formatDate(?Carbon $date): string
{
    return $date ? $date->format('Ymd') : '00000000';
}
```

### 1.3 — Generación de Ventas Cabecera (266 chars por registro)

```php
public function generateVentasCabecera(int $companyId, int $year, int $month): string
{
    $lines = [];

    // Facturas de venta del período
    $invoices = SalesInvoice::where('company_id', $companyId)
        ->whereYear('issue_date', $year)
        ->whereMonth('issue_date', $month)
        ->whereNotNull('cae') // solo las electrónicas autorizadas
        ->with('customer', 'pointOfSale')
        ->get();

    foreach ($invoices as $inv) {
        $tipoDoc = $this->getDocType($inv->customer);
        $nroDoc = $this->getDocNumber($inv->customer);
        $tipoCbte = $this->getVoucherTypeCode($inv->voucher_type, 'ventas');
        $cantAlicuotas = $this->countAlicuotas($inv);

        $line = '';
        $line .= $this->formatDate($inv->issue_date);                    // 1: Fecha (8)
        $line .= $this->formatNumeric($tipoCbte, 3);                     // 2: Tipo cbte (3)
        $line .= $this->formatNumeric($inv->pointOfSale->afip_pos_number ?? $inv->point_of_sale, 5); // 3: PV (5)
        $line .= $this->formatNumeric($inv->afip_voucher_number ?? $inv->invoice_number, 20); // 4: Nro cbte (20)
        $line .= $this->formatNumeric($inv->afip_voucher_number ?? $inv->invoice_number, 20); // 5: Nro cbte hasta (20)
        $line .= $tipoDoc;                                                // 6: Cod doc comprador (2)
        $line .= $this->formatNumeric($nroDoc, 20);                      // 7: Nro id comprador (20)
        $line .= $this->formatText($inv->customer->name ?? 'CONSUMIDOR FINAL', 30); // 8: Nombre (30)
        $line .= $this->formatAmount($inv->total);                       // 9: Importe total (15)
        $line .= $this->formatAmount(0);                                  // 10: Conceptos no gravados (15)
        $line .= $this->formatAmount(0);                                  // 11: Percepción no categ (15)
        $line .= $this->formatAmount(0);                                  // 12: Operaciones exentas (15)
        $line .= $this->formatAmount($inv->percepciones ?? 0);           // 13: Percepciones nacionales (15)
        $line .= $this->formatAmount(0);                                  // 14: Percepciones IIBB (15)
        $line .= $this->formatAmount(0);                                  // 15: Percepciones municipales (15)
        $line .= $this->formatAmount(0);                                  // 16: Impuestos internos (15)
        $line .= self::MONEDA_PESOS;                                      // 17: Moneda (3)
        $line .= self::TIPO_CAMBIO_DEFAULT;                               // 18: Tipo cambio (10)
        $line .= $cantAlicuotas;                                          // 19: Cant alícuotas (1)
        $line .= ' ';                                                     // 20: Código operación (1)
        $line .= $this->formatAmount($inv->otros_impuestos ?? 0);        // 21: Otros tributos (15)
        $line .= $this->formatDate($inv->due_date ?? $inv->issue_date);  // 22: Fecha vto/pago (8)

        $lines[] = $line; // Debe ser exactamente 266 chars
    }

    // Incluir notas de crédito del período
    // (misma lógica con CreditNote, adaptando tipo de comprobante)

    return implode("\r\n", $lines);
}
```

**Nota importante:** Adaptar la lógica para incluir `CreditNote` del mismo período como comprobantes de venta
(NC A = 003, NC B = 008, NC C = 013).

### 1.4 — Generación de Ventas Alícuotas (62 chars por registro)

```php
public function generateVentasAlicuotas(int $companyId, int $year, int $month): string
{
    $lines = [];

    $invoices = SalesInvoice::where('company_id', $companyId)
        ->whereYear('issue_date', $year)
        ->whereMonth('issue_date', $month)
        ->whereNotNull('cae')
        ->with('pointOfSale')
        ->get();

    foreach ($invoices as $inv) {
        $tipoCbte = $this->getVoucherTypeCode($inv->voucher_type, 'ventas');
        $pv = $this->formatNumeric($inv->pointOfSale->afip_pos_number ?? $inv->point_of_sale, 5);
        $nroCbte = $this->formatNumeric($inv->afip_voucher_number ?? $inv->invoice_number, 20);

        // Una línea por cada alícuota con monto > 0
        $alicuotas = $this->getAlicuotasFromInvoice($inv);
        foreach ($alicuotas as $alicuota) {
            $line = '';
            $line .= $this->formatNumeric($tipoCbte, 3);            // 1: Tipo cbte (3)
            $line .= $pv;                                             // 2: PV (5)
            $line .= $nroCbte;                                        // 3: Nro cbte (20)
            $line .= $this->formatAmount($alicuota['neto']);         // 4: Neto gravado (15)
            $line .= $alicuota['codigo'];                             // 5: Alícuota (4)
            $line .= $this->formatAmount($alicuota['iva']);          // 6: IVA liquidado (15)

            $lines[] = $line; // Debe ser exactamente 62 chars
        }
    }

    // Incluir alícuotas de notas de crédito

    return implode("\r\n", $lines);
}
```

Helper para extraer alícuotas de una factura:

```php
private function getAlicuotasFromInvoice($invoice): array
{
    $alicuotas = [];

    // Calcular neto gravado al 21% = iva_21 / 0.21
    if ($invoice->iva_21 > 0) {
        $alicuotas[] = [
            'neto'   => round($invoice->iva_21 / 0.21, 2),
            'iva'    => $invoice->iva_21,
            'codigo' => self::ALICUOTAS['21'],
        ];
    }
    if ($invoice->iva_10_5 > 0) {
        $alicuotas[] = [
            'neto'   => round($invoice->iva_10_5 / 0.105, 2),
            'iva'    => $invoice->iva_10_5,
            'codigo' => self::ALICUOTAS['10.5'],
        ];
    }
    if ($invoice->iva_27 > 0) {
        $alicuotas[] = [
            'neto'   => round($invoice->iva_27 / 0.27, 2),
            'iva'    => $invoice->iva_27,
            'codigo' => self::ALICUOTAS['27'],
        ];
    }

    return $alicuotas;
}
```

### 1.5 — Generación de Compras Cabecera (325 chars por registro)

Misma lógica que ventas pero con `PurchaseInvoice`, campos del vendedor (proveedor), y campos
adicionales de compras (crédito fiscal computable, CUIT emisor/corredor, etc.).

Campos que cambian respecto a ventas:
- Campo 5: Despacho de importación (16 chars alfanumérico) → dejar en blanco (no importamos)
- Campo 12: Percepciones de IVA (en compras se discrimina, en ventas no)
- Campo 21: Crédito fiscal computable (15) → igual al IVA total de la factura
- Campo 23-24: CUIT y denominación emisor/corredor → dejar vacío si no aplica
- Campo 25: IVA comisión → 0

El `cuit_emisor` de la factura de compra (post v1.28.0) se usa como número de identificación del vendedor.

### 1.6 — Generación de Compras Alícuotas (84 chars por registro)

Similar a ventas pero con 8 campos (incluye doc vendedor):
- Tipo cbte (3) + PV (5) + Nro cbte (20) + Cod doc vendedor (2) + Nro id vendedor (20) + Neto gravado (15) + Alícuota (4) + IVA liquidado (15)

```php
class LibroIvaController extends Controller
{
    public function index()
    {
        $this->authorize('ventas.section');
        return view('libro-iva.index');
    }

    public function preview(Request $request)
    {
        $this->authorize('ventas.section');
        $request->validate([
            'year'  => 'required|integer|min:2020|max:2030',
            'month' => 'required|integer|min:1|max:12',
        ]);

        $companyId = active_company_id();
        $year = $request->year;
        $month = $request->month;

        // Resumen de totales para la preview
        $ventasCount = SalesInvoice::where('company_id', $companyId)
            ->whereYear('issue_date', $year)->whereMonth('issue_date', $month)
            ->whereNotNull('cae')->count();
        // + contar CreditNote del período

        $comprasCount = PurchaseInvoice::where('company_id', $companyId)
            ->whereYear('issue_date', $year)->whereMonth('issue_date', $month)->count();

        $ventasTotal = SalesInvoice::where('company_id', $companyId)
            ->whereYear('issue_date', $year)->whereMonth('issue_date', $month)
            ->whereNotNull('cae')->sum('total');

        $comprasTotal = PurchaseInvoice::where('company_id', $companyId)
            ->whereYear('issue_date', $year)->whereMonth('issue_date', $month)->sum('total');

        $debitoFiscal = SalesInvoice::where('company_id', $companyId)
            ->whereYear('issue_date', $year)->whereMonth('issue_date', $month)
            ->whereNotNull('cae')
            ->selectRaw('COALESCE(SUM(iva_21), 0) + COALESCE(SUM(iva_10_5), 0) + COALESCE(SUM(iva_27), 0) as total_iva')
            ->value('total_iva');

        $creditoFiscal = PurchaseInvoice::where('company_id', $companyId)
            ->whereYear('issue_date', $year)->whereMonth('issue_date', $month)
            ->selectRaw('COALESCE(SUM(iva_21), 0) + COALESCE(SUM(iva_10_5), 0) + COALESCE(SUM(iva_27), 0) as total_iva')
            ->value('total_iva');

        return view('libro-iva.preview', compact(
            'year', 'month', 'ventasCount', 'comprasCount',
            'ventasTotal', 'comprasTotal', 'debitoFiscal', 'creditoFiscal'
        ));
    }

    public function download(Request $request)
    {
        $this->authorize('ventas.section');
        $request->validate([
            'year'  => 'required|integer',
            'month' => 'required|integer|min:1|max:12',
        ]);

        $service = new LibroIvaService();
        $files = $service->generate(active_company_id(), $request->year, $request->month);

        $periodo = $request->year . str_pad($request->month, 2, '0', STR_PAD_LEFT);

        // Generar ZIP
        $zipPath = tempnam(sys_get_temp_dir(), 'libro_iva_') . '.zip';
        $zip = new \ZipArchive();
        $zip->open($zipPath, \ZipArchive::CREATE);

        foreach ($files as $name => $content) {
            $zip->addFromString("{$name}_{$periodo}.txt", $content);
        }

        $zip->close();

        return response()->download($zipPath, "libro_iva_{$periodo}.zip")
            ->deleteFileAfterSend(true);
    }
}
```

## Notas / Consideraciones

- **Formato de referencia oficial:** https://www.afip.gob.ar/iva/documentos/libro-iva-digital-diseno-registros.pdf
- Los archivos usan separador de línea `\r\n` (CRLF) como es estándar para archivos de importación AFIP.
- El campo "Código de operación" (1 char) se deja en espacio en blanco para operaciones normales.
  Valores especiales: 'Z' = exportaciones, 'X' = exentas, 'E' = exento, 'N' = no gravado.
- Las notas de crédito de venta deben incluirse en el archivo de ventas con tipo de comprobante 003/008/013.
- El campo "Crédito Fiscal Computable" en compras es el IVA total de la factura (para RI que computan el 100%).
- No se incluyen importaciones de bienes ni servicios (archivos separados en la spec).
  Si el cliente necesita eso en el futuro, se agrega como versión incremental.
- **Moneda:** solo pesos argentinos (PES). Operaciones en moneda extranjera quedan fuera del scope.
- El tipo de cambio es fijo 1.000000 para operaciones en pesos.
- El diseño TurIVA (con campo extra de reintegro decreto 1043/2016) no se implementa por ahora.
  El diseño estándar es suficiente para el 99% de los casos.
