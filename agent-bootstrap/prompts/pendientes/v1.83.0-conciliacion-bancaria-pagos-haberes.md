## Descripción

Extiende el motor de conciliación bancaria para reconocer `PayrollPayment` como tipo
de registro reconciliable. El contador puede, desde la pantalla de conciliación de un
extracto, ver los pagos de haberes confirmados en la columna "Registros del Sistema",
filtrarlos por tipo "Haberes", y vincularlos manualmente al movimiento bancario
correspondiente.

**Realidad operativa:** por cada período hay habitualmente 2 o más débitos de haberes
en el banco (uno por empleados del mismo banco, otro por transferencias interbancarias,
eventualmente uno por liquidación final de empleado). Cada débito se reconcilia contra
el `PayrollPayment` del lote correspondiente.

**La vinculación es siempre manual** — el motor sugiere (badge "Sugerido") pero el
operador confirma haciendo click en "Vincular".

## PASO 1 — Categorización automática en BankMovement

En `app/Models/BankMovement.php`, método `categorize()`, agregar el patrón de haberes:

```php
// Agregar en $conceptPatterns (antes del patrón genérico OG-DEBITO):
'/OG-DEB[^\s]*\s*\d*\s*HABERE/i' => 'haberes',
'/ACRED.*SUELDO|PAGO.*HABER|SUELDO.*PAGO/i' => 'haberes',
```

El patrón debe capturar variantes como:
- `OG-DEB./CRED 58356HABERES OL`
- `OG-DEBITO 58356HABERES OL`
- `PAGO HABERES ABRIL`

**Importante:** agregar este patrón ANTES del catch-all `'/OG-DEBITO|OG-DEB/i' => 'debito_automatico'`
para que los haberes sean categorizados específicamente.

## PASO 2 — BankReconciliationService: buscar PayrollPayments

En `app/Services/BankReconciliationService.php`, extender el método `findMatch()`:

**Agregar búsqueda de `PayrollPayment` para movimientos débito** (después de la búsqueda de `PaymentOrder`):

```php
// 3. Buscar en PayrollPayments (para débitos de haberes)
if ($movement->is_debit) {
    $amount = $movement->debit;
    $date = $movement->date;
    
    $pp = PayrollPayment::where('company_id', $companyId)
        ->where('status', 'confirmado')
        ->where('total', $amount)
        // Ventana de ±5 días (el banco acredita hasta el día siguiente del pago)
        ->whereBetween('payment_date', [
            $date->copy()->subDays(5),
            $date->copy()->addDays(5),
        ])
        ->whereDoesntHave('reconciledMovements')
        ->first();
    
    if ($pp) {
        $exact = $pp->payment_date && $pp->payment_date->eq($date);
        return [
            'confidence' => $exact ? 'exact' : 'probable',
            'record' => $pp,
            'label' => "Haberes {$pp->period_label} ({$pp->employee_count} empleados) - $" . number_format($pp->total, 2, ',', '.'),
            'reason' => $exact ? 'Monto y fecha exactos' : 'Monto exacto, fecha cercana',
        ];
    }
    
    // Match por monto sin fecha (si payment_date es null en el PayrollPayment):
    // Buscar mismo monto + mismo mes en el period (year/month del BankMovement)
    $pp = PayrollPayment::where('company_id', $companyId)
        ->where('status', 'confirmado')
        ->where('total', $amount)
        ->where('year', $date->year)
        ->where('month', $date->month)
        ->whereNull('payment_date')
        ->whereDoesntHave('reconciledMovements')
        ->first();
    
    if ($pp) {
        return [
            'confidence' => 'probable',
            'record' => $pp,
            'label' => "Haberes {$pp->period_label} ({$pp->employee_count} empleados) - $" . number_format($pp->total, 2, ',', '.'),
            'reason' => 'Monto exacto, mismo período',
        ];
    }
}
```

**También agregar** al método `getMatchableRecords()` (o el método que carga los registros del sistema para mostrar en el panel):
- Incluir `PayrollPayment` con status `confirmado`, company_id activo, sin `reconciledMovements`, ordenados por `year DESC, month DESC`
