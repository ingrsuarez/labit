# v1.13.0 — Nomenclador en tiempo real (sin duplicación)

**VERSIÓN:** v1.13.0  
**SLUG:** nomenclador-tiempo-real  
**DEPENDENCIAS:** v1.10.0  
**ESTIMACIÓN:** 1h  
**PRIORIDAD:** alta

---

## Descripción

Actualmente, para que una obra social tenga prácticas en su nomenclador, hay que
"copiar" manualmente desde un nomenclador base. Esto duplica miles de registros
`insurance_tests` por cada obra social y genera confusión: el usuario asigna un
nomenclador base en la edición de la obra social, pero al entrar a `/nomenclator/{id}`
ve 0 prácticas.

El cambio elimina la necesidad de copiar. La obra social solo necesita dos datos:
- `nomenclator_id` → qué nomenclador base usa (ej. "2016UNI")
- `nbu_value` → su valor NBU (ej. $350)

El precio se calcula en tiempo real:

```
precio = nbu_units (del nomenclador base) × nbu_value (de la obra social)
```

Ejemplo: si el acto bioquímico tiene `nbu_units = 6` en el nomenclador base, y la
obra social tiene `nbu_value = $350`, el precio es `6 × $350 = $2.100`.

### 1.2 — Refactorizar getTestPrice()

Cambiar para que busque primero en propios `insurance_tests`, luego en el
nomenclador base si existe:

```php
public function getTestPrice(int $testId): ?float
{
    // 1. Buscar en insurance_tests propios (overrides)
    $own = $this->nomenclator()->where('test_id', $testId)->first();
    if ($own) {
        return $own->price;
    }

    // 2. Buscar en nomenclador base
    if ($this->nomenclator_id) {
        $baseItem = InsuranceTest::where('insurance_id', $this->nomenclator_id)
            ->where('test_id', $testId)
            ->first();
        if ($baseItem) {
            return $baseItem->nbu_units * ($this->nbu_value ?? 0);
        }
    }

    return null;
}
```

### 1.3 — Refactorizar calculateTestPrice()

Misma lógica con fallback al nomenclador base:

```php
public function calculateTestPrice(Test $test): float
{
    // 1. Override propio
    $own = $this->nomenclator()->where('test_id', $test->id)->first();
    if ($own && $own->price) {
        return (float) $own->price;
    }

    // 2. Nomenclador base
    if ($this->nomenclator_id) {
        $baseItem = InsuranceTest::where('insurance_id', $this->nomenclator_id)
            ->where('test_id', $test->id)
            ->first();
        if ($baseItem) {
            return $baseItem->nbu_units * ($this->nbu_value ?? 0);
        }
    }

    // 3. Fallback: NBU del test × NBU de la obra social
    $nbuUnits = $test->nbu ?? 1;
    return $nbuUnits * ($this->nbu_value ?? 0);
}
```

## PASO 2 — Modificar LabAdmissionController::getTestPrice()

En `app/Http/Controllers/LabAdmissionController.php`, método `getTestPrice()`:

Reemplazar la lógica actual que solo busca en `insurance_tests` propios.
Usar el nomenclador base como fuente principal cuando la obra social tiene
`nomenclator_id`.

```php
public function getTestPrice(Request $request)
{
    $insuranceId = $request->get('insurance_id');
    $testId = $request->get('test_id');

    if (! $insuranceId || ! $testId) {
        return response()->json(['error' => 'Parámetros requeridos'], 400);
    }

    $insurance = Insurance::find($insuranceId);
    $test = Test::find($testId);

    if (! $insurance || ! $test) {
        return response()->json(['error' => 'No encontrado'], 404);
    }

    // 1. Buscar en insurance_tests propios (overrides)
    $ownItem = InsuranceTest::where('insurance_id', $insuranceId)
        ->where('test_id', $testId)
        ->first();

    if ($ownItem) {
        return response()->json([
            'price' => $ownItem->price,
            'nbu_units' => $ownItem->nbu_units,
            'requires_authorization' => $ownItem->requires_authorization,
            'copago' => $ownItem->copago,
            'in_nomenclator' => true,
            'source' => 'own',
        ]);
    }

    // 2. Buscar en nomenclador base
    if ($insurance->nomenclator_id) {
        $baseItem = InsuranceTest::where('insurance_id', $insurance->nomenclator_id)
            ->where('test_id', $testId)
            ->first();

        if ($baseItem) {
            $price = $baseItem->nbu_units * ($insurance->nbu_value ?? 0);
            return response()->json([
                'price' => round($price, 2),
                'nbu_units' => $baseItem->nbu_units,
                'requires_authorization' => $baseItem->requires_authorization ?? false,
                'copago' => $baseItem->copago ?? 0,
                'in_nomenclator' => true,
                'source' => 'base',
            ]);
        }
    }

    // 3. Fallback: calcular por NBU del test
    $nbuUnits = $test->nbu ?? 1;
    $price = $nbuUnits * ($insurance->nbu_value ?? 0);

    return response()->json([
        'price' => round($price, 2),
        'nbu_units' => $nbuUnits,
        'requires_authorization' => false,
        'copago' => 0,
        'in_nomenclator' => false,
        'source' => 'fallback',
    ]);
}
```

## PASO 3 — Modificar InsuranceNomenclatorController::show()

En `app/Http/Controllers/InsuranceNomenclatorController.php`, método `show()`:

Cuando la obra social tiene `nomenclator_id` y NO es un nomenclador base, cargar
las prácticas del nomenclador base en lugar de las propias.

```php
public function show(Insurance $insurance)
{
    $isBaseNomenclator = $insurance->type === 'nomenclador';
    $hasBaseNomenclator = !$isBaseNomenclator && $insurance->nomenclator_id;

    if ($hasBaseNomenclator) {
        // Obra social con nomenclador base: mostrar prácticas del base
        $practices = InsuranceTest::where('insurance_id', $insurance->nomenclator_id)
            ->whereIn('test_id', Test::select('id'))
            ->with('test')
            ->orderBy('id')
            ->get();

        $baseNomenclator = Insurance::find($insurance->nomenclator_id);
    } else {
        // Nomenclador base o sin base: mostrar propias
        $practices = InsuranceTest::where('insurance_id', $insurance->id)
            ->whereIn('test_id', Test::select('id'))
            ->with('test')
            ->orderBy('id')
            ->get();

        $baseNomenclator = null;
    }

    $existingTestIds = $practices->pluck('test_id')->toArray();
    $availableTests = Test::whereNotIn('id', $existingTestIds)
        ->whereNull('parent')
        ->orderBy('code')
        ->get();

    $baseNomenclators = [];
    if (!$isBaseNomenclator) {
        $baseNomenclators = Insurance::where('type', 'nomenclador')
            ->withCount('nomenclator')
            ->orderBy('name')
            ->get();
    }

    return view('lab.nomenclator.show', compact(
        'insurance',
        'practices',
        'availableTests',
        'baseNomenclators',
        'hasBaseNomenclator',
        'baseNomenclator'
    ));
}
```

## PASO 5 — Verificación y limpieza

### 5.1 — Verificar que el banner NBU se mantiene

La sección "Valor NBU de la Obra Social" debe seguir visible para obras sociales
(no para nomencladores base), ya que el `nbu_value` es necesario para el cálculo
en tiempo real.

### 5.2 — Verificar que NO se eliminan datos existentes

Este cambio es **no destructivo**. Los `insurance_tests` existentes de obras
sociales (si los hay) siguen ahí y pueden funcionar como overrides.
La lógica siempre prioriza registros propios sobre los del base.

### 5.3 — Probar flujo completo

1. Entrar a editar una obra social → asignar nomenclador base → guardar
2. Ir a `/nomenclator/{insurance}` → verificar que se ven las prácticas del base
3. Verificar que los precios son `nbu_units × nbu_value`
4. Crear admisión con esa obra social → agregar práctica → verificar precio correcto
5. Editar `nbu_value` de la obra social → verificar que precios se recalculan en vista
