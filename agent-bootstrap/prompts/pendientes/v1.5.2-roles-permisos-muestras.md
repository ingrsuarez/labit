**PRIORIDAD:** alta

---

## Descripción

Extender los 3 roles creados en v1.5.1 (`recepcion-lab`, `tecnico-lab`, `bioquimico`)
con permisos para el módulo de muestras de aguas y alimentos. Sigue el mismo patrón
que el laboratorio clínico pero con diferencias clave:

- En muestras, **todos los roles pueden imprimir y enviar informes** (no solo validados)
- La recepcionista crea la muestra (recepción) y puede imprimir etiquetas
- Solo el bioquímico valida

Flujo: Recepcionista recibe muestra → Técnico/Bioquímico carga determinaciones →
Bioquímico valida → Todos pueden imprimir y enviar.

### Matriz de permisos

| Permiso | recepcion-lab | tecnico-lab | bioquimico |
|---|:---:|:---:|:---:|
| `samples.section` | x | x | x |
| `samples.index` | x | x | x |
| `samples.create` | x | | |

## PASO 1 — Agregar los permisos de muestras al seeder

Editar `database/seeders/RolesAndPermissionsSeeder.php`:

1. Agregar los permisos nuevos al array de permisos:
   - `samples.section`
   - `samples.index`, `samples.create`, `samples.edit`, `samples.show`
   - `samples-results.create`, `samples-results.edit`, `samples-results.validate`
   - `samples-reports.preview`, `samples-reports.print`, `samples-reports.send`
   - `samples-labels.print`

2. Extender los 3 roles existentes con los permisos de muestras según la matriz.

3. El rol `admin` debe tener todos los permisos nuevos.

4. Ejecutar:
   ```bash
   php artisan db:seed --class=RolesAndPermissionsSeeder
   php artisan permission:cache-reset
   php artisan cache:clear
   ```

**Verificación:** los 3 roles tienen los permisos de muestras. Verificar con tinker.

## PASO 3 — Proteger las vistas con @can

Recorrer las vistas del módulo de muestras y agregar directivas `@can`:
