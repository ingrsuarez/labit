## Descripción

Cambia el formato de numeración de protocolos en todos los módulos para incluir un prefijo
de módulo y la fecha del día, con un contador diario de 4 dígitos que se reinicia cada día.

Formato: `[PREFIJO][AA][MM][DD][NNNN]`

| Módulo | Prefijo | Ejemplo |
|---|---|---|
| Aguas/Alimentos | A | A2603290001 |
| Clínico | C | C2603290001 |
| Veterinario | V | V2603290001 |

Los protocolos existentes de aguas se mantienen sin modificar. Los de clínico (datos de prueba)
se limpian.

## PASO 1 — Crear trait `GeneratesProtocolNumber`

Crear `app/Traits/GeneratesProtocolNumber.php` con lógica reutilizable para los 3 módulos:

```php
<?php

namespace App\Traits;

trait GeneratesProtocolNumber
{
    /**
     * Genera número de protocolo con formato [PREFIJO][AAMMDD][NNNN]
     * El contador NNNN se reinicia a 0001 cada día, por prefijo independiente.
     *
     * @param string $prefix  Prefijo del módulo: 'A', 'C' o 'V'
     * @param string $column  Nombre de la columna de protocol_number en la tabla
     * @param string|null $dateColumn  Columna de fecha para filtrar por día (null usa created_at)
     */
    public static function generateProtocolNumber(string $prefix = 'A', string $column = 'protocol_number', ?string $dateColumn = null): string
    {
        $today = now();
        $dateStr = $today->format('ymd'); // AAMMDD
        $todayPrefix = $prefix . $dateStr; // ej: A260329

        // Buscar el último protocolo del día con este prefijo
        $lastRecord = static::where($column, 'like', $todayPrefix . '%')
            ->orderBy($column, 'desc')
            ->first();

        if ($lastRecord) {
            $lastNumber = (int) substr($lastRecord->$column, -4);
            $newNumber = $lastNumber + 1;
        } else {
            $newNumber = 1;
        }

        return $todayPrefix . str_pad($newNumber, 4, '0', STR_PAD_LEFT);
    }
}
```

## PASO 2 — Refactorizar `Sample::generateProtocolNumber()`

En `app/Models/Sample.php`:

1. Agregar `use App\Traits\GeneratesProtocolNumber;`
2. Agregar `use GeneratesProtocolNumber;` dentro de la clase
3. **Eliminar** el método `generateProtocolNumber()` existente (el trait lo reemplaza)
4. Agregar un método estático que invoque al trait con el prefijo correcto:

```php
use App\Traits\GeneratesProtocolNumber;

class Sample extends Model
{
    use Auditable, HasFactory, GeneratesProtocolNumber;

    // ... fillable, casts, etc ...

    /**
     * Genera número de protocolo para aguas/alimentos: A[AAMMDD][NNNN]
     */
    public static function generateProtocolNumber(): string
    {
        return static::generatePrefixedProtocolNumber('A');
    }
}
```

> **Nota:** Renombrar el método del trait a `generatePrefixedProtocolNumber` para evitar conflicto
> con el método existente que se usa en los controllers. El método público `generateProtocolNumber()`
> en cada modelo llama internamente a `generatePrefixedProtocolNumber` del trait con su prefijo.

Actualizar el trait para que el método se llame `generatePrefixedProtocolNumber`:

```php
trait GeneratesProtocolNumber
{
    public static function generatePrefixedProtocolNumber(string $prefix, string $column = 'protocol_number'): string
    {
        // ... lógica del PASO 1 ...
    }
}
```

En `SampleController::store()` no hay que cambiar nada — ya llama `Sample::generateProtocolNumber()`.

**Verificación:** Crear una muestra nueva y verificar que el protocolo tiene formato `A2603XX0001`.

## Notas / Consideraciones

- El método `generatePrefixedProtocolNumber` usa `LIKE` sobre la columna `protocol_number` para filtrar por día y prefijo. Esto funciona bien con los índices existentes.
- Si se crean más de 9999 protocolos en un día para un mismo módulo, el sistema seguiría funcionando (5 dígitos), pero el formato visual se extiende. Con 4 dígitos se cubren hasta 9999 protocolos/día que es más que suficiente.
- Los protocolos viejos (`2026-00001`) no empiezan con A/C/V, por lo que no colisionan con el nuevo formato.
- Al buscar protocolos, el buscador actual usa `LIKE` y funciona con ambos formatos.
- El trait queda preparado para que `VetAdmission` (v1.24.0) solo haga `use GeneratesProtocolNumber` y llame `generatePrefixedProtocolNumber('V')`.
