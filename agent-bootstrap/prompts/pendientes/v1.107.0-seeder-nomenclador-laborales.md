## Descripción

Crear un seeder **standalone para producción** que importe el archivo
`docs/Nomenclador laborales.xlsx` como nomenclador base (`Insurance` con
`type = 'nomenclador'`, nombre **"Nomenclador laborales"**).

Este nomenclador se usará manualmente en coberturas de empresas laborales
(ej. Santa Cruz O&G): desde la UI de obras sociales se asigna en el campo
**"Nomenclador para Facturar"** (`nomenclator_id`). El seeder **no** crea ni
modifica coberturas laborales.

Hoy, coberturas `type = 'laborales'` sin `nomenclator_id` hacen fallback al NBU
de la determinación (`test.nbu × insurance.nbu_value`). Con este nomenclador,
el pricing usará `nbu_units` del nomenclador base × `nbu_value` de la cobertura
(comportamiento estándar v1.13.0).

**Estructura del Excel (verificado):**
- Fila 1: título (*"PRACTICAS POR ENTRADA ABRIL 2026 AÑELO"*) — **saltar**
- Fila 2: encabezados (`CODIGO`, `PRACTICA`, `nbu`)
- Filas 3+: datos (~87 prácticas)
- Columna A = código (numérico, ej. `15`, `19`, `22`)
- Columna B = nombre de práctica
- Columna C = valor NBU

**Comando de producción:**
```bash
php artisan db:seed --class=NomencladorLaboralesSeeder
```

**NO incluir** en `DatabaseSeeder` (ejecución manual en prod).

## PASO 1 — Crear `NomencladorLaboralesSeeder.php`

Crear `database/seeders/NomencladorLaboralesSeeder.php`.

### Requisitos

1. **Archivo fuente:** `base_path('docs/Nomenclador laborales.xlsx')`

   - Si ya existía con otro `type`, forzar `type = 'nomenclador'`.

3. **Loop de filas:** desde fila **3** hasta `getHighestRow()`.
   - A = código, B = nombre, C = NBU.
   - Saltar filas con código vacío.
   - Normalizar código a string: `(string) $codigo` (los códigos son numéricos en Excel).

4. **Tests:** mismo criterio que `NomencladoresExcelSeeder`:
   - Buscar `Test::where('code', $codigo)`.
   - Si existe → actualizar `nbu`.
   - Si no → crear con campos null como en el seeder de referencia.

5. **InsuranceTest:** upsert por `(insurance_id, test_id)`:
   - Actualizar `nbu_units` y `requires_authorization = false`.
   - Crear con `price => null`, `copago => 0` si no existe.

6. **`parseNbu()`:** copiar la implementación de `NomencladoresExcelSeeder`
   (soporta formato AR: miles con punto, decimal con coma).

7. **Logging al final:**
   - Tests nuevos / actualizados

## Notas / Consideraciones

- Los códigos del Excel (15, 19, 22…) pueden coincidir con tests existentes del
  nomenclador bioquímico o ser distintos. El seeder matchea **por `code`**; si un
  código ya existe, actualiza nombre/NBU del test existente (mismo comportamiento
  que v1.10.0).
- Santa Cruz usa `SantaCruzTestMapping` para resolver XML → `Test`; el nomenclador
  afecta **pricing**, no el mapeo de códigos de prestación.
- Post-deploy en producción: asignar manualmente `nomenclator_id` en la cobertura
  Santa Cruz O&G (y otras laborales) + configurar `nbu_value` de la cobertura.
- El archivo `.xlsx` debe estar en el repo/deploy; no commitear `.env`.
