# v1.100.0 — Hub de navegación RRHH unificado

**VERSIÓN:** v1.100.0  
**SLUG:** rrhh-hub-navegacion  
**DEPENDENCIAS:** ninguna (complementa v1.66.0 ya completada)  
**ESTIMACIÓN:** 2–3 h  
**PRIORIDAD:** media (UX / navegación)

---

## Descripción

Consolidar la navegación de Recursos Humanos en un único punto de entrada en el sidebar
y en `/rrhh`, eliminando los ítems duplicados **Personal**, **Ausencias** y **Liquidaciones**.

Hoy `/rrhh` muestra el panel analítico (KPIs, gráficos, accesos rápidos) y las tres
secciones tienen páginas intermedias en `admin/personal`, `admin/ausencias` y
`admin/liquidaciones` con grids de cards. Esta versión:

1. Convierte `/rrhh` en un **hub de navegación** con tres bloques titulados y cards debajo.
2. Mueve el panel analítico actual a **`/rrhh/resumen`** (mudanza 1:1, sin cambiar queries).
3. Simplifica el sidebar admin a un solo ítem «Recursos Humanos».
4. Amplía el acceso al hub a usuarios con cualquier permiso de sección RRHH (no solo admin/contador).
5. Redirige las rutas legacy `admin.section.*` al hub con ancla.

**Documento de diseño obligatorio:** `docs/design/v1.100.0-rrhh-hub-navegacion.md`

## PASO 1 — Extraer definición de navegación RRHH

Crear `app/Support/RrhhNavigation.php` (o `app/Services/RrhhNavigationService.php`)
con método estático `sections(): array` que devuelva la estructura de las tres secciones,
**copiando los arrays actuales** de `AdminSectionController` (`personal`, `ausencias`,
`liquidaciones`) sin cambiar textos, rutas ni íconos SVG.

Estructura de cada sección:

```php
[
    'key' => 'personal',
    'permission' => 'personal.section',
    'title' => 'Personal',
    'description' => '...',
    'items' => [ /* mismos ítems */ ],
],
```

Método auxiliar recomendado:

```php
public static function sectionsForUser(?User $user): array
```

- Filtrar secciones: solo si `$user->can($section['permission'])`.
- Filtrar ítems: si existe `$item['permission']`, incluir solo si `$user->can(...)`.
- Si una sección queda con 0 ítems visibles, **no incluirla** en el resultado.

Refactorizar `AdminSectionController::personal()`, `ausencias()` y `liquidaciones()` para
usar `RrhhNavigation::section('personal')` (o equivalente) y seguir retornando
`view('admin.section', compact('section'))` — útil mientras existan redirects.

**Verificación:** Los tres métodos del controller siguen devolviendo los mismos ítems que antes.

## PASO 3 — Separar hub y resumen en `RrhhController`

### 3.1 — `index()` → hub

- Autorización: usuario autenticado con **al menos uno** de:
  `personal.section`, `ausencias.section`, `liquidaciones.section`, o rol `admin`/`contador`.
  Si no cumple → `abort(403)`.
- **Sin queries** de KPI (no Employee::count, etc.).
- `$sections = RrhhNavigation::sectionsForUser(auth()->user());`
- `return view('rrhh.index', compact('sections'));`

### 3.2 — `resumen()` → panel analítico

- Mover **toda** la lógica actual de `index()` (queries + compact) a `resumen()`.
- Autorización: solo `admin` o `contador` (igual que hoy).
- `return view('rrhh.resumen', compact(...));` — mismas variables que hoy.

### 3.3 — Vista `rrhh/resumen.blade.php`

- Copiar el contenido actual de `resources/views/rrhh/index.blade.php`.
- Cambiar título H1 a **«Resumen de Recursos Humanos»** y subtítulo a
  **«Indicadores y actividad reciente»**.
- Agregar link arriba: `← Volver a Recursos Humanos` → `route('rrhh.index')`
  (`text-sm text-gray-500 hover:text-gray-700`).
- Mantener timestamp «Última actualización».

### 3.4 — Vista `rrhh/index.blade.php` → hub

Reemplazar contenido por el hub según diseño:

```blade
<x-admin-layout title="Recursos Humanos">
    <div class="p-6 space-y-8">
        {{-- Header: título, subtítulo, link Ver resumen (solo admin|contador) --}}
        @foreach ($sections as $section)
            <section id="{{ $section['key'] }}" class="border-b border-gray-200 pb-8 last:border-0">
                <h2 class="text-lg font-semibold text-gray-900">{{ $section['title'] }}</h2>
                <p class="text-sm text-gray-500 mt-0.5 mb-4">{{ $section['description'] }}</p>
                @include('admin.partials.section-cards', ['items' => $section['items']])
            </section>
        @endforeach
    </div>
</x-admin-layout>
```

Link «Ver resumen →» solo si `auth()->user()?->hasAnyRole(['admin', 'contador'])`.

**Verificación:**

- `/rrhh` muestra «Recursos Humanos» y bloques con cards (no KPIs).
- `/rrhh/resumen` muestra el panel analítico completo como antes.

### 5.1 — Eliminar

`@can('ausencias.section')` … Ausencias, `@can('liquidaciones.section')` … Liquidaciones.

### 5.2 — Actualizar ítem «Recursos Humanos»

**Visibilidad** — reemplazar `@if (auth()->user()?->hasAnyRole(['admin', 'contador']))` por:

```blade

    auth()->user()?->hasAnyRole(['admin', 'contador'])
    || auth()->user()?->can('personal.section')
    || auth()->user()?->can('ausencias.section')
    || auth()->user()?->can('liquidaciones.section')

```

**Estado activo** — expandir `request()->routeIs(...)`:

```blade
{{ request()->routeIs([
    'rrhh.*',
    'admin.section.personal',
    'admin.section.ausencias',
    'admin.section.liquidaciones',
    'employee.*',
    'job.*',
    'category.*',
    'documents.*',
    'circular.*',
    'manage.chart',
    'non-conformity.*',
    'vacation.*',
    'leave.*',
    'payroll.*',
    'salary.*',
    'payroll-payments.*',
]) ? 'bg-zinc-700 text-white' : '...' }}
```

(Sintaxis válida en Blade: usar `||` encadenado si `routeIs` con array no está disponible en la versión de Laravel del proyecto.)

**Verificación:** Sidebar muestra un solo RRHH; al entrar a `employee.*` el ítem queda activo.
