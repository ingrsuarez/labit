# v3.3.0 — Asientos automáticos desde transacciones

**VERSIÓN:** v3.3.0  
**SLUG:** asientos-automaticos-transacciones  
**DEPENDENCIAS:** v3.0.0 (infra contable base), v3.2.0 (conciliación bancaria), v1.34.0 (opcional — para el campo updates_stock en facturas de compra)  
**ESTIMACIÓN:** 1 día  
**PRIORIDAD:** alta

---

## Descripción

Al guardar o confirmar cada tipo de transacción, el sistema genera automáticamente
el asiento contable correspondiente con partida doble, usando el plan de cuentas
de v3.0.0. Los asientos se generan con `is_automatic = true` y quedan vinculados
al documento fuente vía polimorfismo (`source_type` / `source_id`).

**5 tipos de transacciones cubiertas:**

| Transacción | Cuándo se genera |
|---|---|
| Factura de venta emitida | Al hacer `store()` o al autorizar con AFIP |
| Nota de crédito de venta | Al hacer `store()` de la NC |
| Recibo de cobro | Al hacer `store()` |
| Factura de compra registrada | Al hacer `store()` |
| Orden de pago confirmada | Al cambiar status a `confirmed` |

**Comportamiento de seguridad:**
- Si una cuenta del plan no existe (código no encontrado), el asiento **no se genera**
  y se loguea un warning en `storage/logs/laravel.log` — la operación continúa normalmente
- Si el documento se elimina y tenía asiento automático, el asiento se elimina en cascada
- No se puede editar un asiento automático desde el formulario del documento — solo desde el Libro Diario (futuro)

## PASO 1 — Servicio de asientos automáticos

Crear `app/Services/AccountingEntryService.php`

Este servicio centraliza la lógica de generación de asientos para no contaminar los controladores.

```php
namespace App\Services;

use App\Models\AccountingAccount;
use App\Models\JournalEntry;
use App\Models\JournalEntryLine;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Support\Facades\Log;

class AccountingEntryService
{
    /**
     * Resuelve una cuenta contable por código.
     * Retorna null si no existe (no lanza excepción).
     */
    private function account(string $code): ?AccountingAccount
    {
        return AccountingAccount::where('code', $code)->where('is_active', true)->first();
    }

    /**
     * Crea un JournalEntry con sus líneas.
     * Valida partida doble antes de guardar.
     * Retorna el asiento creado o null si hay error.
     *
     * @param array $lines  [ ['account_code' => '1.1.04', 'debit' => 1210.00, 'credit' => 0, 'description' => '...'], ... ]
     */
    private function createEntry(
        int $companyId,
        \Carbon\Carbon $date,
        string $description,
        Model $source,
        array $lines
    ): ?JournalEntry {
        // Resolver todas las cuentas antes de crear nada
        $resolvedLines = [];
        foreach ($lines as $line) {
            $account = $this->account($line['account_code']);
            if (!$account) {
                Log::warning("AccountingEntryService: cuenta '{$line['account_code']}' no encontrada. Asiento no generado para " . get_class($source) . " #{$source->id}");
                return null;
            }
            $resolvedLines[] = array_merge($line, ['account_id' => $account->id]);
        }

        // Verificar partida doble
        $totalDebit  = collect($resolvedLines)->sum('debit');
        $totalCredit = collect($resolvedLines)->sum('credit');
        if (abs($totalDebit - $totalCredit) > 0.01) {
            Log::warning("AccountingEntryService: asiento no balanceado (DB:{$totalDebit} CR:{$totalCredit}) para " . get_class($source) . " #{$source->id}");
            return null;
        }

        return \DB::transaction(function () use ($companyId, $date, $description, $source, $resolvedLines) {
            $entry = JournalEntry::create([
                'company_id'   => $companyId,
                'date'         => $date,
                'number'       => JournalEntry::nextNumber($companyId, $date->year),
                'description'  => $description,
                'source_type'  => get_class($source),
                'source_id'    => $source->id,
                'is_automatic' => true,
                'created_by'   => auth()->id(),
            ]);

            foreach ($resolvedLines as $line) {
                JournalEntryLine::create([
                    'journal_entry_id'       => $entry->id,
                    'accounting_account_id'  => $line['account_id'],
                    'debit'                  => $line['debit'] ?? 0,
                    'credit'                 => $line['credit'] ?? 0,
                    'description'            => $line['description'] ?? null,
                ]);
            }

            return $entry;
        });
    }

    // ─────────────────────────────────────────────────────────────────
    // 1. FACTURA DE VENTA
    // ─────────────────────────────────────────────────────────────────

    /**
     * Asiento de venta:
     *   DB: 1.1.04 Deudores por Ventas       (total bruto con IVA)
     *   CR: 4.1.01 Ventas                    (neto sin IVA)
     *   CR: 2.1.03 IVA Débito Fiscal         (monto IVA)
     *
     * Si iva_amount = 0 (exento/no gravado):
     *   DB: 1.1.04 Deudores por Ventas       (total)
     *   CR: 4.1.01 Ventas                    (total)
     */
    public function fromSalesInvoice(\App\Models\SalesInvoice $invoice): ?JournalEntry
    {
        $neto = round($invoice->subtotal ?? ($invoice->total - ($invoice->iva_amount ?? 0)), 2);
        $iva  = round($invoice->iva_amount ?? 0, 2);
        $total = round($invoice->total, 2);

        $lines = [
            [
                'account_code' => '1.1.04',
                'debit'        => $total,
                'credit'       => 0,
                'description'  => "FC #{$invoice->full_number} - {$invoice->customer->name}",
            ],
            [
                'account_code' => '4.1.01',
                'debit'        => 0,
                'credit'       => $neto,
                'description'  => "Venta neta",
            ],
        ];

        if ($iva > 0) {
            $lines[] = [
                'account_code' => '2.1.03',
                'debit'        => 0,
                'credit'       => $iva,
                'description'  => "IVA débito fiscal",
            ];
        }

        return $this->createEntry(
            $invoice->company_id,
            $invoice->issue_date,
            "Venta — FC {$invoice->full_number}",
            $invoice,
            $lines
        );
    }

    // ─────────────────────────────────────────────────────────────────
    // 2. NOTA DE CRÉDITO DE VENTA
    // ─────────────────────────────────────────────────────────────────

    /**
     * Asiento inverso al de la factura de venta:
     *   DB: 4.1.01 Ventas                    (neto)
     *   DB: 2.1.03 IVA Débito Fiscal         (IVA)
     *   CR: 1.1.04 Deudores por Ventas       (total)
     */
    public function fromCreditNote(\App\Models\SalesInvoice $creditNote): ?JournalEntry
    {
        $neto  = round($creditNote->subtotal ?? ($creditNote->total - ($creditNote->iva_amount ?? 0)), 2);
        $iva   = round($creditNote->iva_amount ?? 0, 2);
        $total = round($creditNote->total, 2);

        $lines = [
            [
                'account_code' => '4.1.01',
                'debit'        => $neto,
                'credit'       => 0,
                'description'  => "NC #{$creditNote->full_number} - {$creditNote->customer->name}",
            ],
            [
                'account_code' => '1.1.04',
                'debit'        => 0,
                'credit'       => $total,
                'description'  => "NC {$creditNote->full_number}",
            ],
        ];

        if ($iva > 0) {
            $lines[] = [
                'account_code' => '2.1.03',
                'debit'        => $iva,
                'credit'       => 0,
                'description'  => "IVA débito fiscal NC",
            ];
        }

        return $this->createEntry(
            $creditNote->company_id,
            $creditNote->issue_date,
            "Nota de Crédito — {$creditNote->full_number}",
            $creditNote,
            $lines
        );
    }

    // ─────────────────────────────────────────────────────────────────
    // 3. RECIBO DE COBRO
    // ─────────────────────────────────────────────────────────────────

    /**
     * Asiento de cobro:
     *   DB: [cuenta del banco/caja según medio de pago]
     *   CR: 1.1.04 Deudores por Ventas
     *
     * Mapeo de medio de pago → cuenta contable:
     *   - efectivo  → 1.1.01 Caja
     *   - transferencia / deposito / cheque → la cuenta contable del BankAccount vinculado
     *     Si no hay bank_account_id → 1.1.02 Bancos Cuenta Corriente (default)
     */
    public function fromCollectionReceipt(\App\Models\CollectionReceipt $receipt): ?JournalEntry
    {
        $bankAccountCode = $this->resolveBankAccountCode($receipt->payment_method, $receipt->bank_account_id);

        return $this->createEntry(
            $receipt->company_id,
            \Carbon\Carbon::parse($receipt->date),
            "Cobro — RC #{$receipt->number}",
            $receipt,
            [
                [
                    'account_code' => $bankAccountCode,
                    'debit'        => round($receipt->total, 2),
                    'credit'       => 0,
                    'description'  => "Cobro RC #{$receipt->number} - {$receipt->customer->name}",
                ],
                [
                    'account_code' => '1.1.04',
                    'debit'        => 0,
                    'credit'       => round($receipt->total, 2),
                    'description'  => "Cancelación deuda RC #{$receipt->number}",
                ],
            ]
        );
    }

    // ─────────────────────────────────────────────────────────────────
    // 4. FACTURA DE COMPRA
    // ─────────────────────────────────────────────────────────────────

    /**
     * Asiento de compra:
     *   DB: [cuenta de gasto según ítems]  (total neto)
     *   DB: 1.1.06 IVA Crédito Fiscal       (monto IVA — si aplica)
     *   CR: 2.1.01 Proveedores               (total bruto)
     *
     * Mapeo de cuenta de gasto por tipo de ítem:
     *   - Ítem con supply_id (insumo)  → 5.1.01 Costo de Mercaderías Vendidas
     *   - Ítem sin supply_id (servicio) → 5.1.02 Costo de Servicios
     *   - Si todos son del mismo tipo: una sola línea de débito
     *   - Si hay mezcla: dos líneas de débito (una por tipo)
     */
    public function fromPurchaseInvoice(\App\Models\PurchaseInvoice $invoice): ?JournalEntry
    {
        $invoice->loadMissing('items');

        // Separar ítems con supply y sin supply
        $itemsWithSupply    = $invoice->items->whereNotNull('supply_id');
        $itemsWithoutSupply = $invoice->items->whereNull('supply_id');

        // Calcular totales netos (sin IVA)
        $netoInsumos   = round($itemsWithSupply->sum(fn($i) => $i->unit_price * $i->quantity), 2);
        $netoServicios = round($itemsWithoutSupply->sum(fn($i) => $i->unit_price * $i->quantity), 2);
        $totalIva      = round($invoice->total_iva ?? 0, 2);
        $totalBruto    = round($invoice->total, 2);

        $lines = [];

        if ($netoInsumos > 0) {
            $lines[] = [
                'account_code' => '5.1.01',
                'debit'        => $netoInsumos,
                'credit'       => 0,
                'description'  => "Insumos — FC compra {$invoice->full_number}",
            ];
        }

        if ($netoServicios > 0) {
            $lines[] = [
                'account_code' => '5.1.02',
                'debit'        => $netoServicios,
                'credit'       => 0,
                'description'  => "Servicios — FC compra {$invoice->full_number}",
            ];
        }

        if ($totalIva > 0) {
            $lines[] = [
                'account_code' => '1.1.06',
                'debit'        => $totalIva,
                'credit'       => 0,
                'description'  => "IVA crédito fiscal",
            ];
        }

        $lines[] = [
            'account_code' => '2.1.01',
            'debit'        => 0,
            'credit'       => $totalBruto,
            'description'  => "Deuda proveedor {$invoice->supplier->name} — FC {$invoice->full_number}",
        ];

        return $this->createEntry(
            $invoice->company_id,
            \Carbon\Carbon::parse($invoice->issue_date),
            "Compra — FC {$invoice->full_number}",
            $invoice,
            $lines
        );
    }

    // ─────────────────────────────────────────────────────────────────
    // 5. ORDEN DE PAGO CONFIRMADA
    // ─────────────────────────────────────────────────────────────────

    /**
     * Asiento de pago:
     *   DB: 2.1.01 Proveedores
     *   CR: [cuenta del banco/caja según medio de pago]
     */
    public function fromPaymentOrder(\App\Models\PaymentOrder $paymentOrder): ?JournalEntry
    {
        $bankAccountCode = $this->resolveBankAccountCode($paymentOrder->payment_method, $paymentOrder->bank_account_id);

        return $this->createEntry(
            $paymentOrder->company_id,
            \Carbon\Carbon::parse($paymentOrder->date),
            "Pago OP #{$paymentOrder->number}",
            $paymentOrder,
            [
                [
                    'account_code' => '2.1.01',
                    'debit'        => round($paymentOrder->total, 2),
                    'credit'       => 0,
                    'description'  => "Cancelación deuda {$paymentOrder->supplier->name}",
                ],
                [
                    'account_code' => $bankAccountCode,
                    'debit'        => 0,
                    'credit'       => round($paymentOrder->total, 2),
                    'description'  => "OP #{$paymentOrder->number} - {$paymentOrder->supplier->name}",
                ],
            ]
        );
    }

    // ─────────────────────────────────────────────────────────────────
    // Helper: resolver cuenta bancaria/caja por medio de pago
    // ─────────────────────────────────────────────────────────────────

    /**
     * Determina el código de cuenta contable a usar para el banco/caja.
     *
     * - Si payment_method = 'efectivo' → '1.1.01' (Caja)
     * - Si hay bank_account_id y ese BankAccount tiene accounting_account_id → usar ese código
     * - Fallback → '1.1.02' (Bancos Cuenta Corriente)
     */
    private function resolveBankAccountCode(string $paymentMethod, ?int $bankAccountId): string
    {
        if ($paymentMethod === 'efectivo') {
            return '1.1.01';
        }

        if ($bankAccountId) {
            $bankAccount = \App\Models\BankAccount::with('accountingAccount')
                ->find($bankAccountId);

            if ($bankAccount?->accountingAccount) {
                return $bankAccount->accountingAccount->code;
            }
        }

        return '1.1.02'; // Bancos CC por defecto
    }
}
```

**Verificación:** El servicio se puede instanciar sin errores. Los métodos retornan null (no lanzan excepción) cuando una cuenta no existe.

## PASO 2 — Hook en `SalesInvoiceController::store()`

En `app/Http/Controllers/SalesInvoiceController.php`, después de crear exitosamente la factura,
llamar al servicio. Detectar si es NC (nota de crédito) o FC normal por `invoice_type`:

```php
// Al final de store(), antes del return redirect:
try {
    $entryService = new \App\Services\AccountingEntryService();

    // Tipos de NC en Argentina: NC A=3, NC B=8, NC C=13
    $creditNoteTypes = [3, 8, 13];
    if (in_array($invoice->invoice_type_id, $creditNoteTypes)) {
        $entryService->fromCreditNote($invoice->load('customer'));
    } else {
        $entryService->fromSalesInvoice($invoice->load('customer'));
    }
} catch (\Throwable $e) {
    \Log::error("Error generando asiento para FC venta #{$invoice->id}: " . $e->getMessage());
}
```

**Nota:** Verificar en el modelo `SalesInvoice` cuáles son los `invoice_type_id` de las NC.
Si el modelo usa strings en lugar de IDs para el tipo, ajustar la condición.

**Verificación:** Crear una factura de venta → aparece un asiento en `journal_entries` con `source_type=App\Models\SalesInvoice` y las líneas correctas.

## Notas / Consideraciones

- Los hooks en los controladores están en `try/catch` — un error en la generación del asiento
  **nunca rompe la operación principal**. El sistema sigue funcionando sin contabilidad si algo falla.
- El mapeo de cuentas usa los códigos del plan sembrado en v3.0.0. Si el contador modifica
  o elimina esas cuentas, los asientos futuros quedarán sin generar (log warning).
- Para FC de compra, la distinción insumo/servicio es por `supply_id`: con insumo → 5.1.01,
  sin insumo → 5.1.02. En una versión futura se puede agregar una tabla de mapeo configurable
  por proveedor o por categoría de insumo.
- El campo `total_iva` de `SalesInvoice` puede tener diferente nombre según cómo se implementó
  la facturación AFIP en v1.3.0 (puede llamarse `iva_amount`, `tax_amount`, etc.). Verificar
  en el modelo y ajustar la referencia en el servicio.
- Los asientos de sueldos/payroll y de stock/inventario se dejan para versiones futuras
  (mayor complejidad de configuración de cuentas).
