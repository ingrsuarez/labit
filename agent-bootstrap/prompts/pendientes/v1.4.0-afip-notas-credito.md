## Descripción

Agrega la posibilidad de emitir notas de crédito electrónicas asociadas a una factura de venta, autorizándolas contra AFIP igual que las facturas. Cierra el circuito de facturación electrónica — no se puede facturar sin poder corregir o anular con notas de crédito.

Las notas de crédito se crean desde la vista de una factura existente. Se autoriza automáticamente en AFIP si el punto de venta es electrónico.

## PASO 1 — Migración para notas de crédito

Crear las tablas `credit_notes` y `credit_note_items`:

```bash
php artisan make:migration create_credit_notes_table
php artisan make:migration create_credit_note_items_table
```

### Tabla `credit_notes`:
- `id`
- `credit_note_number` — string (número del comprobante)
- `voucher_type` — string (A, B, C)
- `point_of_sale_id` — FK a points_of_sale
- `customer_id` — FK a customers
- `sales_invoice_id` — FK a sales_invoices (factura asociada)
- `issue_date` — date
- `reason` — text (motivo de la NC)
- `subtotal` — decimal(12,2)
- `iva_21`, `iva_10_5`, `iva_27` — decimal(12,2) default 0
- `percepciones`, `otros_impuestos` — decimal(12,2) default 0
- `total` — decimal(12,2)
- `status` — string (pendiente, confirmada, anulada)
- `is_electronic` — boolean default false
- `cae` — string(20) nullable
- `cae_expiration` — date nullable
- `afip_voucher_number` — integer nullable
- `afip_result` — string(20) nullable
- `afip_response` — json nullable
- `created_by` — FK a users
- `timestamps`

### Tabla `credit_note_items`:
- `id`
- `credit_note_id` — FK a credit_notes
- `description` — string
- `quantity` — decimal(10,2)
- `unit_price` — decimal(12,2)
- `iva_rate` — decimal(5,2)
- `iva_amount` — decimal(12,2)
- `total` — decimal(12,2)
- `sort_order` — integer default 0
- `timestamps`

## PASO 3 — Extender AfipService para notas de crédito

Agregar método `createCreditNote(CreditNote $creditNote): array`

Similar a `createVoucher` pero con:
- Tipo comprobante: NC A=3, NC B=8, NC C=13
- Agregar `CbtesAsoc` (comprobantes asociados) — referencia a la factura original:
  - `Tipo` = tipo de la factura (1/6/11)
  - `PtoVta` = punto de venta AFIP
  - `Nro` = número de comprobante AFIP
  - `Cuit` = CUIT del emisor
  - `CbteFch` = fecha de la factura original (YYYYMMDD)

El resto del armado es igual que para facturas: IVA desglosado, importes, documento del cliente.

**Verificación:** Se puede autorizar una NC en homologación vinculada a una factura.

- Requiere `sales_invoice_id` en el request

- Precargar el formulario con datos de la factura (tipo, PdV, cliente, ítems)

### `store(Request $request)`
- Validar datos

### `show(CreditNote $creditNote)`
- Mostrar detalle con datos AFIP si es electrónica
- Link a la factura original

### `destroy(CreditNote $creditNote)`
- Solo permitir eliminar NC en estado pendiente y sin CAE

## Notas / Consideraciones

- Tipos AFIP para NC: NC A=3, NC B=8, NC C=13
- Las NC requieren `CbtesAsoc` obligatorio — la referencia a la factura original
- En homologación, verificar que AFIP acepta la NC con el comprobante asociado
- Una NC puede ser parcial (no tiene que cubrir el total de la factura)
- Múltiples NC pueden estar asociadas a una misma factura
- Las NC electrónicas no se pueden editar ni eliminar después de autorizadas
- El campo "Motivo" es informativo, no se envía a AFIP
