# v1.81.0 — Estados visuales en planilla de trabajo (pendiente / resultado / no pedida)

**Tipo:** minor — mejora visual funcional  
**Prioridad:** alta — problema operativo activo: técnico no puede distinguir "pendiente" de "no pedida" en la planilla  
**Dependencias:** ninguna

---

## Qué hace

En el PDF y preview de la planilla de trabajo, cada celda de la grilla paciente × determinación
pasa a tener **tres estados visuales distintos**:

| Estado | Condición | Render |
|--------|-----------|--------|
| **Pendiente** | Determinación pedida, sin resultado todavía | ✓ en verde teal |
| **Con resultado** | Determinación pedida, resultado cargado o validado | valor numérico/texto (comportamiento actual) |
| **No pedida** | Esa determinación no fue pedida para este protocolo | celda con fondo rayado diagonal (tacha) |

Hoy los estados "pendiente" y "no pedida" son indistinguibles (ambos renderizan `''`).
El fix es cambiar el valor centinela de "no pedida" de `''` a `null` en el controlador,
y actualizar la vista para ramificar los tres casos.

Aplica a **lab clínico** y **muestras** (ambos tipos de planilla).

---

## Backend — `WorksheetController`

Archivo: `app/Http/Controllers/WorksheetController.php`

### Método `buildClinicoData` (línea ~224)

Cambio puntual en el map:

```php
// ANTES
$results[$testId] = $at ? ($at->result ?? '') : '';

// DESPUÉS
$results[$testId] = $at ? ($at->result ?? '') : null;
//                                                ^^^^
// null  = no pedida para este protocolo
// ''    = pedida pero sin resultado todavía
// valor = pedida y con resultado
```

### Método `buildMuestrasData` (línea ~273)

Mismo cambio:

```php
// ANTES
$results[$testId] = $det ? ($det->result ?? '') : '';

// DESPUÉS
$results[$testId] = $det ? ($det->result ?? '') : null;
```

No se necesita ningún otro cambio en el controlador. La lógica de filtrado
(`includeWithout` / `includeWith`) no usa el valor de la celda individual — no hay regresión.

## Vista PDF — `resources/views/worksheets/pdf.blade.php`

### CSS — agregar estilos (dentro del `<style>` existente)

```css
/* Reemplazar td.empty-result con los tres estados nuevos */
td.cell-pending {
    color: #0d9488;
    font-weight: bold;
    font-size: 11pt;
}
td.cell-not-ordered {
    background: repeating-linear-gradient(
        45deg,
        #f5f5f5,
        #f5f5f5 3px,
        #e5e7eb 3px,
        #e5e7eb 6px
    );
    color: #aaa;
}
```

Eliminar la regla `.empty-result` que ya no se usa.

### Loop de celdas (líneas 120–124) — reemplazar el `<td>` del loop de resultados

```blade
{{-- ANTES --}}
@foreach($tests as $test)
<td class="{{ !$row['results'][$test->id] ? 'empty-result' : '' }}">
    {{ $row['results'][$test->id] ?: '' }}
</td>
@endforeach

{{-- DESPUÉS --}}
@foreach($tests as $test)
@php $val = $row['results'][$test->id] ?? null; @endphp
@if($val === null)
    <td class="cell-not-ordered"></td>
@elseif($val === '')
    <td class="cell-pending">✓</td>
@else
    <td>{{ $val }}</td>
@endif
@endforeach
```
