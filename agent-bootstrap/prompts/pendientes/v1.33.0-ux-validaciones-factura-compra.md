# v1.33.0 — UX y validaciones en formulario de factura de compra

**VERSIÓN:** v1.33.0  
**SLUG:** ux-validaciones-factura-compra  
**DEPENDENCIAS:** v1.32.x (buscador insumos, combobox)  
**ESTIMACIÓN:** 2h  
**PRIORIDAD:** alta

---

## Descripción

Mejoras de UX en el formulario de creación/edición de facturas de compra:

1. **Selector de empresa**: campo editable pre-cargado con la empresa activa del header.
   Permite cambiarla antes de guardar (dos empresas comparten stock pero tienen facturas separadas).

2. **Auto-padding de Punto de Venta y Nº Factura**: al salir del campo, completa con ceros a la
   izquierda hasta los dígitos requeridos (5 dígitos para PdV, 8 para Nº Factura).

3. **Validación de duplicado en tiempo real**: al completar proveedor + PdV + Nº Factura, consulta
   al backend si ya existe esa combinación. Muestra alerta inline sin bloquear el formulario.

4. **Fecha de Vencimiento automática**: al seleccionar la Fecha de Emisión, calcula y pre-rellena
   la Fecha de Vencimiento como Emisión + 30 días. El campo sigue siendo editable.

## PASO 3 — Auto-padding con ceros en Punto de Venta y Nº Factura

Usar `@blur` de Alpine.js para formatear los campos al salir:

```html
<!-- Punto de Venta (5 dígitos) -->
<input type="text"
       name="point_of_sale"
       id="point_of_sale"
       maxlength="5"
       x-model="form.pointOfSale"
       @blur="form.pointOfSale = (form.pointOfSale || '').replace(/\D/g, '').padStart(5, '0')"
       placeholder="00001"
       required
       class="w-full rounded-lg border-gray-300 text-sm focus:border-zinc-500 focus:ring-zinc-500">

<!-- Nº Factura (8 dígitos) -->
<input type="text"
       name="invoice_number"
       id="invoice_number"
       maxlength="8"
       x-model="form.invoiceNumber"
       @blur="form.invoiceNumber = (form.invoiceNumber || '').replace(/\D/g, '').padStart(8, '0')"
       @change="checkDuplicate()"
       placeholder="00000001"
       required
       class="w-full rounded-lg border-gray-300 text-sm focus:border-zinc-500 focus:ring-zinc-500">
```

Si el formulario Alpine usa `invoiceForm()` con propiedades `form.*`, adaptar los nombres
de propiedad según lo que ya exista en el `x-data` del formulario.

**Verificación:** Ingresar "9" en PdV → al salir queda "00009". Igual para Nº Factura.

## PASO 4 — Validación de duplicado en tiempo real

### 4.1 — Ruta y método en controlador

En `routes/web.php`:
```php
Route::get('purchase-invoices/check-duplicate', [PurchaseInvoiceController::class, 'checkDuplicate'])
    ->name('purchase-invoices.check-duplicate')
    ->middleware('auth');
```

En `PurchaseInvoiceController`:
```php
public function checkDuplicate(Request $request): \Illuminate\Http\JsonResponse
{
    $exists = PurchaseInvoice::where('supplier_id',    $request->supplier_id)
        ->where('point_of_sale',  $request->point_of_sale)
        ->where('invoice_number', $request->invoice_number)
        ->when($request->exclude_id, fn($q) => $q->where('id', '!=', $request->exclude_id))
        ->exists();

    return response()->json(['duplicate' => $exists]);
}
```

### 4.2 — Lógica Alpine.js para llamar al endpoint

Agregar al objeto Alpine del formulario (`invoiceForm()` o el que ya exista):

```javascript
duplicateWarning: false,
duplicateChecking: false,

async checkDuplicate() {
    const supplierId    = this.form?.supplier_id    || document.getElementById('supplier_id')?.value;
    const pointOfSale   = this.form?.pointOfSale    || document.getElementById('point_of_sale')?.value;
    const invoiceNumber = this.form?.invoiceNumber  || document.getElementById('invoice_number')?.value;
    const excludeId     = document.getElementById('exclude_id')?.value || ''; // en edit

    if (!supplierId || !pointOfSale || !invoiceNumber
        || pointOfSale === '00000' || invoiceNumber === '00000000') {
        this.duplicateWarning = false;
        return;
    }

    this.duplicateChecking = true;
    try {
        const url = `{{ route('purchase-invoices.check-duplicate') }}`
            + `?supplier_id=${supplierId}&point_of_sale=${pointOfSale}&invoice_number=${invoiceNumber}`
            + (excludeId ? `&exclude_id=${excludeId}` : '');
        const resp = await fetch(url);
        const data = await resp.json();
        this.duplicateWarning = data.duplicate;
    } catch (e) {
        this.duplicateWarning = false;
    } finally {
        this.duplicateChecking = false;
    }
},
```

Disparar la validación:
- Agregar `@change="checkDuplicate()"` al selector de proveedor
- Agregar `@blur="...; checkDuplicate()"` al campo Nº Factura (después del padStart)

## PASO 5 — Fecha de Vencimiento automática (+30 días)

```html
<!-- Fecha de Emisión -->
<input type="date"
       name="issue_date"
       id="issue_date"
       x-model="form.issueDate"
       @change="
           if (form.issueDate) {
               const d = new Date(form.issueDate + 'T00:00:00');
               d.setDate(d.getDate() + 30);
               form.dueDate = d.toISOString().split('T')[0];
           }
       "
       required
       class="w-full rounded-lg border-gray-300 text-sm focus:border-zinc-500 focus:ring-zinc-500">

<!-- Fecha de Vencimiento (editable, pre-llenada) -->
<input type="date"
       name="due_date"
       id="due_date"
       x-model="form.dueDate"
       required
       class="w-full rounded-lg border-gray-300 text-sm focus:border-zinc-500 focus:ring-zinc-500">
```

Si las propiedades Alpine no son `form.issueDate` / `form.dueDate`, adaptar a los nombres
exactos que ya usa el formulario (pueden ser `issueDate`, `issue_date`, etc.).

**Verificación:** Seleccionar fecha de emisión → la fecha de vencimiento se pre-rellena
con +30 días automáticamente. Editar la vencimiento manualmente sigue funcionando.
