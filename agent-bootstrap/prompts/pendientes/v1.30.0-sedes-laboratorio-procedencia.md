**VERSIÓN:** v1.30.0

**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 3h  
**PRIORIDAD:** alta

---

## Descripción

Crea un CRUD de sedes/sucursales del laboratorio y asocia cada protocolo a una sede de origen.
Permite filtrar protocolos por sede en los listados de los 3 módulos: lab clínico, muestras
(aguas/alimentos) y veterinario. Cada usuario puede configurar su sede por defecto, que se
preselecciona al crear protocolos y filtra los pendientes que ve. Esto resuelve la necesidad
de trazabilidad de procedencia cuando el laboratorio central recibe muestras de múltiples
ubicaciones geográficas.

### 1.2 — Modelo `LabBranch`

```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class LabBranch extends Model
{
    protected $fillable = [
        'name', 'address', 'city', 'province', 'zip_code',
        'phone', 'email', 'is_central', 'is_active',
    ];

    protected $casts = [
        'is_central' => 'boolean',
        'is_active'  => 'boolean',
    ];

    public function scopeActive($query)
    {
        return $query->where('is_active', true);
    }

    public function admissions()
    {
        return $this->hasMany(Admission::class, 'lab_branch_id');
    }

    public function samples()
    {
        return $this->hasMany(Sample::class, 'lab_branch_id');
    }

    public function vetAdmissions()
    {
        return $this->hasMany(VetAdmission::class, 'lab_branch_id');
    }
}
```

### 7.4 — Ruta para cambiar sede activa en sesión

```php
Route::post('switch-lab-branch', function (Request $request) {
    $branchId = $request->lab_branch_id;
    if ($branchId) {
        $branch = \App\Models\LabBranch::findOrFail($branchId);
        session(['active_lab_branch_id' => $branch->id, 'active_lab_branch_name' => $branch->name]);
    } else {
        session()->forget(['active_lab_branch_id', 'active_lab_branch_name']);
    }
    return back();
})->name('switch-lab-branch');
```

## Notas / Consideraciones

- `lab_branch_id` es nullable para compatibilidad con protocolos existentes. Los protocolos
  creados antes de esta versión no tendrán sede asignada.
- `default_lab_branch_id` es nullable. Usuarios sin sede asignada ven todos los protocolos
  y se les preselecciona la sede central al crear.
- La sede por defecto es una **preferencia**, no una restricción. El usuario puede crear
  protocolos para cualquier sede y puede cambiar el filtro para ver todas.
- A futuro se podría: (1) mostrar la sede en el PDF del informe, (2) notificar a la sede
  al validar, (3) restringir acceso por sede (que un usuario SOLO vea protocolos de su sede).
- El nombre `LabBranch` (en vez de `Branch`) evita colisiones si en el futuro hay sedes
  de otros módulos (ventas, RRHH).
- La desactivación es soft (is_active = false) para no perder la referencia en protocolos existentes.
