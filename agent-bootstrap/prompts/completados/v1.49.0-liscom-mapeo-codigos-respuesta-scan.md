## Contexto

LISCOM ya tiene en producción:

- ✅ **MLLP wrapper** completo (UTF-8 / UTF-16 LE, `hl7_engine/mllp.py`).
- ✅ **HL7Parser** rudimentario pero funcional (`hl7_engine/parser.py` — MSH, segmentos, get_field, get_patient_id).
- ✅ **HL7Builder** con MSH, ORM^O01, QRY^Q01 (`hl7_engine/builder.py`).
- ✅ **ACKGenerator** (`hl7_engine/ack.py`).
- ✅ **TCP server async** con auto-ACK por equipo (`sockets/server.py`, `sockets/manager.py`).
- ✅ **Modelos:** `Equipment`, `Connection`, `HL7Message` (`core/models.py`).
- ✅ **Cache local de protocolos** desde labit (`LocalProtocol`, `LocalDetermination`, `LabitConnection`, `SyncLog`) — v1.48.0.

Lo que **falta** para que el equipo pueda escanear un tubo y recibir la orden HL7:

1. **Mapeo de códigos por equipo** — cada equipo nombra los tests con códigos propietarios
   (ej: Roche Cobas C311 llama `GLUC2` a glucosa, otro equipo le dice `GLU`). Hoy no existe esa tabla.
2. **Parser de barcode con material** — entender `C-2026-001234^EDTA` y separar protocolo / material.
3. **Builders de respuesta** — DSR^Q03 (HL7 v2.3) para responder QRY^Q11, y ORL^O22 (HL7 v2.5+)
   para responder OUL^R22.
4. **Detector y handler en `ConnectionManager._handle_message`** — si el mensaje entrante es
   QRY^Q11 o OUL^R22, **no hacer ACK simple**; en su lugar, lookup contra `LocalProtocol` y armar
   respuesta con orden completa filtrada por material.
5. **UI manual** para gestionar el mapeo (CRUD básico Django admin + páginas custom para QoL).

**Decisiones PM (2026-04-18):**

- **Tipos HL7 soportados:** **QRY^Q11 → DSR^Q03** y **OUL^R22 → ORL^O22**. Sin unsolicited
  (host empuja ORM^O01 al equipo). Si en producción aparece un equipo que solo soporta unsolicited,
  se planifica como v1.49.5.
- **Mapeo de códigos:** **solo UI manual** (alta uno por uno). Sin import CSV ni modo learning
  por ahora. Si crece, se replanifica.
- **Filtrado por material:** activado por default. Si el barcode no trae material (formato viejo,
  sin `^`), se devuelven TODAS las determinaciones del protocolo (modo "compat").
- **Pre-validación del mapeo:** si una determinación del protocolo no tiene mapeo equipo-código
  configurado para el equipo que pregunta, **se omite de la respuesta** y se loguea como
  `unmapped_test` (visible en una bandeja para acción del operador).

```python
"""Parser para el contenido del barcode escaneado desde un equipo."""
from dataclasses import dataclass


@dataclass
class ParsedBarcode:
    """Resultado de parsear un barcode escaneado."""
    raw: str
    protocol_number: str
    material_abbreviation: str = ''  # vacío si formato viejo
    is_extended_format: bool = False  # True si traía '^material'

    @property
    def has_material_filter(self) -> bool:
        return bool(self.material_abbreviation)


def parse_barcode(text: str) -> ParsedBarcode:
    """
    Parsea el contenido de un barcode escaneado.

    Formatos soportados:
        - Extendido (v1.48.5+): "C-2026-001234^EDTA"
        - Viejo (compat):       "C-2026-001234"

    Reglas:
        - Trim de whitespace.
        - Separator '^' (single character, primer ocurrencia).
        - Si no hay '^', se asume formato viejo y material vacío.
        - Si hay '^' pero el material está vacío después de trim, se trata como viejo.

    Args:
        text: contenido crudo del barcode (puede venir con espacios, CR, etc.)

    Returns:
        ParsedBarcode con protocol_number y material_abbreviation.
    """
    cleaned = (text or '').strip().replace('\r', '').replace('\n', '')

    if '^' not in cleaned:
        return ParsedBarcode(
            raw=text,
            protocol_number=cleaned,
            material_abbreviation='',
            is_extended_format=False,
        )

    parts = cleaned.split('^', 1)
    protocol = parts[0].strip()
    material = parts[1].strip() if len(parts) > 1 else ''

    return ParsedBarcode(
        raw=text,
        protocol_number=protocol,
        material_abbreviation=material,
        is_extended_format=bool(material),
    )
```

## Detección de queries en el parser HL7

```python
# === Detección de mensajes de query (host query) ===

QUERY_MESSAGE_TYPES = {
    'QRY^Q11',   # Query for results / order (HL7 v2.3)
    'QRY^Q01',   # Query (genérico v2.3)
    'OUL^R22',   # Unsolicited Lab Observation (HL7 v2.5+, también usado para query de orden)
    'OUL^R21',   # Variante v2.4
    'OUL^R23',   # Variante v2.5
    'OML^O21',   # Lab Order Message (también puede traer query)
}

def is_query_message(self, parsed: 'ParsedMessage') -> bool:
    """¿El mensaje es una query de orden desde el equipo (host query)?"""
    return parsed.message_type in self.QUERY_MESSAGE_TYPES

def extract_query_barcode(self, message: str) -> str:
    """
    Extrae el contenido del barcode escaneado desde un mensaje de query.

    Reglas según el tipo de mensaje:

    QRY^Q11 / QRY^Q01:
        El barcode (specimen ID o accession number) suele venir en QRD-8
        (Who Subject Filter) o en SPM-2.

    OUL^R22:
        Viene en SPM-2 (Specimen ID) o en SAC-3 (Container ID).

    OML^O21:
        Viene en SPM-2 o en ORC-3 (Filler Order Number).

    Estrategia: probar en orden y devolver el primer hit no vacío.
    """
    parsed = self.parse(message)

    candidates = []

    if parsed.message_type.startswith('QRY'):
        qrd = self.get_segment(message, 'QRD')
        if qrd:
            candidates.append(self.get_field(qrd, 8))

    spm = self.get_segment(message, 'SPM')
    if spm:
        candidates.append(self.get_field(spm, 2))

    sac = self.get_segment(message, 'SAC')
    if sac:
        candidates.append(self.get_field(sac, 3))

    orc = self.get_segment(message, 'ORC')
    if orc:
        candidates.append(self.get_field(orc, 3))

    obr = self.get_segment(message, 'OBR')
    if obr:
        candidates.append(self.get_field(obr, 3))
        candidates.append(self.get_field(obr, 2))

    for c in candidates:
        if c:
            cleaned = c.strip()
            if cleaned:
                return cleaned

    return ''
```

> **Nota implementación:** algunos equipos ponen el barcode en componentes anidados con `^`
> dentro del campo (ej: `OBR-3 = "C-2026-001234^EDTA^^M"`). El parser de barcode (`barcode_parser.py`)
> ya maneja el split por `^` y se queda con los dos primeros tokens. No hace falta lógica especial acá.

## Builders de respuesta HL7

Agregar al final de `hl7_engine/builder.py` (clase `HL7Builder`):

```python
def build_dsr_q03(
    self,
    original_query_control_id: str,
    protocol_number: str,
    patient_id: str,
    patient_name: str,
    determinations: list[dict],
    sending_app: str = 'LISCOM',
    receiving_app: str = '',
) -> str:
    """
    Construye una respuesta DSR^Q03 a una QRY^Q11 (HL7 v2.3 query model).

    Estructura:
        MSH | MSA | QRD | DSP × N

    Donde DSP es Display Data — devuelve la orden completa para el equipo.

    Args:
        original_query_control_id: MSH-10 de la query original (correlación).
        protocol_number: número de protocolo de labit (ej "C-2026-001234").
        patient_id: documento o ID interno del paciente.
        patient_name: apellido^nombre.
        determinations: lista de dicts con keys:
            - equipment_code (str)  - código que el equipo entiende
            - test_name (str)
            - material_abbreviation (str, opcional)
            - priority (str, opcional, default 'R' = routine)
        sending_app: nombre del app emisor (default LISCOM).
        receiving_app: nombre del equipo receptor.

    Returns:
        str: Mensaje DSR^Q03 completo, sin MLLP wrapping (eso lo hace el manager).
    """
    msh = self.build_msh(
        sending_app=sending_app,
        receiving_app=receiving_app,
        message_type='DSR^Q03',
    )

    msa = f'MSA|AA|{original_query_control_id}'

    timestamp = datetime.now().strftime('%Y%m%d%H%M%S')
    qrd = f'QRD|{timestamp}|R|I|{original_query_control_id}|||RD|{protocol_number}|OTH|'

    segments = [msh, msa, qrd]

    safe_patient = (patient_name or '').replace('|', ' ').replace('~', ' ')
    safe_pid = (patient_id or '').replace('|', ' ')
    pid_line = f'PID|1||{safe_pid}||{safe_patient}'
    segments.append(pid_line)

    for idx, det in enumerate(determinations, start=1):
        code = det.get('equipment_code', '')
        name = det.get('test_name', '').replace('|', ' ').replace('^', ' ')
        priority = det.get('priority', 'R')

        dsp_test = f'DSP|{idx}|1|{code}^{name}'
        dsp_material = f'DSP|{idx}|2|{det.get("material_abbreviation", "")}'
        dsp_priority = f'DSP|{idx}|3|{priority}'
        segments.extend([dsp_test, dsp_material, dsp_priority])

    dsc = f'DSC|{original_query_control_id}'
    segments.append(dsc)

    return self.build_message(segments)


def build_orl_o22(
    self,
    original_query_control_id: str,
    protocol_number: str,
    patient_id: str,
    patient_name: str,
    determinations: list[dict],
    sending_app: str = 'LISCOM',
    receiving_app: str = '',
    accept_code: str = 'AA',
) -> str:
    """
    Construye una respuesta ORL^O22 a una OUL^R22 (HL7 v2.5+ lab automation).

    Estructura simplificada:
        MSH | MSA | PID | { ORC | OBR } × N

    Args:
        accept_code: 'AA' (accept), 'AE' (error), 'AR' (reject).
        Demás args: igual que build_dsr_q03.

    Returns:
        str: Mensaje ORL^O22 completo.
    """
    msh = self.build_msh(
        sending_app=sending_app,
        receiving_app=receiving_app,
        message_type='ORL^O22',
    )
    msa = f'MSA|{accept_code}|{original_query_control_id}'

    safe_patient = (patient_name or '').replace('|', ' ').replace('~', ' ')
    safe_pid = (patient_id or '').replace('|', ' ')
    pid = f'PID|1||{safe_pid}||{safe_patient}'

    segments = [msh, msa, pid]
    timestamp = datetime.now().strftime('%Y%m%d%H%M%S')

    for idx, det in enumerate(determinations, start=1):
        code = det.get('equipment_code', '')
        name = det.get('test_name', '').replace('|', ' ').replace('^', ' ')
        material = det.get('material_abbreviation', '')

        orc = (
            f'ORC|NW|{protocol_number}-{idx}|{protocol_number}-{idx}|||'
            f'||||{timestamp}||||||{material}'
        )
        obr = (
            f'OBR|{idx}|{protocol_number}-{idx}|{protocol_number}-{idx}|'
            f'{code}^{name}|||{timestamp}||||||||{material}'
        )
        segments.extend([orc, obr])

    return self.build_message(segments)
```

> **Nota:** Los formatos exactos de DSR^Q03 y ORL^O22 varían entre fabricantes. Estos builders
> generan estructuras estándar que cubren ~80% de los equipos. Cuando aparezca un equipo concreto
> con problemas, se ajusta puntualmente (no rompe lo demás porque cada equipo tiene su row en
> `Equipment` con su propio `model`/`manufacturer` que se puede ramificar).

## Servicio: `OrderResponseService`

Crear `core/services/order_builder.py`:

```python
"""Servicio que construye la respuesta HL7 para una query de orden."""
import logging
from dataclasses import dataclass, field
from typing import Optional

from core.models import Equipment, EquipmentTestMapping, LocalProtocol, LocalDetermination
from core.services.barcode_parser import ParsedBarcode, parse_barcode
from hl7_engine.builder import HL7Builder

logger = logging.getLogger('order_builder')


@dataclass
class OrderResponse:
    """Resultado de construir una respuesta de orden."""
    success: bool
    hl7_message: str = ''
    error_code: str = ''  # 'NOT_FOUND', 'NO_MAPPINGS', 'NO_DETERMINATIONS', 'INVALID_BARCODE'
    error_message: str = ''
    determinations_included: int = 0
    determinations_unmapped: list = field(default_factory=list)
    parsed_barcode: Optional[ParsedBarcode] = None
    protocol: Optional[LocalProtocol] = None


class OrderResponseService:
    """Construye la respuesta HL7 a una query de orden de un equipo."""

    def __init__(self, builder: Optional[HL7Builder] = None):
        self.builder = builder or HL7Builder()

    def build_response(
        self,
        equipment: Equipment,
        barcode_text: str,
        query_message_control_id: str,
        query_type: str,
    ) -> OrderResponse:
        """
        Construye la respuesta para una query.

        Args:
            equipment: el equipo que preguntó.
            barcode_text: contenido crudo del barcode escaneado.
            query_message_control_id: MSH-10 de la query (para correlación).
            query_type: tipo de mensaje original ('QRY^Q11', 'OUL^R22', etc.)

        Returns:
            OrderResponse con el mensaje HL7 listo para enviar (sin wrapper MLLP).
        """
        parsed_barcode = parse_barcode(barcode_text)

        if not parsed_barcode.protocol_number:
            return OrderResponse(
                success=False,
                error_code='INVALID_BARCODE',
                error_message=f'Barcode vacío o inválido: "{barcode_text!r}"',
                parsed_barcode=parsed_barcode,
            )

        try:
            protocol = LocalProtocol.objects.prefetch_related('determinations').get(
                protocol_number=parsed_barcode.protocol_number,
            )
        except LocalProtocol.DoesNotExist:
            logger.warning(
                f'Protocolo no encontrado en cache local: {parsed_barcode.protocol_number}. '
                f'¿Hace falta forzar sync?'
            )
            return OrderResponse(
                success=False,
                error_code='NOT_FOUND',
                error_message=f'Protocolo {parsed_barcode.protocol_number} no encontrado en cache local',
                parsed_barcode=parsed_barcode,
            )

        all_dets = list(protocol.determinations.all())

        if parsed_barcode.has_material_filter:
            filtered_dets = [
                d for d in all_dets
                if (d.material_abbreviation or '').upper() == parsed_barcode.material_abbreviation.upper()
            ]
        else:
            filtered_dets = all_dets

        if not filtered_dets:
            return OrderResponse(
                success=False,
                error_code='NO_DETERMINATIONS',
                error_message=(
                    f'Protocolo {protocol.protocol_number} no tiene determinaciones '
                    f'para material {parsed_barcode.material_abbreviation or "(cualquiera)"}'
                ),
                parsed_barcode=parsed_barcode,
                protocol=protocol,
            )

        mappings_by_test = {
            m.labit_test_id: m
            for m in EquipmentTestMapping.objects.filter(
                equipment=equipment,
                active=True,
                labit_test_id__in=[d.test_id for d in filtered_dets if d.test_id],
            )
        }

        det_payload = []
        unmapped = []

        for det in filtered_dets:
            mapping = mappings_by_test.get(det.test_id)

            if not mapping:
                unmapped.append({
                    'test_id': det.test_id,
                    'test_name': det.test_name,
                    'test_code': det.test_code,
                })
                continue

            if mapping.material_filter and (det.material_abbreviation or '').upper() != mapping.material_filter.upper():
                continue

            det_payload.append({
                'equipment_code': mapping.equipment_code,
                'test_name': det.test_name or mapping.labit_test_name,
                'material_abbreviation': det.material_abbreviation or '',
                'priority': 'R',
            })

        if not det_payload:
            return OrderResponse(
                success=False,
                error_code='NO_MAPPINGS',
                error_message=(
                    f'Ninguna determinación de {protocol.protocol_number} tiene mapeo activo '
                    f'para equipo "{equipment.name}". Pendientes: {len(unmapped)}.'
                ),
                determinations_unmapped=unmapped,
                parsed_barcode=parsed_barcode,
                protocol=protocol,
            )

        patient = protocol.patient_data or {}
        patient_id = patient.get('document') or patient.get('id') or ''
        patient_name = self._format_patient_name(patient)

        if query_type.startswith('QRY'):
            hl7 = self.builder.build_dsr_q03(
                original_query_control_id=query_message_control_id,
                protocol_number=protocol.protocol_number,
                patient_id=str(patient_id),
                patient_name=patient_name,
                determinations=det_payload,
                receiving_app=equipment.name,
            )
        else:
            hl7 = self.builder.build_orl_o22(
                original_query_control_id=query_message_control_id,
                protocol_number=protocol.protocol_number,
                patient_id=str(patient_id),
                patient_name=patient_name,
                determinations=det_payload,
                receiving_app=equipment.name,
            )

        logger.info(
            f'Respuesta {query_type[:3]} para {equipment.name}: '
            f'protocolo={protocol.protocol_number}, '
            f'material={parsed_barcode.material_abbreviation or "any"}, '
            f'incluidas={len(det_payload)}, no_mapeadas={len(unmapped)}'
        )

        return OrderResponse(
            success=True,
            hl7_message=hl7,
            determinations_included=len(det_payload),
            determinations_unmapped=unmapped,
            parsed_barcode=parsed_barcode,
            protocol=protocol,
        )

    @staticmethod
    def _format_patient_name(patient_data: dict) -> str:
        """Convierte patient_data a "APELLIDO^NOMBRE" (formato HL7 PID-5)."""
        if not patient_data:
            return ''

        full = patient_data.get('name') or patient_data.get('full_name') or ''
        last = patient_data.get('last_name') or patient_data.get('surname') or ''
        first = patient_data.get('first_name') or patient_data.get('name') or ''

        if last and first and last != first:
            return f'{last.strip()}^{first.strip()}'
        if full:
            return full.strip().replace(' ', '^')
        return ''
```

```python
        f'Query recibida de {equipment.name}: tipo={parsed.message_type}, '
        f'barcode={barcode!r}, control_id={parsed.message_control_id}'

            original_message=message,
            error_code='AE',
            error_message='No se pudo extraer barcode/specimen ID',
        )

        messages_to_save.append((nak, self.parser.parse(nak), 'OUT'))
        continue

        service = OrderResponseService()
        result = await sync_to_async(service.build_response, thread_sensitive=False)(

            barcode_text=barcode,

        )
    except Exception as exc:
        logger.exception(f'Error armando respuesta de orden: {exc}')
        nak = self.ack_generator.generate_nak(
            original_message=message,
            error_code='AE',
            error_message=f'Internal error: {exc}',
        )
        all_acks += mllp.encode(nak, encoding=detected_encoding)
        messages_to_save.append((nak, self.parser.parse(nak), 'OUT'))
        continue

    if result.success:
        encoded = mllp.encode(result.hl7_message, encoding=detected_encoding)
        all_acks += encoded
        logger.info(
            f'Respuesta de orden enviada: {result.determinations_included} dets, '
            f'{len(result.determinations_unmapped)} sin mapear'
        )
        messages_to_save.append((result.hl7_message, self.parser.parse(result.hl7_message), 'OUT'))
    else:
        nak = self.ack_generator.generate_nak(
            original_message=message,
            error_code='AE',
            error_message=f'{result.error_code}: {result.error_message}',
        )
        all_acks += mllp.encode(nak, encoding=detected_encoding)
        messages_to_save.append((nak, self.parser.parse(nak), 'OUT'))

    continue  # ya generamos respuesta, no caer al ACK auto
```

```python
from django.contrib.auth.decorators import login_required
from django.shortcuts import render, get_object_or_404, redirect
from django.contrib import messages
from .models import Equipment, EquipmentTestMapping, LocalProtocol


@login_required
def equipment_mappings(request, equipment_id):
    """Lista de mapeos de un equipo + form inline para alta rápida."""
    equipment = get_object_or_404(Equipment, pk=equipment_id)

    if request.method == 'POST':
        EquipmentTestMapping.objects.create(
            equipment=equipment,
            equipment_code=request.POST['equipment_code'].strip(),
            labit_test_id=int(request.POST['labit_test_id']),
            labit_test_name=request.POST.get('labit_test_name', '').strip(),
            labit_test_code=request.POST.get('labit_test_code', '').strip(),
            material_filter=request.POST.get('material_filter', '').strip(),
            notes=request.POST.get('notes', '').strip(),
            active=True,
        )
        messages.success(request, 'Mapeo creado.')
        return redirect('equipment_mappings', equipment_id=equipment.id)

    mappings = equipment.test_mappings.order_by('equipment_code')

    available_tests = (
        LocalProtocol.objects.values('determinations__test_id', 'determinations__test_name', 'determinations__test_code')
        .distinct()
        .filter(determinations__test_id__isnull=False)
        .order_by('determinations__test_name')[:500]
    )

    return render(request, 'core/equipment_mappings.html', {
        'equipment': equipment,
        'mappings': mappings,
        'available_tests': available_tests,
    })


@login_required
def toggle_mapping(request, mapping_id):
    """Activa/desactiva un mapeo (botón rápido)."""
    m = get_object_or_404(EquipmentTestMapping, pk=mapping_id)
    m.active = not m.active
    m.save(update_fields=['active', 'updated_at'])
    messages.info(request, f'Mapeo {m.equipment_code} → {"activado" if m.active else "desactivado"}.')
    return redirect('equipment_mappings', equipment_id=m.equipment_id)


@login_required
def delete_mapping(request, mapping_id):
    m = get_object_or_404(EquipmentTestMapping, pk=mapping_id)
    eq_id = m.equipment_id
    m.delete()
    messages.warning(request, 'Mapeo eliminado.')
    return redirect('equipment_mappings', equipment_id=eq_id)
```

```python
def test_extended_format():
    r = parse_barcode('C-2026-001234^EDTA')
    assert r.protocol_number == 'C-2026-001234'
    assert r.material_abbreviation == 'EDTA'
    assert r.is_extended_format is True
    assert r.has_material_filter is True


def test_legacy_format():
    r = parse_barcode('C-2026-001234')
    assert r.protocol_number == 'C-2026-001234'
    assert r.material_abbreviation == ''
    assert r.is_extended_format is False
    assert r.has_material_filter is False


def test_strips_whitespace_and_crlf():
    r = parse_barcode('  C-2026-001234^EDTA\r\n  ')
    assert r.protocol_number == 'C-2026-001234'
    assert r.material_abbreviation == 'EDTA'


def test_separator_with_empty_material_treated_as_legacy():
    r = parse_barcode('C-2026-001234^')
    assert r.protocol_number == 'C-2026-001234'
    assert r.material_abbreviation == ''
    assert r.is_extended_format is False
```

## Tensiones (anotar en handoff)

1. **Estructura exacta de DSR^Q03 / ORL^O22 varía por fabricante.** Los builders cubren el caso
   estándar HL7. Cuando aparezca un equipo concreto que no acepte estos formatos, hay que
   ramificar por `equipment.manufacturer` o `equipment.model` en `OrderResponseService`.
   No es bloqueante para v1.49.0 (sale después en producción).
2. **Cache local desactualizada.** Si el operador escanea un protocolo que se creó en labit DESPUÉS
   del último sync, devuelve `NOT_FOUND`. Mitigación: el equipo recibe NAK con mensaje claro y el
   operador puede forzar sync desde la UI (comando `sync_protocols --on-demand` que ya existe en
   v1.48.0). Posible follow-up v1.49.5: trigger automático de sync `on_demand` cuando hay NOT_FOUND.
3. **Equipos con formatos no soportados.** Si llega un mensaje de tipo no listado en
   `QUERY_MESSAGE_TYPES` (ej: `OML^O33` raro), cae al ACK normal. No rompe pero el equipo no
   recibe orden. Si pasa, se agrega el tipo al set y eventualmente el builder específico.
4. **Performance del lookup.** Cada query hace 2 queries SQL (LocalProtocol + EquipmentTestMapping).
   Con índices ya definidos debería ser <10ms en SQLite local. Si crece, considerar cache LRU.
5. **Patient PII en respuestas HL7.** Hoy ponemos solo apellido^nombre. Si un equipo necesita más
   (DOB, sexo, etc.) hay que extender `_format_patient_name` o sumar más campos de `patient_data`.
   Tener en cuenta que liscom ya recibió `patient_data` con o sin DNI según `api_clients.patient_data_level`
   de v1.46.0/v1.47.0.
