# v3.1.0 — Conciliación Bancaria: Cuentas e Importación

**VERSIÓN:** v3.1.0
**SLUG:** conciliacion-bancaria-import
**DEPENDENCIAS:** v3.0.0 (infraestructura contable base)
**ESTIMACIÓN:** medio día
**PRIORIDAD:** alta

---

## Descripción

Crear la infraestructura de cuentas bancarias por empresa, permitir la importación de
extractos bancarios en formato XLS (BBVA Francés / Banco Francés) y mostrar los
movimientos importados en una tabla filtrable con categorización automática.

Cada empresa puede tener múltiples cuentas bancarias. Los extractos se importan
como "statements" y cada línea se guarda como "bank movement" con estado de
conciliación (pendiente por defecto).

El formato XLS del banco tiene 2 hojas: "Movimientos del Día" y "Movimientos Históricos".
Ambas comparten estructura similar pero la histórica tiene un campo "Detalle" adicional.

**No se implementa el matcheo/conciliación contra registros internos en esta versión** —
solo se importan y categorizan los movimientos.

### Estructura del XLS de referencia (BBVA Francés)

**Hoja 1: "Movimientos del Día"** (9 columnas)
```
Filas 0-4: Header
  Empresa: NOMBRE (CUIT)
  Cuenta: XXX-XXXXXX/X(CC $)
  Sucursal: XXX - NOMBRE
  Saldo: XX.XXX.XXX,XX
  Movimientos de: DD-MM-YYYY

Fila 6: Encabezados de columna
  Fecha | Fecha Valor | Concepto | Codigo | Nro de cheque | Oficina | Crédito | Débito | Saldo Parcial

Filas 7+: Datos
  31-03-2026 | 31-03-2026 | COMISION X TRANSFERENCIA | 526 | | 733 - | | -100.0 | 20446952.95
```

**Hoja 2: "Movimientos Históricos"** (10 columnas)
```
Mismas filas 0-4 de header.
Fila 5: "Movimientos de: Ultimos 60 Días."

Fila 6: Encabezados
  Fecha | Fecha Valor | Concepto | Codigo | Número Documento | Oficina | Crédito | Débito | Detalle | (col vacía con saldo disponible en primera fila)

Filas 7+: Datos con campo Detalle extra (CTE, CTA.DESTINO, etc.)
```

**Formatos numéricos del XLS:**
- Crédito: valor positivo (ej: `8946460.95`) o vacío
- Débito: valor negativo (ej: `-350000.0`) o vacío
- Saldo: valor numérico sin formato (ej: `20446952.95`)
- Fechas: string `DD-MM-YYYY`

```php
class BankStatementImporter
{
    public function import(BankAccount $bankAccount, UploadedFile $file, User $user): BankStatement
    {
        // 1. Leer el XLS con PhpSpreadsheet (ya disponible via maatwebsite/excel)
        //    usar \PhpOffice\PhpSpreadsheet\IOFactory::load($file->getPathname())

        // 2. Detectar la hoja con más datos (Históricos > Día)
        //    Preferir "Movimientos Históricos" si existe, fallback a primera hoja

        // 3. Parsear header (filas 0-4):
        //    - Fila 0: Empresa + CUIT
        //    - Fila 1: Cuenta
        //    - Fila 2: Sucursal
        //    - Fila 3: Saldo (closing_balance)
        //    - Fila 4: Período

        // 4. Detectar fila de encabezados de columna (buscar "Fecha" en col 0)

        // 5. Parsear cada fila de movimientos:
        //    - Fecha: string "DD-MM-YYYY" → Carbon::createFromFormat('d-m-Y', $val)
        //    - Crédito: float o vacío → abs(valor) si > 0
        //    - Débito: float negativo o vacío → abs(valor) para guardar positivo
        //    - Saldo: float
        //    - Concepto, Codigo, Documento, Oficina: strings
        //    - Detalle: solo en hoja histórica (columna 8 o 9)

        // 6. Calcular período (min/max de fechas), totales de créditos y débitos

        // 7. Verificar duplicados: si ya existe un statement para esta cuenta
        //    con el mismo período y filename → lanzar excepción o devolver el existente

        // 8. Crear BankStatement en transacción DB

        // 9. Crear BankMovements en bulk, aplicando categorize() a cada uno

        // 10. Actualizar totales del statement

        return $statement;
    }

    // Método privado para categorizar por código/concepto
    private function categorizeMovement(string $concept, ?string $bankCode): ?string
    {
        // Mapeo por código primero (más preciso)
        $codeMap = [
            '136' => 'transferencia', '135' => 'transferencia',
            '129' => 'transferencia', '262' => 'transferencia',
            '265' => 'transferencia', '319' => 'transferencia',
            '986' => 'cobro', '983' => 'cobro',
            '005' => 'cheque', '082' => 'cheque',
            '381' => 'impuesto', '589' => 'impuesto',
            '573' => 'impuesto', '609' => 'impuesto',
            '526' => 'comision', '241' => 'comision',
            '575' => 'comision', '392' => 'comision',
            '236' => 'iva_comision',
            '522' => 'pago_tarjeta', '137' => 'pago_servicio',
            '081' => 'debin',
            '761' => 'debito_automatico', '879' => 'debito_automatico',
            '247' => 'debito_automatico', '493' => 'debito_automatico',
            '744' => 'pago_tarjeta_credito',
            '016' => 'movimiento_interno',
        ];

        if ($bankCode && isset($codeMap[trim($bankCode)])) {
            return $codeMap[trim($bankCode)];
        }

        // Fallback: buscar por concepto (para códigos desconocidos)
        $conceptPatterns = [
            '/TRANSF/i' => 'transferencia',
            '/CHEQUE|CH\/CLEAR|DEPOS\.CHQ/i' => 'cheque',
            '/AFIP|IMPUESTO|IB NEUQUEN|LEY NRO/i' => 'impuesto',
            '/COMISION|COM EMISION/i' => 'comision',
            '/IVA TASA/i' => 'iva_comision',
            '/PAGO CON VIS|PAGO SERVICI/i' => 'pago_tarjeta',
            '/DEBIN/i' => 'debin',
            '/OG-DEBITO|OG-DEB/i' => 'debito_automatico',
            '/ACRED SUELDO|DNET CREDITO/i' => 'cobro',
        ];

        foreach ($conceptPatterns as $pattern => $category) {
            if (preg_match($pattern, $concept)) {
                return $category;
            }
        }

        return null; // sin categorizar
    }
}
```
