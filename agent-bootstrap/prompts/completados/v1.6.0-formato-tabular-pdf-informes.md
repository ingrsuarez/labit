## Descripción

Reformatear los PDFs de informes de protocolo (laboratorio clínico y muestras de
aguas/alimentos) para mostrar las determinaciones en formato tabular compacto:
una fila por test con columnas de Análisis, Resultado, Unidad y Valores de referencia.

Actualmente cada determinación ocupa 2-3 líneas verticales (nombre en bold, 
"Resultado: X" en línea aparte, "Método: Y" en otra línea). El formato deseado
es una tabla con columnas alineadas donde cada determinación es una fila.

### Formato deseado

### Reglas de renderizado

1. **Encabezados de sección** (HEMATOLOGÍA, BIOQUÍMICA, MICROBIOLÓGICO, FISICOQUÍMICO):
   - Fondo gris, texto bold mayúscula, ancho completo
   - Separador visual entre secciones

2. **Tests padre** (Hemograma, Fórmula Leucocitaria, Hepatograma, Orina Completa):
   - Nombre en **bold**, sin resultado/unidad/referencia
   - Ocupan una fila completa como sub-encabezado
   - Si tienen método, se muestra debajo en texto italic/gris

3. **Tests hijo** (Glóbulos Rojos, VCM, Bilirrubina total, etc.):
   - Nombre indentado (padding-left)
   - Resultado, unidad y valores de referencia en columnas
   - Si tienen método, se muestra debajo en texto italic/gris más pequeño

4. **Tests standalone** (Glucemia, Urea, Eritrosedimentación):
   - Fila normal sin indentación
   - Resultado, unidad y valores de referencia en columnas
   - Si tienen método, se muestra debajo en texto italic/gris

5. **Campo Método**: se mantiene como subtexto debajo de la determinación correspondiente,
   en tipografía más pequeña e italic/gris. No ocupa columna propia.

6. **Valores de referencia**: pueden ser simples ("< 500") o con rangos ("0.70 - 1.0 gr/l").
   Usar el formato tal como viene del modelo.

## PASO 2 — Reestructurar el body del PDF

Reemplazar el bloque de determinaciones actual por una estructura tabular.

### Lógica de agrupación:

1. Agrupar determinaciones por la sección/categoría del test (si existe el campo)
2. Dentro de cada sección, identificar:
   - **Tests padre:** tienen `childTests->count() > 0` o `parentTests->count() == 0 && childTests->count() > 0`
   - **Tests hijo:** tienen `parentTests->count() > 0`
   - **Tests standalone:** no son padre ni hijo
3. Ordenar: padres primero con sus hijos debajo, luego standalones

### Estructura HTML para mPDF:

```html
<!-- Por cada sección -->
<table width="100%" cellpadding="4" cellspacing="0" style="border-collapse: collapse; margin-bottom: 10px;">
    <!-- Header de sección -->
    <tr style="background-color: #f0f0f0;">
        <td colspan="4" style="font-weight: bold; font-size: 11pt; text-transform: uppercase; border-bottom: 2px solid #333; padding: 6px;">
            NOMBRE DE SECCIÓN
            <span style="font-size: 8pt; font-weight: normal; font-style: italic;">Valores de referencia según ...</span>
        </td>
    </tr>
    <!-- Header de columnas -->
    <tr style="border-bottom: 1px solid #ccc;">
        <td width="40%" style="font-weight: bold; font-size: 9pt; padding: 4px;">Análisis</td>
        <td width="18%" style="font-weight: bold; font-size: 9pt; text-align: center; padding: 4px;">Resultado</td>
        <td width="12%" style="font-weight: bold; font-size: 9pt; text-align: center; padding: 4px;">Unidad</td>
        <td width="30%" style="font-weight: bold; font-size: 9pt; text-align: right; padding: 4px;">Valores de ref.</td>
    </tr>

    <!-- Test padre -->
    <tr>
        <td colspan="4" style="font-weight: bold; font-size: 10pt; padding: 6px 4px 2px 4px; border-bottom: none;">
            Hemograma Completo
        </td>
    </tr>
    <!-- Método del padre (si tiene) -->
    <tr>
        <td colspan="4" style="font-size: 8pt; color: #666; font-style: italic; padding: 0 4px 4px 12px;">
            Mindray 5390
        </td>
    </tr>
    <!-- Test hijo -->
    <tr style="border-bottom: 1px solid #eee;">
        <td style="font-size: 9pt; padding: 3px 4px 3px 20px;">Glóbulos Rojos</td>
        <td style="font-size: 9pt; text-align: center; font-weight: bold;">5640000</td>
        <td style="font-size: 9pt; text-align: center;">/mm3</td>
        <td style="font-size: 9pt; text-align: right;">4500000 - 5500000 /mm3</td>
    </tr>

    <!-- Test standalone -->
    <tr style="border-bottom: 1px solid #eee;">
        <td style="font-size: 9pt; padding: 3px 4px;">Eritrosedimentación</td>
        <td style="font-size: 9pt; text-align: center; font-weight: bold;">10</td>
        <td style="font-size: 9pt; text-align: center;">mm/hr</td>
        <td style="font-size: 9pt; text-align: right;">0 - 10 mm/hr</td>
    </tr>
    <!-- Método del standalone (si tiene) -->
    <tr>
        <td colspan="4" style="font-size: 8pt; color: #666; font-style: italic; padding: 0 4px 4px 12px;">
            Método: Westergreen
        </td>
    </tr>
</table>
```

### Identificar padre/hijo/standalone:

```php
@php
    $parentTestIds = [];
    $childTestIds = [];
    
    foreach ($sample->determinations as $det) {
        if ($det->test && $det->test->childTests->count() > 0) {
            $parentTestIds[] = $det->test_id;
        }
        if ($det->test && $det->test->parentTests->count() > 0) {
            $childTestIds[] = $det->test_id;
        }
    }
@endphp
```

Para cada determinación:
- Si `test_id` está en `$parentTestIds` → renderizar como padre (bold, sin resultado)
- Si `test_id` está en `$childTestIds` → renderizar como hijo (indentado)
- Si no está en ninguno → renderizar como standalone
