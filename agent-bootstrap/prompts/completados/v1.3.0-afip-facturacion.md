## Descripción

Integra el flujo de creación de facturas de venta con el Web Service de Factura Electrónica (WSFEv1) de AFIP. Cuando el punto de venta es electrónico, al guardar la factura se autoriza automáticamente en AFIP, se obtiene el CAE (Código de Autorización Electrónico) y el número de comprobante. Todo en homologación.

Modifica el modelo SalesInvoice, el controlador y las vistas de creación y detalle.

## PASO 3 — Extender AfipService con métodos WSFEv1

Agregar a `app/Services/AfipService.php`:

### Método `createVoucher(SalesInvoice $invoice): array`

Este es el método principal. Debe:

1. Obtener el punto de venta AFIP (`$invoice->pointOfSale->afip_pos_number`)
2. Obtener el tipo de comprobante AFIP (mapear A→1, B→6, C→11)
3. Obtener el último número autorizado via `FECompUltimoAutorizado`
4. Calcular el siguiente número
5. Armar el array de datos para `FECAESolicitar`:
   - `CantReg` = 1
   - `PtoVta` = número PdV AFIP
   - `CbteTipo` = tipo comprobante AFIP
   - `Concepto` = 1 (productos), 2 (servicios) o 3 (productos y servicios) — para laboratorio usar 3
   - `DocTipo` = tipo documento del cliente (80=CUIT, 96=DNI, 99=consumidor final)
   - `DocNro` = CUIT/DNI del cliente
   - `CbteDesde` y `CbteHasta` = siguiente número
   - `CbteFch` = fecha de emisión (formato YYYYMMDD)
   - `ImpTotal` = total de la factura
   - `ImpTotConc` = importe neto no gravado
   - `ImpNeto` = importe neto gravado (subtotal)
   - `ImpOpEx` = importe exento
   - `ImpIVA` = total IVA
   - `ImpTrib` = percepciones + otros impuestos
   - `MonId` = 'PES' (pesos argentinos)
   - `MonCotiz` = 1
   - Array `Iva` con desglose por alícuota (Id: 5=21%, 4=10.5%, 6=27%, 3=0%)
   - Si `Concepto` >= 2: `FchServDesde`, `FchServHasta`, `FchVtoPago`

6. Llamar a `$afip->ElectronicBilling->CreateVoucher($data)` o el método equivalente de la librería
7. Retornar array con: `cae`, `cae_expiration`, `voucher_number`, `result` (A/R/O), `observations`, `full_response`

### Mapeo de tipo de documento del cliente

Determinar el tipo de documento según la condición fiscal del cliente (`Customer.tax`):
- Responsable Inscripto → DocTipo 80 (CUIT), DocNro = CUIT del cliente
- Monotributista → DocTipo 80 (CUIT)
- Consumidor Final → DocTipo 99, DocNro = 0 (si total <= $xxxx) o DocTipo 96 (DNI)
- Exento → DocTipo 80 (CUIT)

### Manejo de errores

- Si AFIP rechaza: capturar los `Errors` y `Observations` de la respuesta
- Retornar error estructurado para que el controlador pueda mostrar al usuario
- Loggear la respuesta completa para auditoría

**Verificación:** El método se puede llamar con una factura de prueba (en homologación). El manejo de errores funciona con datos inválidos.

## PASO 4 — Modificar SalesInvoiceController@store

El flujo cambia así:

1. Validar datos (igual que ahora)
2. Crear la factura y sus ítems (igual que ahora, pero sin invoice_number si es electrónico)
3. Si el punto de venta es electrónico (`$pointOfSale->is_electronic`):
   a. Llamar a `AfipService::createVoucher($invoice)`
   b. Si AFIP aprueba: guardar CAE, vencimiento, número AFIP, resultado, marcar `is_electronic = true`
   c. Si AFIP rechaza: eliminar la factura recién creada y volver al form con error
4. Recalcular totales
5. Redirigir al show

**Para el número de factura:**
- Si es electrónico: el `invoice_number` se asigna con el número devuelto por AFIP (formateado con ceros a la izquierda, 8 dígitos)
- Si no es electrónico: se mantiene el comportamiento actual (ingresado por el usuario)

**Modificar la validación:**
- `invoice_number` pasa a ser `required_unless` el punto de venta es electrónico
- Cuando es electrónico, no validar uniqueness antes (AFIP asigna el número)

**Verificación:** Se puede crear una factura electrónica en homologación y obtener CAE. Se puede crear una factura normal sin cambios.

## Notas / Consideraciones

- En homologación, AFIP siempre aprueba con CAE si los datos están bien formados
- Los tipos de comprobante AFIP son: Factura A=1, B=6, C=11
- El DocTipo depende de la condición fiscal del cliente: CUIT=80, DNI=96, Consumidor Final=99
- La librería AfipSDK cachea el token/sign del WSAA — no hace login en cada request
- Guardar la respuesta completa de AFIP en `afip_response` (JSON) para auditoría
- Las facturas electrónicas NO se pueden editar una vez autorizadas
- El número de comprobante es secuencial y lo controla AFIP — nunca asignar manualmente
