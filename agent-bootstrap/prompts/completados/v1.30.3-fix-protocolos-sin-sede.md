# v1.30.3 — Fix protocolos sin sede: visibilidad y asignación masiva

**VERSIÓN:** v1.30.3  
**SLUG:** fix-protocolos-sin-sede  
**DEPENDENCIAS:** v1.30.2  
**ESTIMACIÓN:** 1.5h  
**PRIORIDAD:** crítica (bug en producción)

---

## Descripción

Los protocolos creados antes de v1.30.0 tienen `lab_branch_id = NULL`. Desde que v1.30.2
implementó el filtro automático por `active_lab_branch_id()`, estos protocolos desaparecen
de la UI cuando el usuario tiene una sede activa (por defecto o sesión).

**Problema:** `->where('lab_branch_id', $activeBranch)` excluye filas con `lab_branch_id IS NULL`.
El usuario no puede ver ni editar los protocolos huérfanos para asignarles sede.

**Solución:**
1. Incluir protocolos sin sede (`NULL`) en el filtro activo para que sigan siendo visibles
2. Marcarlos visualmente con un badge de advertencia ("Sin sede")
3. Permitir asignación masiva de sede a protocolos huérfanos desde una vista dedicada

## PASO 1 — Fix del filtro en los 3 controllers

### 1.1 — `LabAdmissionController::index()`

Cambiar:

```php
} elseif ($activeBranch = active_lab_branch_id()) {
    $query->where('lab_branch_id', $activeBranch);
}
```

Por:

```php
} elseif ($activeBranch = active_lab_branch_id()) {
    $query->where(function ($q) use ($activeBranch) {
        $q->where('lab_branch_id', $activeBranch)
          ->orWhereNull('lab_branch_id');
    });
}
```

Esto muestra los protocolos de la sede activa **más** los que no tienen sede asignada.

## PASO 4 — Asignación masiva de sede

Crear una vista/herramienta para asignar sede a protocolos huérfanos en lote.

### 4.1 — Ruta y método en `LabBranchController`

```php
public function assignOrphans()
{
    $this->authorize('lab.section');

    $orphanAdmissions = Admission::whereNull('lab_branch_id')->count();
    $orphanSamples = Sample::whereNull('lab_branch_id')->count();
    $orphanVet = VetAdmission::whereNull('lab_branch_id')->count();
    $branches = LabBranch::active()->orderByDesc('is_central')->orderBy('name')->get();

    return view('lab-branches.assign-orphans', compact(
        'orphanAdmissions', 'orphanSamples', 'orphanVet', 'branches'
    ));
}

public function assignOrphansStore(Request $request)
{
    $this->authorize('lab.section');

    $request->validate([
        'lab_branch_id' => 'required|exists:lab_branches,id',
        'modules' => 'required|array|min:1',
        'modules.*' => 'in:admissions,samples,vet_admissions',
    ]);

    $branchId = $request->lab_branch_id;
    $counts = [];

    if (in_array('admissions', $request->modules)) {
        $counts['admissions'] = Admission::whereNull('lab_branch_id')
            ->update(['lab_branch_id' => $branchId]);
    }

    if (in_array('samples', $request->modules)) {
        $counts['samples'] = Sample::whereNull('lab_branch_id')
            ->update(['lab_branch_id' => $branchId]);
    }

    if (in_array('vet_admissions', $request->modules)) {
        $counts['vet_admissions'] = VetAdmission::whereNull('lab_branch_id')
            ->update(['lab_branch_id' => $branchId]);
    }

    $total = array_sum($counts);
    $branchName = LabBranch::find($branchId)->name;

    return redirect()->route('lab-branches.index')
        ->with('success', "Se asignaron {$total} protocolos a la sede '{$branchName}'.");
}
```

### 4.2 — Rutas

```php
Route::get('lab-branches/assign-orphans', [LabBranchController::class, 'assignOrphans'])
    ->name('lab-branches.assign-orphans');
Route::post('lab-branches/assign-orphans', [LabBranchController::class, 'assignOrphansStore'])
    ->name('lab-branches.assign-orphans.store');
```

**IMPORTANTE:** Estas rutas deben ir ANTES del `Route::resource('lab-branches', ...)` para
que no sean interceptadas por el parámetro `{lab_branch}` del resource.

## Notas / Consideraciones

- El fix incluye `orWhereNull` solo cuando se filtra por sede activa (implícito). Cuando el
  usuario filtra explícitamente por una sede específica, no se incluyen huérfanos — eso es
  intencional porque el usuario eligió ver solo esa sede.
- La asignación masiva es una herramienta de migración/transición. Una vez que todos los
  protocolos tengan sede, el banner desaparece automáticamente.
- Los protocolos individuales ya se pueden editar y asignar sede desde el formulario `edit`
  de cada módulo (campo de sede agregado en v1.30.0).
- A futuro, si se desea que todos los protocolos nuevos tengan sede obligatoria, cambiar la
  validación de `nullable` a `required` en los controllers (no en esta versión).
