## Descripción

Cuando un usuario selecciona un padre en una determinación y luego quiere deseleccionarlo
(para dejarla como standalone), no puede hacerlo con un click simple. La razón es que se usa
un `<select multiple>` nativo de HTML, que requiere Ctrl+Click para deseleccionar — un
comportamiento no intuitivo para los usuarios.

La solución es reemplazar los `<select multiple>` de padres por una lista de checkboxes
que permitan seleccionar/deseleccionar con un click normal. Se mantiene el buscador existente
y la misma lógica de envío de datos al backend (`parent_ids[]`).

## PASO 3 — Actualizar la función `filterParentOptions()` para checkboxes

La función actual opera sobre `<select>` options. Hay que adaptarla para que filtre los `<label>` del contenedor de checkboxes.

**Reemplazar la función `filterParentOptions` por:**
```javascript
function filterParentOptions(searchInputId, containerId) {
    const searchInput = document.getElementById(searchInputId);
    const container = document.getElementById(containerId);
    const countDiv = document.getElementById(containerId.replace('-container', '') + '-count');
    const searchTerm = searchInput.value.toLowerCase().trim();
    
    let visibleCount = 0;
    let totalCount = 0;
    
    container.querySelectorAll('label[data-search]').forEach(label => {
        totalCount++;
        const searchData = label.getAttribute('data-search');
        if (searchTerm === '' || searchData.includes(searchTerm)) {
            label.style.display = '';
            visibleCount++;
        } else {
            label.style.display = 'none';
        }
    });
    
    if (countDiv) {
        if (searchTerm !== '' && visibleCount < totalCount) {
            countDiv.textContent = visibleCount + ' de ' + totalCount;
        } else {
            countDiv.textContent = '';
        }
    }
}
```

**Verificación:** El buscador filtra las opciones correctamente en ambos modales.
