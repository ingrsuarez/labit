# v3.2.0 — Conciliación Bancaria: Motor de Matcheo

**VERSIÓN:** v3.2.0
**SLUG:** conciliacion-bancaria-matcheo
**DEPENDENCIAS:** v3.1.0 (conciliación bancaria — cuentas e importación)
**ESTIMACIÓN:** medio día
**PRIORIDAD:** alta

---

## Descripción

Implementar el motor de conciliación bancaria que permite matchear los movimientos
importados del banco contra los registros internos del sistema (pagos, cobros,
transferencias). Incluye matching automático por monto/fecha/referencia, matching
manual desde una interfaz lado a lado, y reportes de estado de conciliación.

El flujo es: el contador abre un extracto importado, ejecuta la conciliación automática,
revisa los matches propuestos, confirma o corrige manualmente, y marca como ignorados
los que no correspondan (comisiones bancarias, impuestos, etc.).

## PASO 1 — Servicio de conciliación automática

Crear `app/Services/BankReconciliationService.php`

```php
class BankReconciliationService
{
    /**
     * Ejecuta la conciliación automática de todos los movimientos pendientes
     * de un extracto contra registros internos.
     *
     * Devuelve: ['matched' => N, 'pending' => N, 'suggestions' => [...]]
     */
    public function autoReconcile(BankStatement $statement): array
    {
        $movements = $statement->movements()
            ->where('reconciliation_status', 'pending')
            ->get();

        $matched = 0;
        $suggestions = [];

        foreach ($movements as $movement) {
            $result = $this->findMatch($movement, $statement->bankAccount->company_id);

            if ($result['confidence'] === 'exact') {
                // Match exacto: vincular automáticamente
                $this->linkMovement($movement, $result['record'], auth()->user());
                $matched++;
            } elseif ($result['confidence'] === 'probable') {
                // Match probable: sugerir pero no vincular
                $suggestions[] = [
                    'movement_id' => $movement->id,
                    'record_type' => get_class($result['record']),
                    'record_id' => $result['record']->id,
                    'record_label' => $result['label'],
                    'reason' => $result['reason'],
                ];
            }
        }

        return [
            'matched' => $matched,
            'pending' => $movements->count() - $matched,
            'suggestions' => $suggestions,
        ];
    }

    /**
     * Busca un registro interno que matchee con el movimiento bancario.
     *
     * Estrategia de búsqueda (en orden de prioridad):
     *
     * 1. Match EXACTO: mismo monto + misma fecha + referencia coincide
     *    → confidence: 'exact'
     *
     * 2. Match PROBABLE: mismo monto + fecha cercana (±3 días)
     *    → confidence: 'probable'
     *
     * 3. Sin match → confidence: 'none'
     */
    private function findMatch(BankMovement $movement, int $companyId): array
    {
        $amount = $movement->is_debit ? $movement->debit : $movement->credit;
        $date = $movement->date;

        // 1. Buscar en PaymentOrders (para débitos)
        if ($movement->is_debit) {
            // Match exacto por monto + fecha
            $po = PaymentOrder::where('company_id', $companyId)
                ->where('status', 'confirmed')
                ->where('total', $amount)
                ->whereBetween('date', [$date->copy()->subDays(3), $date->copy()->addDays(3)])
                ->whereDoesntHave('reconciledMovements')
                ->first();

            if ($po) {
                $exact = $po->date->eq($date);
                return [
                    'confidence' => $exact ? 'exact' : 'probable',
                    'record' => $po,
                    'label' => "OP #{$po->number} - {$po->supplier->name} - $" . number_format($po->total, 2, ',', '.'),
                    'reason' => $exact ? 'Monto y fecha exactos' : 'Monto exacto, fecha cercana',
                ];
            }
        }

        // 2. Buscar en CollectionReceipts (para créditos)
        if ($movement->is_credit) {
            $cr = CollectionReceipt::where('company_id', $companyId)
                ->where('total', $amount)
                ->whereBetween('date', [$date->copy()->subDays(3), $date->copy()->addDays(3)])
                ->whereDoesntHave('reconciledMovements')
                ->first();

            if ($cr) {
                $exact = $cr->date->eq($date);
                return [
                    'confidence' => $exact ? 'exact' : 'probable',
                    'record' => $cr,
                    'label' => "RC #{$cr->number} - {$cr->customer->name} - $" . number_format($cr->total, 2, ',', '.'),
                    'reason' => $exact ? 'Monto y fecha exactos' : 'Monto exacto, fecha cercana',
                ];
            }
        }

        return ['confidence' => 'none', 'record' => null, 'label' => null, 'reason' => null];
    }

    /**
     * Vincula un movimiento bancario a un registro interno.
     */
    public function linkMovement(BankMovement $movement, Model $record, User $user): void
    {
        $movement->update([
            'reconciliation_status' => 'matched',
            'reconciled_type' => get_class($record),
            'reconciled_id' => $record->id,
            'reconciled_at' => now(),
            'reconciled_by' => $user->id,
        ]);
    }

    /**
     * Desvincula un movimiento (para corregir un match erróneo).
     */
    public function unlinkMovement(BankMovement $movement): void
    {
        $movement->update([
            'reconciliation_status' => 'pending',
            'reconciled_type' => null,
            'reconciled_id' => null,
            'reconciled_at' => null,
            'reconciled_by' => null,
        ]);
    }

    /**
     * Marca un movimiento como ignorado (ej: comisiones, impuestos que no
     * tienen contrapartida en el sistema).
     */
    public function ignoreMovement(BankMovement $movement, User $user, ?string $notes = null): void
    {
        $movement->update([
            'reconciliation_status' => 'ignored',
            'reconciled_at' => now(),
            'reconciled_by' => $user->id,
            'notes' => $notes,
        ]);
    }
}
```

## PASO 3 — Controlador de conciliación

Crear `app/Http/Controllers/BankReconciliationController.php`

**index(BankAccount $bankAccount, BankStatement $statement):**
- `$this->authorize('contabilidad.bank_statements.index')`
- Cargar statement con movements, eager load reconciledRecord
- Cargar registros internos no conciliados de la empresa:
  - PaymentOrders confirmadas sin match (`whereDoesntHave('reconciledMovements')`)
  - CollectionReceipts sin match
- Pasar ambas listas a la vista de conciliación

**autoReconcile(BankAccount $bankAccount, BankStatement $statement):**
- `$this->authorize('contabilidad.reconciliation.execute')`
- POST — ejecutar `BankReconciliationService::autoReconcile()`
- Devolver redirect con resumen: "Se conciliaron X movimientos automáticamente. Y sugerencias pendientes."

**link(Request $request, BankMovement $movement):**
- `$this->authorize('contabilidad.reconciliation.manual')`
- POST — validar: reconciled_type (required, in: PaymentOrder, CollectionReceipt), reconciled_id (required, exists)
- Usar `BankReconciliationService::linkMovement()`
- Redirect back con éxito

**unlink(BankMovement $movement):**
- `$this->authorize('contabilidad.reconciliation.manual')`
- POST — Usar `BankReconciliationService::unlinkMovement()`
- Redirect back

**ignore(Request $request, BankMovement $movement):**
- `$this->authorize('contabilidad.reconciliation.manual')`
- POST — validar: notes (nullable, max:500)
- Usar `BankReconciliationService::ignoreMovement()`
- Redirect back

**bulkIgnore(Request $request, BankStatement $statement):**
- `$this->authorize('contabilidad.reconciliation.manual')`
- POST — recibir array de movement_ids + categoría (ej: ignorar todas las "comision")
- Marcar todos como ignored en bulk
- Redirect back con conteo

## PASO 4 — Vista de conciliación

### 4.1 — `resources/views/accounting/bank-statements/reconcile.blade.php`

Vista principal de conciliación, dividida en dos paneles:

**Layout:** Dos columnas (responsive: stack en mobile, lado a lado en desktop)

**Panel izquierdo: Movimientos del Banco**
- Header: "Movimientos del Banco" con conteo y filtros
- Filtros (Alpine.js): estado (todos, pendiente, conciliado, ignorado), tipo (crédito/débito), categoría
- Lista scrolleable de movimientos, cada uno como card compacta:
  - Fecha | Concepto (truncado) | Monto (verde crédito, rojo débito)
  - Badge de categoría
  - Badge de estado: pendiente (gris), conciliado (verde con nombre del registro vinculado), ignorado (amarillo)
  - Si pendiente: botón "Ignorar" (icono X gris)
  - Si conciliado: botón "Desvincular" (icono unlink)
  - Si seleccionado para match manual: highlight azul

**Panel derecho: Registros Internos**
- Header: "Registros del Sistema" con conteo y filtros
- Filtros: tipo (pagos, cobros, todos), fecha, búsqueda por texto
- Lista scrolleable de registros no conciliados:
  - Para PaymentOrder: "OP #N — Proveedor — $Monto — Fecha — Método"
  - Para CollectionReceipt: "RC #N — Cliente — $Monto — Fecha — Método"
  - Botón "Vincular" que vincula este registro con el movimiento seleccionado en el panel izquierdo

**Barra superior:**
- Botón "Conciliación Automática" (POST) — ejecuta autoReconcile y recarga
- Botón "Ignorar Comisiones/Impuestos" — bulk ignore por categoría
- Indicador de progreso: barra con % conciliado + conteos

**Barra inferior (resumen):**
- Saldo banco: $X.XXX.XXX,XX
- Saldo sistema: $X.XXX.XXX,XX (suma de registros internos del período)
- Diferencia: $X.XXX.XXX,XX (debe tender a 0 cuando todo está conciliado)
- Movimientos: N pendientes, N conciliados, N ignorados

**Interacción (Alpine.js):**
- Click en un movimiento del banco → se selecciona (highlight azul)
- Click en "Vincular" en un registro interno → POST link con el movimiento seleccionado
- Click en "Ignorar" → POST ignore
- Click en "Desvincular" → POST unlink
- Después de cada acción → reload de la página (o AJAX si es práctico)

## Notas / Consideraciones

- El matching automático es conservador: solo vincula automáticamente en caso de match exacto (monto + fecha exacta). Los probables se sugieren para revisión manual.
- El margen de ±3 días para matches probables es configurable conceptualmente pero hardcoded por ahora.
- Movimientos como comisiones bancarias, IVA sobre comisiones e impuestos generalmente se ignoran (no tienen contrapartida directa en el sistema).
- La conciliación de sueldos (ACRED SUELDO) requerirá integración con Payroll en el futuro.
- En una versión futura se podría agregar generación automática de asientos contables desde movimientos ignorados/sin match.
- El matching por referencia (nro de cheque, nro de transferencia) se puede mejorar a futuro cruzando con `payment_reference` de PaymentOrder y CollectionReceipt.
