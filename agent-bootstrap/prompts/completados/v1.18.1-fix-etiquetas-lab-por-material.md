**PRIORIDAD:** alta

## Descripción

Cambia la lógica de etiquetas del laboratorio clínico para generar una etiqueta por cada material
único del protocolo. Si un protocolo tiene prácticas con EDTA y SUERO, se imprimen 2 etiquetas
separadas (una para cada tubo). Además agrega el botón de etiquetas al listado de admisiones
para acceso rápido sin entrar al detalle.

**Problema actual:** `labelData()` devuelve 1 etiqueta con materiales concatenados ("EDTA/SUERO").
Eso no sirve para pegar en cada tubo. Se necesita 1 etiqueta por material.

## PASO 1 — Modificar `labelData()` en `LabAdmissionController`

Cambiar de devolver 1 objeto a devolver un **array de etiquetas**, una por material único.

```php
public function labelData(Admission $admission)
{
    $this->authorize('lab-labels.print');

    $admission->load('patient', 'admissionTests.test.materialRelation', 'admissionTests.test.parents');

    // Agrupar tests por material, excluyendo hijos cuando el padre ya está
    $parentTestIds = $admission->admissionTests->pluck('test_id')->toArray();
    $materialGroups = [];

    foreach ($admission->admissionTests as $at) {
        $test = $at->test;
        if (!$test || !$test->materialRelation) continue;

        // Excluir hijos si su padre ya está en el protocolo
        $isChild = $test->parents->whereIn('id', $parentTestIds)->isNotEmpty();
        if ($isChild) continue;

        $materialId = $test->materialRelation->id;
        if (!isset($materialGroups[$materialId])) {
            $materialGroups[$materialId] = [
                'material_name' => $test->materialRelation->name,
                'material_code' => $test->material_abbreviation,
                'tests' => [],
            ];
        }
        $materialGroups[$materialId]['tests'][] = $test->name;
    }

    $labels = [];
    foreach ($materialGroups as $group) {
        $labels[] = [
            'protocol_number' => $admission->protocol_number,
            'customer_name'   => $admission->patient?->full_name ?? 'Sin paciente',
            'material'        => $group['material_code'],
            'material_name'   => $group['material_name'],
            'sample_type'     => 'CLINICO',
            'entry_date'      => $admission->date?->format('d/m/Y') ?? '',
            'tests_count'     => count($group['tests']),
        ];
    }

    // Si no hay materiales (tests sin material asignado), al menos 1 etiqueta genérica
    if (empty($labels)) {
        $labels[] = [
            'protocol_number' => $admission->protocol_number,
            'customer_name'   => $admission->patient?->full_name ?? 'Sin paciente',
            'material'        => '?',
            'material_name'   => 'Sin material',
            'sample_type'     => 'CLINICO',
            'entry_date'      => $admission->date?->format('d/m/Y') ?? '',
            'tests_count'     => $admission->admissionTests->count(),
        ];
    }

    return response()->json([
        'labels' => $labels,
        'total_labels' => count($labels),
    ]);
}
```

**Verificación:** GET `/lab/admissions/{id}/label-data` devuelve JSON con array `labels` de N elementos,
donde N = cantidad de materiales únicos del protocolo.

## PASO 4 — Actualizar `zebra-label-print.js` para múltiples etiquetas

Modificar la función que genera ZPL para iterar sobre el array de labels:

```js
// Antes: generaba 1 ZPL con data.materials concatenados
// Después: genera 1 ZPL con múltiples etiquetas (una ^XA...^XZ por material)

generateZPL(data) {
    // data ahora tiene { labels: [...], total_labels: N }
    let zpl = '';

    for (const label of data.labels) {
        const name = this.sanitize(label.customer_name).substring(0, 30);
        const protocol = label.protocol_number;
        const material = label.material || '?';
        const entryDate = label.entry_date || '';

        zpl += '^XA\n';
        zpl += '^CI28\n';
        zpl += '^PW400\n';
        zpl += '^LL200\n';
        // Barcode Code 128 con número de protocolo
        zpl += '^FO30,10^BCN,60,Y,N,N^FD' + protocol + '^FS\n';
        // Nombre del paciente
        zpl += '^FO30,90^A0N,22,22^FD' + name + '^FS\n';
        // Línea: CLINICO | MATERIAL | fecha
        zpl += '^FO30,120^A0N,20,20^FDCLINICO | ' + material + ' | ' + entryDate + '^FS\n';
        zpl += '^PQ1\n';
        zpl += '^XZ\n';
    }

    return zpl;
}
```

La impresora Zebra interpreta cada bloque `^XA...^XZ` como una etiqueta separada.

Actualizar también `printSampleLabel()` (o el método equivalente para lab clínico) para
pasar el nuevo formato de datos:

```js
async printLabLabel(labelDataUrl) {
    const response = await fetch(labelDataUrl);
    const data = await response.json();
    // data.labels es un array
    const zpl = this.generateZPL(data);
    await this.sendToPrinter(zpl);
}
```

**Verificación:** Protocolo con 2 materiales → impresora imprime 2 etiquetas físicas.

## Notas / Consideraciones

- El ZPL envía múltiples bloques `^XA...^XZ` en un solo string. La impresora Zebra los interpreta
  como etiquetas separadas — no hace falta enviar uno por uno.
- El filtro de padre/hijo replica la lógica de `SampleController::labelData()` (muestras) para
  consistencia entre módulos.
- Si un test no tiene material asignado (`materialRelation` es null), se genera una etiqueta
  genérica con "?" como material. Esto evita que se pierdan protocolos sin materiales configurados.
- El tamaño de etiqueta (60×30mm, PW400, LL200) no cambia — es el mismo de v1.18.0.
- A futuro se podría agregar la lista de tests por material en la etiqueta (ej: "EDTA: Hemograma,
  Eritrosedimentación") pero la etiqueta es chica y no entra mucho texto.
