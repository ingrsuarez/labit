**VERSIÓN:** v1.35.0

## Descripción

Vista de cuenta corriente por proveedor que muestra el historial completo de facturas
de compra y pagos, con saldo acumulado siguiendo el criterio contable de cuenta de pasivo:

- **Haber** → aumenta la deuda (facturas de compra recibidas, notas de crédito negativas)
- **Debe** → disminuye la deuda (órdenes de pago confirmadas, notas de crédito de proveedor)
- **Saldo acreedor** (Haber > Debe) → le debemos al proveedor (situación normal)
- **Saldo deudor** (Debe > Haber) → anticipo o pago en exceso

Filtros: proveedor + rango de fechas. Descarga en PDF.

### `index(Request $request)` — vista principal con filtros

```php
public function index(Request $request)
{
    $this->authorize('compras.section');

    $companyId = active_company_id(); // helper existente o session('selected_company_id')

    $suppliers = Supplier::orderBy('name')->get(['id', 'name', 'cuit']);

    $movements   = collect();
    $supplier    = null;
    $openBalance = 0;
    $closeBalance = 0;
    $totalHaber  = 0;
    $totalDebe   = 0;

    if ($request->filled('supplier_id')) {
        $supplier  = Supplier::findOrFail($request->supplier_id);
        $dateFrom  = $request->filled('date_from')
            ? \Carbon\Carbon::parse($request->date_from)->startOfDay()
            : now()->startOfYear();
        $dateTo    = $request->filled('date_to')
            ? \Carbon\Carbon::parse($request->date_to)->endOfDay()
            : now()->endOfDay();

        // Saldo inicial: movimientos anteriores a date_from
        $openBalance = $this->calculateOpenBalance($supplier->id, $companyId, $dateFrom);

        // Facturas de compra del período → Haber (aumentan deuda)
        $invoices = PurchaseInvoice::where('supplier_id', $supplier->id)
            ->where('company_id', $companyId)
            ->whereBetween('issue_date', [$dateFrom, $dateTo])
            ->orderBy('issue_date')
            ->orderBy('id')
            ->get(['id', 'issue_date', 'total', 'point_of_sale', 'invoice_number', 'invoice_type']);

        // Órdenes de pago confirmadas del período → Debe (disminuyen deuda)
        $payments = PaymentOrder::where('supplier_id', $supplier->id)
            ->where('company_id', $companyId)
            ->where('status', 'confirmed')
            ->whereBetween('date', [$dateFrom, $dateTo])
            ->orderBy('date')
            ->orderBy('id')
            ->get(['id', 'date', 'total', 'number', 'payment_method']);

        // Unir y ordenar cronológicamente
        $movements = collect();

        foreach ($invoices as $inv) {
            $movements->push([
                'date'        => \Carbon\Carbon::parse($inv->issue_date),
                'type'        => 'invoice',
                'reference'   => $inv->full_number ?? "{$inv->invoice_type} {$inv->point_of_sale}-{$inv->invoice_number}",
                'detail'      => 'Factura de compra',
                'debe'        => 0,
                'haber'       => $inv->total,
                'sort_key'    => $inv->issue_date . '_' . str_pad($inv->id, 10, '0', STR_PAD_LEFT),
            ]);
        }

        foreach ($payments as $pay) {
            $movements->push([
                'date'        => \Carbon\Carbon::parse($pay->date),
                'type'        => 'payment',
                'reference'   => "OP #{$pay->number}",
                'detail'      => 'Orden de pago — ' . ucfirst(str_replace('_', ' ', $pay->payment_method)),
                'debe'        => $pay->total,
                'haber'       => 0,
                'sort_key'    => $pay->date . '_' . str_pad($pay->id, 10, '0', STR_PAD_LEFT),
            ]);
        }

        // Ordenar cronológicamente
        $movements = $movements->sortBy('sort_key')->values();

        // Calcular saldo acumulado
        $running = $openBalance;
        $movements = $movements->map(function ($mov) use (&$running) {
            // Saldo acreedor: Haber aumenta, Debe disminuye
            $running += $mov['haber'] - $mov['debe'];
            $mov['saldo'] = $running;
            return $mov;
        });

        $totalHaber   = $movements->sum('haber');
        $totalDebe    = $movements->sum('debe');
        $closeBalance = $openBalance + $totalHaber - $totalDebe;
    }

    return view('suppliers.statement', compact(
        'suppliers', 'supplier', 'movements',
        'openBalance', 'closeBalance',
        'totalHaber', 'totalDebe',
        'request'
    ));
}
```

### `pdf(Request $request)` — descarga PDF

```php
public function pdf(Request $request)
{
    $this->authorize('compras.section');

    $request->validate([
        'supplier_id' => 'required|exists:suppliers,id',
        'date_from'   => 'nullable|date',
        'date_to'     => 'nullable|date',
    ]);

    // Reusar la misma lógica de index()
    // Llamar al método privado que construye los datos
    $data = $this->buildStatementData(
        $request->supplier_id,
        active_company_id(),
        $request->date_from,
        $request->date_to
    );

    $pdf = \PDF::loadView('suppliers.statement-pdf', $data);
    $pdf->setPaper('A4', 'portrait');

    $filename = 'cuenta-corriente-' . \Str::slug($data['supplier']->name) . '-' . now()->format('Ymd') . '.pdf';

    return $pdf->download($filename);
}
```

### `calculateOpenBalance()` — método privado

```php
private function calculateOpenBalance(int $supplierId, int $companyId, \Carbon\Carbon $dateFrom): float
{
    // Facturas antes del período (Haber)
    $invoicesBefore = PurchaseInvoice::where('supplier_id', $supplierId)
        ->where('company_id', $companyId)
        ->where('issue_date', '<', $dateFrom->toDateString())
        ->sum('total');

    // Pagos antes del período (Debe)
    $paymentsBefore = PaymentOrder::where('supplier_id', $supplierId)
        ->where('company_id', $companyId)
        ->where('status', 'confirmed')
        ->where('date', '<', $dateFrom->toDateString())
        ->sum('total');

    // Saldo acreedor: Haber - Debe
    return $invoicesBefore - $paymentsBefore;
}
```

### `buildStatementData()` — método privado reutilizable

Extraer la lógica de `index()` que construye `$movements`, `$openBalance`, etc.
en un método privado `buildStatementData(int $supplierId, int $companyId, ?string $dateFrom, ?string $dateTo): array`
para que tanto `index()` como `pdf()` lo usen sin duplicar código.

## Notas / Consideraciones

- Los tipos de movimiento incluidos son facturas de compra (PurchaseInvoice) y órdenes
  de pago confirmadas (PaymentOrder). Si en el futuro se agregan Notas de Débito de proveedor,
  irían al Haber también (aumentan la deuda). Las NC de proveedor irían al Debe.
- El PDF usa mPDF (ya instalado) con estilos inline para máxima compatibilidad.
  No usar Tailwind en el template PDF — mPDF no lo soporta.
- El saldo deudor (en naranja) indica que el proveedor nos debe dinero — puede ocurrir
  si se hizo un anticipo antes de recibir la factura.
- No se paginan los movimientos en la vista HTML — se muestran todos.
  Si un proveedor tiene cientos de facturas en un año, el contador puede acotar el rango de fechas.
