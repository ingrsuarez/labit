# v1.34.0 — Asociación de remitos y control de stock en facturas de compra

**VERSIÓN:** v1.34.0  
**SLUG:** remitos-asociados-control-stock  
**DEPENDENCIAS:** v1.33.0, v1.27.0, v1.32.0  
**ESTIMACIÓN:** medio día  
**PRIORIDAD:** alta

---

## Descripción

Extiende el formulario de factura de compra para:

1. **Asociar un remito existente** al crear/editar la factura. Al seleccionar el remito,
   se importan automáticamente sus ítems (insumo + cantidad) sin precio para agilizar la carga.
2. **Control de stock por ítem**: si la factura no tiene remito, cada ítem tiene un checkbox
   "Actualiza stock" tildado por defecto. Al guardar, solo los ítems con la tilde activa
   generan movimiento de stock. Si hay remito asociado, el stock ya fue actualizado por él
   y el checkbox aparece deshabilitado con badge informativo.

El campo `delivery_note_id` ya existe en `purchase_invoices` (v1.27.0). Esta versión
lo expone en el formulario y agrega control granular por ítem.

## PASO 2 — Endpoint para listar remitos por proveedor

Agregar un endpoint en `DeliveryNoteController` que devuelve los remitos de un proveedor
disponibles para asociar (sin factura asociada aún, o todos — ver nota):

```php
// routes/web.php
Route::get('delivery-notes/by-supplier', [DeliveryNoteController::class, 'bySupplier'])
    ->name('delivery-notes.by-supplier');
```

```php
// DeliveryNoteController.php
public function bySupplier(Request $request)
{
    $request->validate(['supplier_id' => 'required|integer']);

    $notes = DeliveryNote::where('supplier_id', $request->supplier_id)
        ->with('items.supply:id,code,name')
        ->orderByDesc('date')
        ->limit(50)
        ->get(['id', 'number', 'date', 'supplier_id']);

    return response()->json($notes);
}
```

**Nota:** No filtrar por "sin factura" porque puede haber remitos que generaron múltiples
facturas parciales. El usuario elige qué remito asociar.

**Verificación:** `GET /delivery-notes/by-supplier?supplier_id=1` retorna JSON con remitos del proveedor.

## PASO 3 — Endpoint para obtener ítems de un remito

```php
// routes/web.php
Route::get('delivery-notes/{deliveryNote}/items', [DeliveryNoteController::class, 'getItems'])
    ->name('delivery-notes.items');
```

```php
// DeliveryNoteController.php
public function getItems(DeliveryNote $deliveryNote)
{
    $items = $deliveryNote->items()->with('supply:id,code,name,tracks_lot')->get([
        'id', 'supply_id', 'quantity', 'lot_number', 'notes'
    ]);

    return response()->json([
        'delivery_note' => [
            'id'     => $deliveryNote->id,
            'number' => $deliveryNote->number,
            'date'   => $deliveryNote->date,
        ],
        'items' => $items->map(fn($item) => [
            'supply_id'    => $item->supply_id,
            '_supply_name' => $item->supply?->name ?? '',
            '_supply_code' => $item->supply?->code ?? '',
            '_tracks_lot'  => $item->supply?->tracks_lot ?? false,
            'quantity'     => $item->quantity,
            'lot_number'   => $item->lot_number,
            'unit_price'   => null,   // se completa en la factura
            'iva_rate'     => 21,     // default, editable
            'updates_stock'=> false,  // ya actualizado por el remito
            'description'  => $item->supply?->name ?? '',
        ]),
    ]);
}
```

**Verificación:** `GET /delivery-notes/1/items` retorna el remito y sus ítems con supply_id y quantity.

### 4.2 — Estado y funciones en `invoiceForm()`

Agregar al estado:

```javascript
// Estado del remito
delivery_note_id: null,
selectedDeliveryNoteId: '',
selectedDeliveryNoteNumber: '',
availableDeliveryNotes: [],
loadingDeliveryNotes: false,
```

Agregar las funciones:

```javascript
// Cuando cambia el proveedor, cargar sus remitos
async onSupplierChange() {
    this.delivery_note_id = null;
    this.selectedDeliveryNoteId = '';
    this.selectedDeliveryNoteNumber = '';
    this.availableDeliveryNotes = [];
    if (!this.supplier_id) return;

    this.loadingDeliveryNotes = true;
    try {
        const resp = await fetch(`{{ route('delivery-notes.by-supplier') }}?supplier_id=${this.supplier_id}`);
        this.availableDeliveryNotes = await resp.json();
    } catch (e) {
        this.availableDeliveryNotes = [];
    } finally {
        this.loadingDeliveryNotes = false;
    }
    this.checkDuplicate(); // también re-verificar duplicados al cambiar proveedor
},

// Cuando se selecciona o cambia el remito en el select
async onDeliveryNoteChange() {
    if (!this.selectedDeliveryNoteId) {
        this.delivery_note_id = null;
        this.selectedDeliveryNoteNumber = '';
        return;
    }

    // Confirmar si hay ítems cargados
    if (this.items.length > 0 && !confirm('Al asociar el remito se van a reemplazar los ítems actuales. ¿Continuás?')) {
        this.selectedDeliveryNoteId = '';
        return;
    }

    try {
        const resp = await fetch(`/delivery-notes/${this.selectedDeliveryNoteId}/items`);
        const data = await resp.json();

        this.delivery_note_id = data.delivery_note.id;
        this.selectedDeliveryNoteNumber = data.delivery_note.number;

        // Importar ítems del remito (sin precio)
        this.items = data.items.map(item => ({
            ...item,
            iva_rate: 21,
            updates_stock: false, // ya actualizado por remito
        }));
    } catch (e) {
        alert('Error al cargar los ítems del remito. Intentá de nuevo.');
        this.selectedDeliveryNoteId = '';
    }
},

// Quitar remito asociado
removeDeliveryNote() {
    if (this.items.length > 0 && !confirm('Al quitar el remito, los ítems importados quedan pero el stock ahora se actualizará al guardar. ¿Continuás?')) return;
    this.delivery_note_id = null;
    this.selectedDeliveryNoteId = '';
    this.selectedDeliveryNoteNumber = '';
    // Marcar ítems como updates_stock = true (ahora sin remito)
    this.items = this.items.map(item => ({ ...item, updates_stock: true }));
},
```

Agregar `@change="onSupplierChange()"` al select de proveedor en el formulario.

**Verificación:** Seleccionar proveedor → el select de remito se habilita con sus remitos.
Seleccionar un remito → se importan sus ítems con insumo y cantidad, precio vacío.

## PASO 6 — Modificar `store()` para respetar `updates_stock` por ítem

En `PurchaseInvoiceController::store()`, reemplazar la condición actual de stock
(que impacta todos los ítems con supply_id cuando no hay delivery_note_id) por
la lógica granular con `updates_stock`:

```php
// Reemplazar bloque de stock existente de v1.27.0:

// Generar entrada de stock solo para ítems que lo requieren
foreach ($invoice->items()->whereNotNull('supply_id')->where('updates_stock', true)->get() as $item) {
    $supply = $item->supply;
    if (!$supply) continue;

    $previousStock = $supply->stock;
    $newStock = $previousStock + $item->quantity;

    StockMovement::create([
        'supply_id'       => $supply->id,
        'type'            => 'entrada',
        'quantity'        => $item->quantity,
        'previous_stock'  => $previousStock,
        'new_stock'       => $newStock,
        'reason'          => 'compra',
        'reference_type'  => PurchaseInvoice::class,
        'reference_id'    => $invoice->id,
        'lot_number'      => $item->lot_number ?? null,
        'expiration_date' => $item->expiration_date ?? null,
        'notes'           => "Factura de compra {$invoice->full_number}",
        'user_id'         => auth()->id(),
    ]);

    $supply->update([
        'stock'      => $newStock,
        'last_price' => $item->unit_price,
    ]);
}
```

**Nota importante:** Eliminar la condición `if (!$invoice->delivery_note_id)` del bloque original.
Ahora el control granular viene del campo `updates_stock` de cada ítem — si tiene remito,
los ítems vienen con `updates_stock = false` desde el frontend, así que no generan stock de todas formas.

## Notas / Consideraciones

- La condición `if (!$invoice->delivery_note_id)` de v1.27.0 en `store()` es reemplazada por
  el control granular `updates_stock` por ítem. El resultado es el mismo para el flujo original
  (sin remito = todos los ítems con supply_id actualizan stock) pero ahora el usuario tiene
  control explícito sobre cada ítem.
- No se filtra remitos "sin factura" en el selector — un remito puede relacionarse con múltiples
  facturas (ej: facturación parcial). El usuario es responsable de no duplicar el stock.
- Los precios de los ítems importados desde el remito vienen vacíos intencionalmente:
  el remito registra recepción física, la factura registra el precio acordado.
