# v1.61.0 — Enter agrega ítem en Factura de Compra (igual que en remito)

**VERSIÓN:** v1.61.0
**SLUG:** enter-agrega-item-fc
**DEPENDENCIAS:** ninguna
**ESTIMACIÓN:** 30min
**PRIORIDAD:** alta (UX)

---

## Descripción

Al presionar Enter en cualquier campo editable de una fila de ítem de la FC
(Cantidad, Precio Unit., Lote, Vencimiento), se debe agregar automáticamente
una nueva fila y posicionar el foco en el campo de búsqueda de insumo de esa
fila nueva.

El remito ya tiene este comportamiento implementado. La FC no lo tiene:
solo previene el submit con `@keydown.enter="if ($event.target.tagName === 'INPUT') $event.preventDefault()"`.

**Mecanismo del remito a replicar:**
- El div raíz escucha `@add-remito-item="addFreeItemAndFocusNext()"`
- Los inputs disparan `@keydown.enter.prevent="$dispatch('add-remito-item')"`
- `addFreeItemAndFocusNext()` agrega el ítem y hace focus en `item-supply-{idx}` vía `$nextTick`

## PASO 1 — Agregar listener en el elemento raíz con x-data="invoiceForm()"

Localizar el elemento HTML (div o similar) que tiene `x-data="invoiceForm()"`.

Agregarle el listener del evento personalizado:

**ANTES:**
```html
x-data="invoiceForm()"
```

**DESPUÉS:**
```html
x-data="invoiceForm()" @add-fc-item="addItemAndFocusNext()"
```

## PASO 2 — Agregar addItemAndFocusNext() en invoiceForm()

Dentro de la función `invoiceForm()`, inmediatamente después de `addItem()` (línea ~1018):

**Agregar:**
```javascript
addItemAndFocusNext() {
    this.addItem();
    this.$nextTick(() => {
        const idx = this.items.length - 1;
        const el = document.getElementById('item-desc-' + idx);
        el?.focus();
    });
},
```
