# v1.77.0 — Vista recepción-lab: determinaciones leaf + CRUD restringido + eliminar protocolo

**Tipo:** minor — nueva funcionalidad
**Prioridad:** alta — requerimiento operativo en producción
**Dependencias:** v1.5.1 (roles y permisos lab), v2.5.0 (trait Auditable) — ambas completadas

---

## Qué hace

Adapta la experiencia del rol `recepcion-lab` en los tres módulos de protocolo
(clínico, veterinario, muestras):

1. **Vista filtrada de determinaciones**: el recepcionista ve solo filas **padre** y,
   de los hijos, únicamente los que **no tienen su padre cargado en el protocolo** (huérfanos).
   Los hijos cuyo padre ya está en el protocolo se ocultan — están representados por la fila padre.
   Ejemplo: si el protocolo tiene "hemograma" (padre) + sus hijos, el recepcionista ve solo
   "hemograma". Si en cambio un hijo fue cargado directamente sin su padre, sí lo ve.

2. **CRUD restringido por estado**:
   - Puede agregar prácticas (sin restricción adicional — flujo existente).
   - Puede eliminar una determinación **solo si está pendiente**:
     - Lab clínico: `!$test->is_validated && !$test->hasResult()`
     - Vet y muestras: `$determination->status === 'pending'`
   - Si la determinación está en proceso o superior: el botón de eliminar **no aparece**.
   - Bioquímico y admin pueden eliminar cualquier determinación (sin cambio en su flujo).

3. **Eliminar protocolo completo**: el recepcionista puede eliminar un protocolo entero
   solo si **todas** sus determinaciones están pendientes (condición igual a la de arriba).
   Hard delete. Se registra en `audit_logs`.

## PASO 4 — Implementar método `destroy()` en cada controller

### 4.1 — `LabAdmissionController::destroy()`

```php
public function destroy(Admission $admission)
{
    $this->authorize('lab-admissions.delete');

    if (auth()->user()->hasRole('recepcion-lab')) {
        $hasInProcess = $admission->tests->contains(fn($t) => $t->is_validated || $t->hasResult());
        if ($hasInProcess) {
            return redirect()->back()
                ->with('error', 'Solo se puede eliminar el protocolo si todas las prácticas están pendientes.');
        }
    }

    $protocolNumber = $admission->protocol_number ?? $admission->id;

    if (in_array(\App\Traits\Auditable::class, class_uses_recursive($admission))) {
        $admission->logAudit('deleted', "Protocolo clínico #{$protocolNumber} eliminado por " . auth()->user()->name);
    }

    $admission->delete();

    return redirect()->route('lab.admissions.index')
        ->with('success', "Protocolo #{$protocolNumber} eliminado correctamente.");
}
```

### 4.2 — `VetAdmissionController::destroy()`

```php
public function destroy(VetAdmission $vetAdmission)
{
    $this->authorize('vet-admissions.delete');

    if (auth()->user()->hasRole('recepcion-lab')) {
        $allPending = $vetAdmission->vetAdmissionTests->every(fn($t) => $t->status === 'pending');
        if (! $allPending) {
            return redirect()->back()
                ->with('error', 'Solo se puede eliminar el protocolo si todas las prácticas están pendientes.');
        }
    }

    $protocolNumber = $vetAdmission->protocol_number ?? $vetAdmission->id;

    if (in_array(\App\Traits\Auditable::class, class_uses_recursive($vetAdmission))) {
        $vetAdmission->logAudit('deleted', "Protocolo veterinario #{$protocolNumber} eliminado por " . auth()->user()->name);
    }

    $vetAdmission->delete();

    return redirect()->route('vet.admissions.index')
        ->with('success', "Protocolo #{$protocolNumber} eliminado correctamente.");
}
```

### 4.3 — `SampleController::destroy()`

```php
public function destroy(Sample $sample)
{
    $this->authorize('samples.delete');

    if (auth()->user()->hasRole('recepcion-lab')) {
        $allPending = $sample->determinations->every(fn($d) => $d->status === 'pending');
        if (! $allPending) {
            return redirect()->back()
                ->with('error', 'Solo se puede eliminar el protocolo si todas las determinaciones están pendientes.');
        }
    }

    $protocolNumber = $sample->protocol_number ?? $sample->id;

    if (in_array(\App\Traits\Auditable::class, class_uses_recursive($sample))) {
        $sample->logAudit('deleted', "Protocolo muestras #{$protocolNumber} eliminado por " . auth()->user()->name);
    }

    $sample->delete();

    return redirect()->route('sample.index')
        ->with('success', "Protocolo #{$protocolNumber} eliminado correctamente.");
}
```

## PASO 6 — Adaptar vistas show para rol `recepcion-lab`

### 6.1 — `resources/views/lab/admissions/show.blade.php`

**6.1.1 — Filtrar filas hijo con padre presente**

El recepcionista debe ver:
- Determinaciones **padre** (su test tiene hijos en el protocolo) → siempre visibles.
- Determinaciones **hijo huérfano** (su padre no está cargado en el protocolo) → visibles.
- Determinaciones **hijo con padre presente** → ocultas para `recepcion-lab`.

Calcular `$testIdsInProtocol` una vez antes del bucle (evita N queries):

```blade
@php $testIdsInProtocol = $admission->tests->pluck('test_id')->toArray(); @endphp

@foreach($admissionTests as $admissionTest)
    @php
        $parentTest = $admissionTest->test->parentTest ?? null;
        $parentIsInProtocol = $parentTest && in_array($parentTest->id, $testIdsInProtocol);
    @endphp
    @if(!$isRecepcionLab || !$parentIsInProtocol)
        {{-- fila completa de la determinación --}}
    @endif
@endforeach
```

> `$admissionTest->test->parentTest` es la relación al padre directo del Test en el catálogo.
> Verificar el nombre exacto de la relación en `app/Models/Test.php` (puede ser `parent`,
> `parentTest`, `parentTests`, etc.) y adaptar. Asegurarse de que esté eager-loaded en
> `show()`: `$admission->load('tests.test.parentTest')` o equivalente.

**6.1.2 — Botón de eliminar determinación individual**

Localizar el botón de eliminar práctica existente (busca la llamada a
`route('lab.admissions.removeTest', ...)`). Envolver con condición:

```blade
@if(!$isRecepcionLab || (!$admissionTest->is_validated && !$admissionTest->hasResult()))
    {{-- botón eliminar existente sin modificar --}}
@endif
```

**6.1.3 — Botón "Eliminar protocolo"**

Agregar en la sección de acciones del header (cerca de los botones de PDF, email, etc.):

```blade
@can('lab-admissions.delete')
    @php
        $allPending = $admission->tests->every(fn($t) => !$t->is_validated && !$t->hasResult());
    @endphp
    @if($allPending)
        <button
            type="button"
            onclick="if(confirm('¿Eliminar este protocolo? Esta acción no se puede deshacer.')) submitAction('{{ route('lab.admissions.destroy', $admission) }}', 'DELETE')"
            class="px-3 py-1.5 text-sm bg-red-600 text-white rounded hover:bg-red-700 font-medium">
            Eliminar protocolo
        </button>
    @endif
@endcan
```

> Verificar que la función `submitAction(url, method)` ya existe en el Blade (se usa para
> `removeTest`). Si usa Alpine u otro mecanismo, adaptar al patrón existente.
