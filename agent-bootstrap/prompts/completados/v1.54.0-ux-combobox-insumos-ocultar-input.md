# v1.54.0 — UX combobox insumos: ocultar input al seleccionar

**VERSIÓN:** v1.54.0
**SLUG:** ux-combobox-insumos-ocultar-input
**DEPENDENCIAS:** v1.32.0, v1.36.0
**ESTIMACIÓN:** 30min
**PRIORIDAD:** alta

---

## Descripción

Al seleccionar un insumo en el buscador de ítems libres (remitos y facturas de compra),
el campo de texto (`searchText`) queda visible con el nombre del insumo Y el badge
de código+nombre también aparece. El resultado es que el insumo se muestra duplicado.

El fix debe:
1. Ocultar el input de búsqueda cuando hay un insumo seleccionado (`item.supply_id` no vacío)
2. Mostrar solo el badge (código + nombre + botón ×)
3. Al clickear × (unlinkSupply), mostrar el input vacío y darle foco automáticamente
4. Corregir el atributo `required` para que no bloquee validación HTML5 cuando el input está oculto

El mismo `supplySearch()` está definido en 4 vistas — el fix va en todas.

## PASO 1 — Fix en delivery-notes/create.blade.php

Archivo: `resources/views/delivery-notes/create.blade.php`

### 1.1 — HTML: ocultar input cuando hay selección

Localizar el `<input type="text" x-model="searchText" ...>` dentro del bloque
`<template x-if="!item.from_po">`. Agregarle `x-show="!item.supply_id"`:

```html
<input type="text"
       x-show="!item.supply_id"
       x-model="searchText"
       @input.debounce.300ms="doSearch()"
       @focus="if (searchText.length >= 2) showResults = true"
       @keydown.arrow-down.prevent="highlightNext()"
       @keydown.arrow-up.prevent="highlightPrev()"
       @keydown.enter.prevent="onSupplySearchEnter()"
       @keydown.escape="showResults = false"
       @keydown.tab="onTab($event)"
       placeholder="Buscar insumo..."
       :required="!item.supply_id"
       :id="'item-supply-' + index"
       class="flex-1 min-w-0 rounded border-gray-300 text-sm focus:border-zinc-500 focus:ring-zinc-500">
```

Cambios aplicados:
- Agregar `x-show="!item.supply_id"` al input
- Cambiar `required` por `:required="!item.supply_id"` (evita error de validación HTML5 cuando el input está oculto pero hay supply seleccionado)

### 1.2 — JS: limpiar searchText al seleccionar, completar reset en unlinkSupply

En la función `supplySearch()` del `<script>` al final de la vista:

**En `selectSupply(supply)`**, cambiar la línea que asigna el label al searchText:

```js
// ANTES:
this.searchText = label;

// DESPUÉS:
this.searchText = '';
```

**En `unlinkSupply()`**, agregar el reset de `_supply_label` y el refocus al input:

```js
unlinkSupply() {
    item.supply_id = '';
    item._supply_code = '';
    item._supply_name = '';
    item._supply_label = '';
    this.searchText = '';
    this.$nextTick(() => {
        const el = document.querySelector(`#item-supply-${index}`);
        if (el) el.focus();
    });
},
```

**Verificación:** Al seleccionar un insumo, el input desaparece y queda solo el badge.
Al clickear ×, el badge desaparece y el input aparece vacío con el foco puesto.
