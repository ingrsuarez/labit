## Descripción

Obras sociales y clientes (aguas, veterinario, etc.) necesitan más de un correo — por ejemplo uno para **resultados** y otro para **facturación/pagos**. Hoy `customers.email` e `insurances.email` son un solo campo.

Esta versión agrega persistencia polimórfica, migra datos existentes, ABM en formularios y mantiene la columna legacy `email` sincronizada con el principal para compatibilidad.

**Diseño obligatorio:** `docs/designs/DISEÑO_v1.116.0-multiples-emails-cliente.md` (secciones repeater + listados).

**Fuera de scope:** cambios en envío de protocolos (v1.116.1), pacientes, presupuestos.
