# v1.7.0 — Cobro a particulares y control de deuda

**VERSIÓN:** v1.7.0  
**SLUG:** cobro-particulares-control-deuda  
**DEPENDENCIAS:** v1.6.1 (Particular debe existir como obra social)  
**ESTIMACIÓN:** 5h  
**PRIORIDAD:** alta

---

## Descripción

Agrega el flujo de cobro para pacientes con cobertura "Particular": desglose de
precios por determinación en la admisión, registro de pago (total o parcial) con
medio de pago, estado de deuda, y listado de deudores.

### Medios de pago

- Efectivo
- Transferencia bancaria
- Mercado Pago

(Solo registro manual — no hay integración con pasarelas de pago.)

### Estados de pago

- `pagado` — pagó el total
- `parcial` — pagó menos del total, queda saldo
- `pendiente` — no pagó nada (debe)

### Flujo de cobro

1. Al crear admisión con "Particular": se muestran precios por determinación y total
2. El operador puede registrar el pago inmediato (total o parcial) con medio de pago
3. Si no paga, queda como "pendiente" (debe)
4. Desde el show de la admisión se puede registrar pago posterior
5. Desde el listado de deudores se ven todas las admisiones con saldo pendiente

## PASO 1 — Migración: campos de pago en admissions

Crear migración `add_payment_fields_to_admissions_table`.

```php
Schema::table('admissions', function (Blueprint $table) {
    $table->string('payment_status')->default('not_applicable')->after('status')
          ->comment('not_applicable|pendiente|parcial|pagado');
    $table->string('payment_method')->nullable()->after('payment_status')
          ->comment('efectivo|transferencia|mercadopago');
    $table->decimal('paid_amount', 12, 2)->default(0)->after('payment_method');
    $table->timestamp('payment_date')->nullable()->after('paid_amount');
    $table->text('payment_notes')->nullable()->after('payment_date');
});
```

**Notas:**
- `payment_status = 'not_applicable'` para admisiones con obra social (no particular)
- `payment_status = 'pendiente'` por defecto cuando la obra social es Particular
- `paid_amount` permite cobro parcial: si `paid_amount < total`, status = 'parcial'
- `payment_method` es nullable (null = no pagó)

**Verificación:** La migración corre sin errores. Los registros existentes quedan con
`payment_status = 'not_applicable'`.

public function isParticular(): bool
{
    return $this->insuranceRelation?->type === 'particular';
}

public function getBalanceAttribute(): float
{
    $total = $this->total_patient ?: ($this->patient_price ?: 0);
    return max(0, $total - ($this->paid_amount ?? 0));

## PASO 4 — Guardar pago en store()

En `LabAdmissionController::store()`, después de crear la admisión y calcular totales:

```php
// Determinar estado de pago
$insurance = Insurance::find($request->insurance_id);
if ($insurance && $insurance->type === 'particular') {
    $paidAmount = (float) ($request->paid_amount ?? 0);
    $total = $admission->total_patient ?: $admission->patient_price;

    if ($paidAmount <= 0) {
        $paymentStatus = 'pendiente';
    } elseif ($paidAmount >= $total) {
        $paymentStatus = 'pagado';
        $paidAmount = $total; // No pagar de más
    } else {
        $paymentStatus = 'parcial';
    }

    $admission->update([
        'payment_status' => $paymentStatus,
        'payment_method' => $request->payment_method,
        'paid_amount' => $paidAmount,
        'payment_date' => $paidAmount > 0 ? now() : null,
        'payment_notes' => $request->payment_notes,
    ]);
} else {
    $admission->update(['payment_status' => 'not_applicable']);
}
```

## PASO 6 — Endpoint registerPayment

En `LabAdmissionController`, agregar método para registrar pago posterior:

```php
public function registerPayment(Request $request, Admission $admission)
{
    $this->authorize('lab-admissions.edit');
    
    $request->validate([
        'payment_method' => 'required|in:efectivo,transferencia,mercadopago',
        'amount' => 'required|numeric|min:0.01|max:' . $admission->balance,
        'payment_notes' => 'nullable|string|max:255',
    ]);

    $newPaid = $admission->paid_amount + $request->amount;
    $total = $admission->total_to_pay;

    $admission->update([
        'paid_amount' => $newPaid,
        'payment_method' => $request->payment_method,
        'payment_date' => now(),
        'payment_status' => $newPaid >= $total ? 'pagado' : 'parcial',
        'payment_notes' => trim(($admission->payment_notes ? $admission->payment_notes . ' | ' : '') . $request->payment_notes),
    ]);

    return back()->with('success', 'Pago registrado correctamente.');
}
```

Agregar la ruta en `routes/web.php`:
```php
Route::post('/lab/admissions/{admission}/payment', [...LabAdmissionController::class, 'registerPayment'])->name('lab.admissions.registerPayment');
```

Agregar método en `LabAdmissionController`:

```php
public function debtors(Request $request)
{
    $this->authorize('lab-admissions.index');
    
    $query = Admission::debtors()
        ->with(['patient', 'insuranceRelation'])
        ->orderBy('date', 'desc');

    if ($request->filled('search')) {
        $search = $request->search;
        $query->whereHas('patient', function ($q) use ($search) {
            $q->where('name', 'like', "%{$search}%")
              ->orWhere('lastName', 'like', "%{$search}%")
              ->orWhere('patientId', 'like', "%{$search}%");
        });
    }

    if ($request->filled('date_from')) {
        $query->where('date', '>=', $request->date_from);
    }
    if ($request->filled('date_to')) {
        $query->where('date', '<=', $request->date_to);
    }

    $admissions = $query->paginate(20)->withQueryString();
    
    $totalDebt = Admission::debtors()->sum(DB::raw('
        CASE 
            WHEN total_patient > 0 THEN total_patient - paid_amount
            WHEN patient_price > 0 THEN patient_price - paid_amount
            ELSE 0
        END
    '));

    return view('lab.admissions.debtors', compact('admissions', 'totalDebt'));
}
```
