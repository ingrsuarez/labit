## Descripción

Crear un seeder **standalone para producción** que importe el archivo Excel del nomenclador
de la Obra Social de Conductores Camioneros como nomenclador base (`Insurance` con
`type = 'nomenclador'`, nombre **"Nomenclador Camioneros"**).

El archivo fuente está hoy en `customer_files/nomenclador 2016 camioneros.xlsx` y debe
copiarse a `docs/Nomenclador camioneros.xlsx` para que viaje en el deploy (mismo criterio
que v1.107.0 con laborales).

Este nomenclador es **distinto** del "Nomenclador 2016" genérico ya importado en v1.10.0
(`docs/2016.xlsx`, ~1264 prácticas): el Excel de Camioneros tiene ~1270 filas con pequeñas
diferencias en nombres y cantidad de prácticas. No sobrescribir el nomenclador existente.

El seeder **no** crea ni modifica coberturas de obra social. Post-deploy en producción se
asigna manualmente en la UI de obras sociales el campo **"Nomenclador para Facturar"**
(`nomenclator_id`).

**Estructura del Excel (verificado):**
- Fila 1: encabezados (`CODIGO ANALISIS`, `NOMBRE`, `NIVEL`)
- Filas 2+: datos (~1270 prácticas)
- Columna A = código (numérico, ej. `15`, `19`, `22`)
- Columna B = nombre de práctica
- Columna C = valor NBU (columna titulada "NIVEL" en el Excel)

**Comando de producción:**
```bash
php artisan db:seed --class=NomencladorCamionerosSeeder
```

**NO incluir** en `DatabaseSeeder` (ejecución manual en prod).

## PASO 2 — Crear `NomencladorCamionerosSeeder.php`

Crear `database/seeders/NomencladorCamionerosSeeder.php`.

### Requisitos

1. **Archivo fuente:** `base_path('docs/Nomenclador camioneros.xlsx')`
   - Si no existe → `$this->command->error(...)` y `return`.

2. **Nomenclador base:**
   ```php
   Insurance::firstOrCreate(
       ['name' => 'Nomenclador Camioneros'],
       [
           'type' => 'nomenclador',
           'tax_id' => null,
           'tax' => null,
           'phone' => null,
           'email' => null,
           'address' => null,
           'state' => null,
           'country' => 'argentina',
           'price' => 0,
           'nbu' => 0,
           'nbu_value' => 1,
           'group' => null,
           'instructions' => 'Nomenclador de prácticas Obra Social Camioneros. Importado desde docs/Nomenclador camioneros.xlsx',
       ]
   );
   ```
   - Si ya existía con otro `type`, forzar `type = 'nomenclador'`.

3. **Loop de filas:** desde fila **2** hasta `getHighestRow()` (fila 1 = encabezados).
   - A = código, B = nombre, C = NBU.
   - Saltar filas con código vacío.
   - Normalizar código a string: `trim((string) $codigo)`.

4. **Tests:** mismo criterio que `NomencladoresExcelSeeder`:
   - Buscar `Test::where('code', $codigo)`.
   - Si existe → actualizar `nbu`.
   - Si no → crear con campos null como en el seeder de referencia.

5. **InsuranceTest:** upsert por `(insurance_id, test_id)`:
   - Actualizar `nbu_units` y `requires_authorization = false`.
   - Crear con `price => null`, `copago => 0` si no existe.

6. **`parseNbu()`:** copiar la implementación de `NomencladoresExcelSeeder`
   (soporta formato AR: miles con punto, decimal con coma).

7. **Logging al final:**
   - Tests nuevos creados
   - Prácticas creadas / actualizadas
   - Filas saltadas

**Verificación:** el seeder corre sin error en entorno local con el Excel presente.

## Notas / Consideraciones

- Los códigos del Excel pueden coincidir con tests existentes del nomenclador bioquímico
  o de "Nomenclador 2016". El seeder matchea **por `code`**; si un código ya existe,
  actualiza NBU del test existente (mismo comportamiento que v1.10.0/v1.107.0).
- Post-deploy en producción: asignar manualmente `nomenclator_id` en la cobertura de
  Obra Social Camioneros + configurar `nbu_value` de la cobertura según convenio vigente.
- El Excel de Camioneros es una variante del nomenclador 2016 (~6 prácticas más que
  `docs/2016.xlsx`); mantener ambos nomencladores separados en el sistema.
