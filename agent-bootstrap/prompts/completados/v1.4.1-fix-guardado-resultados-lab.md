# v1.4.1 — Fix guardado de resultados de protocolo de paciente

**VERSIÓN:** v1.4.1  
**SLUG:** fix-guardado-resultados-lab  
**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 30min  
**PRIORIDAD:** crítica (bug en producción)

---

## Descripción

El director técnico reporta un error al intentar guardar resultados de un protocolo
de paciente en el módulo de laboratorio clínico. Al presionar "Guardar", el sistema
lanza un `MethodNotAllowedHttpException` indicando que el método DELETE no es
compatible con la ruta `lab/admissions/{id}/results`.

El problema es que el formulario o la acción del botón está enviando un request HTTP
con método DELETE cuando debería usar POST o PUT. Esto bloquea la carga de resultados
de pacientes, que es la función principal del laboratorio.

**Error exacto:** `Symfony\Component\HttpKernel\Exception\MethodNotAllowedHttpException`  
**Ruta afectada:** `lab/admissions/{id}/results`  
**Entorno:** producción (`sma.ipac.com.ar`), PHP 8.2.29, Laravel 11.48.0

## PASO 1 — Identificar la ruta y el controlador

Buscar en `routes/web.php` todas las rutas que contengan `lab/admissions` y `results`:

```bash
grep -n "results\|lab.admissions\|lab-admissions" routes/web.php
```

Identificar:
- ¿Qué métodos HTTP acepta la ruta `lab/admissions/{id}/results`?
- ¿Qué controlador y método maneja esa ruta?
- ¿Existe una ruta para store/update de resultados?

**Verificación:** se identificó la ruta y el controlador responsable.

---

## PASO 2 — Encontrar el formulario que envía el request incorrecto

Buscar en las vistas del módulo de laboratorio el formulario o botón que envía
el request a la ruta de resultados:

```bash
grep -rn "results\|@method" resources/views/lab-admissions/
```

Buscar específicamente:
- Un `<form>` con `@method('DELETE')` que debería ser `@method('PUT')` o `@method('POST')`
- Un botón de "Guardar" que tenga la acción incorrecta
- Un link o acción JavaScript que esté enviando DELETE en vez de POST/PUT

**Verificación:** se encontró el formulario/botón con el método incorrecto.

---

## PASO 3 — Corregir el método HTTP

Cambiar el método HTTP del formulario de DELETE al método correcto (POST o PUT según
la ruta definida en web.php).

Si el problema es un `@method('DELETE')`:
- Cambiarlo a `@method('PUT')` o `@method('POST')` según corresponda
- Verificar que la action del form apunte a la ruta correcta

Si el problema es JavaScript (fetch/axios):
- Cambiar el método en la llamada HTTP

**Verificación:** el formulario ahora envía el método HTTP correcto.
