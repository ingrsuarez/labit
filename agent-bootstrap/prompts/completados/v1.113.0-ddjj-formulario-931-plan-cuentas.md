# v1.113.0 — DDJJ Formulario 931 + plan de cuentas aportes/contribuciones patronales

**VERSIÓN:** v1.113.0  
**SLUG:** ddjj-formulario-931-plan-cuentas  
**DEPENDENCIAS:** v3.3.0 (asientos contables), v2.0.0 (multi-empresa)  
**ESTIMACIÓN:** 1 día  
**PRIORIDAD:** alta — prerrequisito del calendario de flujo de caja (v1.113.1)

---

## Descripción

Implementar la **declaración jurada del Formulario 931** (aportes y contribuciones
patronales SUSS/AFIP) por empresa y período mensual, con asiento contable al confirmar.

Hoy Labit no permite cargar el F931. El calendario de vencimientos (v1.113.1) necesita
la **última DDJJ confirmada** como monto estimado en la fecha de vencimiento configurable.

Incluye ampliación del **plan de cuentas** con cuentas específicas de aportes y
contribuciones patronales.

**Handoff PM:** `agent-bootstrap/handoffs/v1.113.0-pm-to-dev.md`

**No incluye:** calendario visual, planes de pago ARCA, pago bancario del 931.

## PASO 3 — Servicio `Form931DeclarationService`

Crear `app/Services/Form931DeclarationService.php`:

- `confirm(Form931Declaration $decl): void` — genera asiento vía `AccountingEntryService`:
  - Db `5.2.11` por `amount_aportes_patronales`
  - Db `5.2.12` por `amount_contribuciones_patronales`
  - Cr `2.1.11` / `2.1.12` respectivamente
  - Descripción: `DDJJ Form 931 {MM/YYYY}`
- `cancel(Form931Declaration $decl): void` — asiento reverso, status `cancelled`
- `latestConfirmed(?int $companyId): ?Form931Declaration` — para v1.113.1
- `latestConfirmedTotal(?int $companyId): ?float` — shortcut

Idempotencia de asientos: mismo patrón que `TaxReturnService` / `fromPayrollPayment`.

## PASO 4 — Controller, rutas, permisos

```bash
php artisan make:controller Form931DeclarationController --resource
```

Rutas bajo middleware auth + permiso `form931.manage`:

- `GET form931-declarations` — index con filtros año/mes/estado
- `GET form931-declarations/create` — formulario período + montos
- `POST form931-declarations` — store borrador
- `GET form931-declarations/{id}` — show con asiento vinculado
- `GET form931-declarations/{id}/edit` — edit solo si draft
- `PUT form931-declarations/{id}` — update draft
- `POST form931-declarations/{id}/confirm` — confirmar
- `POST form931-declarations/{id}/cancel` — anular confirmada

Seeder de permisos (nuevo o extensión de `TaxDeclarationsPermissionsSeeder`):

- `form931.manage` → roles admin, contador

Entrada en hub Compras (`PurchaseSectionController`) o junto a DDJJ impuestos:

- Nombre: **Declaraciones Form 931**
- Descripción: DDJJ mensual SUSS — aportes y contribuciones patronales
