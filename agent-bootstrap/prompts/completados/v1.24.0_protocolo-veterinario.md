## Descripción

Crea el flujo completo de protocolo veterinario: modelos `VetAdmission` y `VetAdmissionTest`,
controlador con CRUD, formulario de creación con datos del animal y selección de determinaciones,
carga de resultados y validación.

El protocolo veterinario sigue el patrón del clínico (`Admission` / `AdmissionTest` /
`LabAdmissionController`) pero adaptado a veterinaria: en vez de paciente humano + obra social,
tiene animal (nombre, especie, raza, edad, dueño) + veterinaria (Customer tipo veterinario)
+ veterinario derivante.

Las determinaciones vienen de la tabla `tests` con `categories: ["veterinario"]`.
Los valores de referencia se resuelven automáticamente desde `test_species_references`
según la especie seleccionada.

Usa numeración `V[AAMMDD][NNNN]` del trait `GeneratesProtocolNumber` (v1.23.0).

## PASO 3 — Modelo VetAdmission

Crear `app/Models/VetAdmission.php`:

```php
<?php

namespace App\Models;

use App\Traits\Auditable;
use App\Traits\GeneratesProtocolNumber;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class VetAdmission extends Model
{
    use Auditable, HasFactory, GeneratesProtocolNumber;

    protected $fillable = [
        'protocol_number', 'date', 'customer_id', 'veterinarian_id',
        'species_id', 'animal_name', 'owner_name', 'owner_phone',
        'breed', 'age', 'status', 'observations', 'total_price', 'created_by',
    ];

    protected $casts = [
        'date' => 'date',
        'total_price' => 'decimal:2',
    ];

    public static function generateProtocolNumber(): string
    {
        return static::generatePrefixedProtocolNumber('V');
    }

    // --- Relaciones ---

    public function customer()
    {
        return $this->belongsTo(Customer::class);
    }

    public function veterinarian()
    {
        return $this->belongsTo(Veterinarian::class);
    }

    public function species()
    {
        return $this->belongsTo(Species::class);
    }

    public function vetTests()
    {
        return $this->hasMany(VetAdmissionTest::class);
    }

    public function creator()
    {
        return $this->belongsTo(User::class, 'created_by');
    }

    // --- Status helpers ---

    public function getStatusLabelAttribute(): string
    {
        return match ($this->status) {
            'pending' => 'Pendiente',
            'in_progress' => 'En Proceso',
            'completed' => 'Completado',
            'cancelled' => 'Cancelado',
            default => $this->status,
        };
    }

    public function getStatusColorAttribute(): string
    {
        return match ($this->status) {
            'pending' => 'yellow',
            'in_progress' => 'blue',
            'completed' => 'green',
            'cancelled' => 'red',
            default => 'gray',
        };
    }

    /**
     * Calcula el estado real basado en las determinaciones
     */
    public function getCalculatedStatusAttribute(): string
    {
        $total = $this->vetTests->count();
        if ($total === 0) return 'pending';

        $validated = $this->vetTests->where('is_validated', true)->count();
        $completed = $this->vetTests->where('status', 'completed')->count();

        if ($validated === $total) return 'validated';
        if ($completed === $total || $validated > 0) return 'completed';
        return 'pending';
    }
}
```

3. Calcular `total_price` y actualizar la admisión

## PASO 11 — Resolución de valores de referencia por especie

Crear un método helper (puede ir en `VetAdmissionController` o como método estático en `VetAdmission`):

```php
/**
 * Resuelve el valor de referencia para un test según la especie.
 * Prioridad: test_species_references → test.low/high → test.other_reference
 */
private function buildReferenceValue(Test $test, int $speciesId): ?string
{
    // 1. Buscar en test_species_references
    $speciesRef = $test->getReferenceForSpecies($speciesId);
    if ($speciesRef) {
        return $speciesRef->formatted_range;
    }

    // 2. Fallback a test.low/high
    if ($test->low || $test->high) {
        $ref = ($test->low ?? '') . ' - ' . ($test->high ?? '');
        if ($test->other_reference) {
            $ref .= ' | ' . $test->other_reference;
        }
        return trim($ref, ' -');
    }

    // 3. Fallback a other_reference solo
    if ($test->other_reference) {
        return $test->other_reference;
    }

    return null;
}
```

Este método se usa en `store()` al crear cada `VetAdmissionTest`.

## Notas / Consideraciones

- **No crear PDF ni email en esta versión** — va en v1.25.0.
- **No crear etiquetas** — se puede agregar en un patch posterior.
- El campo `age` es string (no integer) porque el usuario puede poner "3 años", "6 meses", "1 año y medio".

- La vista `create` debe ser responsive y consistente con el diseño del proyecto (Tailwind, teal para lab, amber para sección veterinaria).
- Los permisos se pueden reutilizar del lab por ahora. Si se necesitan permisos específicos de veterinario, se agregan en un patch.
- El `vet.customer.veterinarians` endpoint retorna JSON y se consume con fetch desde Alpine.js en el formulario.
