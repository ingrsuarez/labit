## Descripción

En producción, el envío masivo desde `/sample` falla con modal vacío y alerta *"No hay protocolos para enviar"*, aunque el protocolo muestra badge **Validado** y el checkbox está habilitado.

**Causa:** desincronización entre criterios:
- Checkbox / badge → `calculated_status === 'validated'`
- Modal JS (`sampleMeta.is_validated`) y backend `batchEmail` → `Sample::isValidated()` que lee `validation_status === 'validated'`

Protocolos validados antes de v1.102.0 o sin pasar por `updateSampleValidationStatus()` quedan con `calculated_status = validated` pero `validation_status ≠ validated`.

Esta versión:
1. Unifica el criterio de envío masivo en `calculated_status`
2. Agrega envío **individual desde el listado** (paridad `lab/admissions/index` → link Email + `?open_email=1`)
3. Pulido UX del modal masivo (chips multi-email por cliente, mensaje opcional, defaults correctos)

**Destino siempre empresa/cliente** — no hay email de paciente en muestras.

**Fuera de scope:** Space10, filtrar por etiqueta Resultados/Facturación, modal inline en listado sin navegar al detalle.
