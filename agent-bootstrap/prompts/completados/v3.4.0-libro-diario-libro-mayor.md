# v3.4.0 — Libro Diario y Libro Mayor

**VERSIÓN:** v3.4.0  
**SLUG:** libro-diario-libro-mayor  
**DEPENDENCIAS:** v3.3.0 (asientos automáticos desde transacciones)  
**ESTIMACIÓN:** medio día  
**PRIORIDAD:** alta

---

## Descripción

Tres funcionalidades de consulta y carga contable que cierran el ciclo básico del módulo:

1. **Libro Diario** — vista cronológica de todos los asientos con líneas expandibles,
   filtros y link al documento de origen
2. **Libro Mayor** — detalle de movimientos de una cuenta con saldo acumulado por período
3. **Asiento manual** — formulario para registrar ajustes contables que el sistema no genera

Los asientos automáticos (generados por v3.3.0) son de solo lectura en estas vistas.
Solo los asientos manuales pueden editarse o eliminarse.

### `index(Request $request)` — Libro Diario

```php
public function index(Request $request)
{
    $this->authorize('contabilidad.entries.index');

    $company = session('selected_company_id') ?? auth()->user()->companies()->first()?->id;

    $query = JournalEntry::with(['lines.account', 'creator'])
        ->where('company_id', $company)
        ->orderByDesc('date')
        ->orderByDesc('number');

    // Filtros
    if ($request->filled('date_from')) {
        $query->whereDate('date', '>=', $request->date_from);
    }
    if ($request->filled('date_to')) {
        $query->whereDate('date', '<=', $request->date_to);
    }
    if ($request->filled('type')) {
        $query->where('is_automatic', $request->type === 'automatic');
    }
    if ($request->filled('search')) {
        $query->where('description', 'like', "%{$request->search}%");
    }

    $entries = $query->paginate(20)->withQueryString();

    return view('accounting.journal.index', compact('entries'));
}
```

### `store(Request $request)` — Guardar asiento manual

```php
public function store(Request $request)
{
    $this->authorize('contabilidad.entries.create');

    $request->validate([
        'date'                        => 'required|date',
        'description'                 => 'required|string|max:500',
        'lines'                       => 'required|array|min:2',
        'lines.*.accounting_account_id' => 'required|exists:accounting_accounts,id',
        'lines.*.debit'               => 'nullable|numeric|min:0',
        'lines.*.credit'              => 'nullable|numeric|min:0',
        'lines.*.description'         => 'nullable|string|max:255',
    ]);

    $lines = collect($request->lines)->map(fn($l) => [
        'accounting_account_id' => $l['accounting_account_id'],
        'debit'                 => (float) ($l['debit'] ?? 0),
        'credit'                => (float) ($l['credit'] ?? 0),
        'description'           => $l['description'] ?? null,
    ]);

    // Validar partida doble
    $totalDebit  = $lines->sum('debit');
    $totalCredit = $lines->sum('credit');

    if (abs($totalDebit - $totalCredit) > 0.01) {
        return back()->withInput()
            ->withErrors(['lines' => "El asiento no está balanceado: Debe $" . number_format($totalDebit, 2) . " ≠ Haber $" . number_format($totalCredit, 2)]);
    }

    $companyId = session('selected_company_id') ?? auth()->user()->companies()->first()?->id;
    $date = \Carbon\Carbon::parse($request->date);

    \DB::transaction(function () use ($request, $lines, $companyId, $date) {
        $entry = JournalEntry::create([
            'company_id'   => $companyId,
            'date'         => $date,
            'number'       => JournalEntry::nextNumber($companyId, $date->year),
            'description'  => $request->description,
            'is_automatic' => false,
            'created_by'   => auth()->id(),
        ]);

        foreach ($lines as $line) {
            JournalEntryLine::create(array_merge($line, ['journal_entry_id' => $entry->id]));
        }
    });

    return redirect()->route('accounting.journal.index')
        ->with('success', 'Asiento registrado correctamente.');
}
```

### `update(Request $request, JournalEntry $journalEntry)` — Actualizar asiento manual

Mismo proceso que `store()`, pero reemplazar las líneas existentes dentro de una transacción:

```php
public function update(Request $request, JournalEntry $journalEntry)
{
    $this->authorize('contabilidad.entries.edit');

    if ($journalEntry->is_automatic) {
        return back()->with('error', 'Los asientos automáticos no se pueden editar.');
    }

    // Misma validación que store()
    // ...

    \DB::transaction(function () use ($request, $lines, $journalEntry, $date) {
        $journalEntry->lines()->delete();
        $journalEntry->update([
            'date'        => $date,
            'description' => $request->description,
        ]);
        foreach ($lines as $line) {
            JournalEntryLine::create(array_merge($line, ['journal_entry_id' => $journalEntry->id]));
        }
    });

    return redirect()->route('accounting.journal.index')
        ->with('success', 'Asiento actualizado.');
}
```

### `destroy(JournalEntry $journalEntry)` — Eliminar asiento manual

```php
public function destroy(JournalEntry $journalEntry)
{
    $this->authorize('contabilidad.entries.delete');

    if ($journalEntry->is_automatic) {
        return back()->with('error', 'Los asientos automáticos no se pueden eliminar.');
    }

    $journalEntry->lines()->delete();
    $journalEntry->delete();

    return redirect()->route('accounting.journal.index')
        ->with('success', 'Asiento eliminado.');
}
```

## PASO 3 — Controlador del Libro Mayor

Crear `app/Http/Controllers/AccountingLedgerController.php`

```php
class AccountingLedgerController extends Controller
{
    public function index(Request $request)
    {
        $this->authorize('contabilidad.ledger.index');

        $companyId = session('selected_company_id') ?? auth()->user()->companies()->first()?->id;
        $accounts  = AccountingAccount::active()->imputable()->orderBy('code')->get(['id', 'code', 'name']);

        $movements   = collect();
        $account     = null;
        $openBalance = 0;
        $closeBalance = 0;

        if ($request->filled('account_id')) {
            $account = AccountingAccount::findOrFail($request->account_id);

            // Período: mes/año (default: mes actual)
            $year  = $request->integer('year', now()->year);
            $month = $request->integer('month', now()->month);

            $periodStart = \Carbon\Carbon::create($year, $month, 1)->startOfMonth();
            $periodEnd   = $periodStart->copy()->endOfMonth();

            // Saldo inicial: suma de todos los movimientos anteriores al período
            $before = JournalEntryLine::where('accounting_account_id', $account->id)
                ->whereHas('journalEntry', fn($q) => $q
                    ->where('company_id', $companyId)
                    ->whereDate('date', '<', $periodStart)
                )
                ->selectRaw('SUM(debit) as total_debit, SUM(credit) as total_credit')
                ->first();

            // Para cuentas de activo/resultado negativo: saldo = debit - credit
            // Para pasivo/PN/resultado positivo: saldo = credit - debit
            $deudora = in_array($account->type, ['activo', 'resultado_negativo']);
            $openBalance = $deudora
                ? ($before->total_debit - $before->total_credit)
                : ($before->total_credit - $before->total_debit);

            // Movimientos del período
            $movements = JournalEntryLine::where('accounting_account_id', $account->id)
                ->whereHas('journalEntry', fn($q) => $q
                    ->where('company_id', $companyId)
                    ->whereBetween('date', [$periodStart, $periodEnd])
                )
                ->with(['journalEntry' => fn($q) => $q->select('id', 'date', 'number', 'description', 'source_type', 'source_id', 'is_automatic')])
                ->orderBy(fn($q) => $q->select('date')->from('journal_entries')->whereColumn('journal_entries.id', 'journal_entry_lines.journal_entry_id'))
                ->get();

            // Ordenar por fecha del asiento y calcular saldo acumulado
            $movements = $movements->sortBy('journalEntry.date');
            $running = $openBalance;
            $movements = $movements->map(function ($line) use ($deudora, &$running) {
                $delta = $deudora ? ($line->debit - $line->credit) : ($line->credit - $line->debit);
                $running += $delta;
                $line->running_balance = $running;
                return $line;
            });

            $closeBalance = $running;
        }

        return view('accounting.ledger.index', compact(
            'accounts', 'account', 'movements', 'openBalance', 'closeBalance',
            'request'
        ));
    }
}
```

**Nota sobre el orderBy:** el `orderBy` de líneas por fecha del asiento requiere un join o
un sortBy en PHP. Usar `sortBy('journalEntry.date')` en la colección ya cargada es más sencillo
y suficiente para volúmenes normales de contabilidad.

**Componente Alpine `journalForm()`:**

```javascript
function journalForm(initialLines = null) {
    return {
        lines: initialLines || [
            { account_id: '', description: '', debit: 0, credit: 0 },
            { account_id: '', description: '', debit: 0, credit: 0 },
        ],

        get totalDebit()  { return this.lines.reduce((s, l) => s + (parseFloat(l.debit) || 0), 0); },
        get totalCredit() { return this.lines.reduce((s, l) => s + (parseFloat(l.credit) || 0), 0); },
        get isBalanced()  { return Math.abs(this.totalDebit - this.totalCredit) < 0.01; },

        addLine() {
            this.lines.push({ account_id: '', description: '', debit: 0, credit: 0 });
        },

        removeLine(index) {
            if (this.lines.length > 2) this.lines.splice(index, 1);
        },
    };
}
```

## Notas / Consideraciones

- El Libro Mayor calcula el saldo en PHP (no en SQL) para simplicidad y portabilidad.
  Para volúmenes muy grandes podría optimizarse con una query window function en MySQL 8+.
- La naturaleza del saldo (deudora/acreedora) se determina por `account->type`:
  activo y resultado_negativo son deudoras (debit - credit);
  pasivo, patrimonio_neto y resultado_positivo son acreedoras (credit - debit).
- Los asientos manuales se usan para: apertura de ejercicio, ajustes de inventario,
  correcciones, depreciaciones, etc. No hay restricción de qué cuentas se usan.
- En una versión futura se puede agregar export a PDF (para presentación a contador externo)
  y el Balance de Sumas y Saldos / Estado de Resultados.
