# v1.32.4 — Editar y eliminar remitos con sincronización de stock

**ESTIMACIÓN:** 2h  
**PRIORIDAD:** alta

## Descripción

Los remitos (delivery notes) permiten actualmente la creación y consulta, pero no edición
ni eliminación. Esta versión habilita ambas operaciones con las siguientes reglas:

**Eliminar un remito:**
- Solo permitido si el remito **no tiene factura de compra asociada**.
- Al eliminar, se revierten los movimientos de stock generados por ese remito
  (se genera un movimiento de salida por cada ítem que había generado entrada, o se eliminan
  directamente los movimientos si aún no impactan otros documentos).
- Si tiene factura asociada, el botón de eliminar está deshabilitado con tooltip explicativo.

**Editar un remito:**
- Solo permitido si el remito **no tiene factura de compra asociada**.
- Al guardar los cambios, se sincroniza el stock:
  - Ítems eliminados: revertir su movimiento de stock.
  - Ítems modificados en cantidad: ajustar el movimiento de stock con la diferencia.
  - Ítems nuevos: crear nuevo movimiento de stock de entrada.
- Si tiene factura asociada, el formulario de edición muestra un aviso y es de solo lectura
  (o no se puede acceder directamente).

## PASO 2 — Crear `DeliveryNoteStockService`

Crear `app/Services/DeliveryNoteStockService.php` que centralice la lógica de stock:

```php
<?php

namespace App\Services;

use App\Models\DeliveryNote;
use App\Models\StockMovement;

class DeliveryNoteStockService
{
    /**
     * Revierte todos los movimientos de stock de un remito (para delete).
     * Elimina directamente los StockMovements cuyo source sea este remito.
     */
    public function reverseStockForDeletion(DeliveryNote $deliveryNote): void
    {
        StockMovement::where('source_type', DeliveryNote::class)
            ->where('source_id', $deliveryNote->id)
            ->delete();
    }

    /**
     * Sincroniza el stock al editar un remito.
     * Compara ítems viejos vs nuevos y ajusta movimientos.
     *
     * @param DeliveryNote $deliveryNote  El remito ya actualizado en DB
     * @param array        $oldItems      Colección de ítems ANTES de la edición
     *                                   [['supply_id' => x, 'quantity' => y], ...]
     */
    public function syncStockAfterUpdate(DeliveryNote $deliveryNote, array $oldItems): void
    {
        // Indexar items viejos por supply_id
        $oldMap = collect($oldItems)->keyBy('supply_id');

        // Ítems actuales (ya guardados)
        $newItems = $deliveryNote->items()->get(['supply_id', 'quantity']);

        // Eliminar movimientos del remito y recrear desde cero
        // (más simple y seguro que calcular diferencias)
        StockMovement::where('source_type', DeliveryNote::class)
            ->where('source_id', $deliveryNote->id)
            ->delete();

        foreach ($newItems as $item) {
            if ($item->quantity > 0) {
                StockMovement::create([
                    'supply_id'   => $item->supply_id,
                    'quantity'    => $item->quantity,
                    'type'        => 'in',
                    'reason'      => 'delivery_note',
                    'source_type' => DeliveryNote::class,
                    'source_id'   => $deliveryNote->id,
                    'notes'       => "Remito #{$deliveryNote->id} (editado)",
                ]);
            }
        }
    }
}
```

> **Nota al agente:** Verificar el campo `reason` en `StockMovement`. Si usa un enum, usar
> el valor correcto que corresponde a remitos (ej: `'delivery_note'`, `'purchase'`, etc.).
> Si no hay un `reason` separado y se usa solo `type`, adaptar según el modelo existente.
> También verificar si `StockMovement` requiere `company_id` u otros campos obligatorios.

## PASO 3 — Actualizar `DeliveryNoteController`

### 3.1 — Método `destroy()`

```php
public function destroy(DeliveryNote $deliveryNote)
{
    if ($deliveryNote->hasPurchaseInvoice()) {
        return back()->with('error',
            'No se puede eliminar el remito porque tiene una factura de compra asociada.');
    }

    DB::transaction(function () use ($deliveryNote) {
        // Revertir stock
        app(\App\Services\DeliveryNoteStockService::class)
            ->reverseStockForDeletion($deliveryNote);

        // Eliminar ítems y el remito
        $deliveryNote->items()->delete();
        $deliveryNote->delete();
    });

    return redirect()->route('delivery-notes.index')
        ->with('success', 'Remito eliminado y stock revertido correctamente.');
}
```
