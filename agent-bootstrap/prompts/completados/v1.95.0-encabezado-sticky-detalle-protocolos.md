## Causa raíz del fallo inicial

`position: sticky` respecto al **viewport** con scroll en `document` + columna flex sin altura acotada suele ser frágil. Aquí el scroll efectivo debe ocurrir en **`<main>`** para que el encabezado de protocolo tenga un **ancestro con overflow estable** y `sticky top-*` funcione de forma predecible.

## Implementación final

1. **`resources/views/components/lab-layout.blade.php`**
   - `body`: `flex min-h-screen min-h-dvh flex-col overflow-hidden` (columna app + banner).
   - Contenedor shell: `flex min-h-0 min-w-0 flex-1 overflow-hidden`.
   - Columna derecha: `flex min-h-0 min-w-0 flex-1 flex-col md:ml-64`.
   - **`<main>`**: `min-h-0 min-w-0 flex-1 overflow-y-auto` → **único scroll vertical** del área de trabajo del lab.

2. **`resources/views/lab/partials/header.blade.php`**
   - Sustituir `sticky top-0` por **`shrink-0`**: el header del lab ya no depende del scroll del documento; queda fijo en la columna encima de `<main>`.

3. **Cabeceras de protocolo** (`lab/admissions/show`, `vet/admissions/show`, `sample/show`):
   - Contenedor: `sticky top-14 z-20 … md:top-0` + fondo `bg-gray-100`, `border-b`, `shadow-sm`.
   - **`top-14`**: barra móvil fija del lab (`sidebar` partial, `z-50`).
   - **`md:top-0`**: bajo el header de escritorio (fuera de `<main>`), el tope del scrollport de `main` ya deja el hueco correcto.

4. **`sample/show`**: `mt-14 md:mt-0` en el wrapper de página (alineado a clínico/vet).

5. **Assets:** `npm run build` para incluir utilidades nuevas (`min-h-dvh`, etc.) en `public/build/`.
