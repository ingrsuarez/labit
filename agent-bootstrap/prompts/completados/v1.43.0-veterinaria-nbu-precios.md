# v1.43.0 — Precios protocolo veterinario: NBU veterinaria × NBU determinación

**DEPENDENCIAS:** v1.42.1 (completada en repo / tag)

## Objetivo

- Cada cliente tipo **veterinario** tiene un **valor monetario de 1 NBU** configurable (`customers.veterinary_nbu_value`).
- En alta de protocolo veterinario, el precio de cada práctica padre = `veterinary_nbu_value × tests.nbu` (redondeo 2 decimales).
- El backend **recalcula** el precio al guardar (no confiar solo en el POST).
- `search-tests` exige `customer_id` y devuelve `price`, `nbu` por ítem.

## PASO 1 — Migración y modelo

- Migración `veterinary_nbu_value` en `customers` (decimal 12,2 nullable).
- `Customer`: fillable, cast, método `veterinaryNbuRate()`.

## PASO 2 — Clientes (admin)

- `CustomerController` store/update: validar `veterinary_nbu_value` nullable numeric min 0.
- Vistas `customer/create` y `customer/edit`: campo visible si tipo incluye veterinario; `@change` para sincronizar `hasVetType` en create.

## PASO 3 — Veterinaria

- `VetAdmissionController::create`: pasar `customerNbuValues` a la vista.
- `searchTests`: filtrar por `customer_id`, calcular precio con `veterinaryPriceFromNbu`.
- `store`: precio server-side desde cliente + test.
- Vista `vet/admissions/create`: enviar `customer_id` en búsqueda; manejar error 422; guardar `nbu` en filas; al cambiar veterinaria, recalcular precios de filas ya agregadas.

## PASO 4 — Verificación

- `php artisan migrate`
- `php vendor\bin\pint --test`
- `php artisan test`
- Test Feature mínimo (auth + ruta search-tests con customer y test vet) si el proyecto ya usa Pest para HTTP.
