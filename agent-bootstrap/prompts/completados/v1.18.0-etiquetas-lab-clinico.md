## Descripción

Agrega impresión de etiquetas Zebra para tubos de pacientes en el módulo de laboratorio clínico.
Un botón "Imprimir Etiqueta" en la vista show de admisiones abre el mismo modal Zebra que ya
funciona en muestras, genera ZPL con barcode del protocolo, nombre del paciente y materiales
de los tests del protocolo.

Reutiliza toda la infraestructura de v1.17.0 (`zebra-label-print.js`, componente Alpine
`zebraPrintModal()`) sin modificarla. Solo agrega endpoints, permiso, botón y vista fallback.

## PASO 2 — Métodos en `LabAdmissionController`

En `app/Http/Controllers/LabAdmissionController.php`, agregar dos métodos:

### 2.1 — `labelData(Admission $admission)`

Devuelve JSON con los datos necesarios para generar la etiqueta ZPL.
Replicar la misma estructura que `SampleController::labelData()` pero adaptada a admisiones:

```php
public function labelData(Admission $admission)
{
    $this->authorize('lab-labels.print');

    $admission->load(['patient', 'admissionTests.test']);

    $materials = $admission->admissionTests
        ->pluck('test.material_abbreviation')
        ->unique()
        ->filter()
        ->implode('/');

    return response()->json([
        'protocol_number' => $admission->protocol_number,
        'customer_name' => $admission->patient->full_name ?? $admission->patient->name ?? 'N/A',
        'materials' => $materials ?: 'N/A',
        'sample_type' => 'CLINICO',
        'entry_date' => $admission->date->format('d/m/Y'),
    ]);
}
```

IMPORTANTE: Las claves del JSON deben ser exactamente las mismas que usa `SampleController::labelData()`
(`protocol_number`, `customer_name`, `materials`, `sample_type`, `entry_date`) porque `generateZPL()`
en el JS espera esos campos. Verificar en `zebra-label-print.js` método `generateZPL(data)` qué
campos usa y respetarlos.

> Investigar cuál es el campo correcto para el nombre del paciente en el modelo `Patient`.
> Puede ser `name`, `full_name`, o `first_name` + `last_name`. Verificar en el modelo.

### 2.2 — `printLabel(Admission $admission)`

Vista HTML fallback para impresoras no-Zebra (mismo patrón que `SampleController::printLabel`):

```php
public function printLabel(Admission $admission)
{
    $this->authorize('lab-labels.print');

    $admission->load(['patient', 'admissionTests.test']);

    $materials = $admission->admissionTests
        ->pluck('test.material_abbreviation')
        ->unique()
        ->filter()
        ->implode('/');

    $barcode = new \Picqer\Barcode\BarcodeGeneratorSVG;
    $barcodeSvg = $barcode->getBarcode(
        $admission->protocol_number,
        $barcode::TYPE_CODE_128,
        2,
        60
    );

    return view('lab.admissions.label', [
        'admission' => $admission,
        'materials' => $materials ?: 'N/A',
        'barcodeSvg' => $barcodeSvg,
    ]);
}
```

**Verificación:** `GET lab/admissions/{id}/label-data` devuelve JSON correcto. `GET lab/admissions/{id}/label` renderiza la etiqueta HTML.

## Notas / Consideraciones

- El campo `customer_name` en el JSON se llama así (no `patient_name`) para mantener compatibilidad
  con `generateZPL()` que espera ese nombre de campo. Es un detalle de interfaz, no de dominio.
- Los materiales se obtienen del accessor `material_abbreviation` del modelo Test que mapea:
  1=EDTA, 2=SUE, 3=ORI, 4=CIT, 5=HEP, 6=FE, 7=BV, 8=HEM.
- Si una admisión tiene tests con distintos materiales (ej: hemograma=EDTA + glucemia=SUE),
  la etiqueta mostrará "EDTA/SUE". En una versión futura se podría imprimir una etiqueta
  por cada material diferente (una para el tubo EDTA, otra para el de suero).
- El modal Zebra y el JS se reutilizan sin cambios — la genericidad del diseño de v1.17.0 lo permite.
- No se agrega link al sidebar del lab — la impresión se accede desde la vista individual de cada admisión.
