# v1.80.0 — Nombre corto / sigla en clientes y obras sociales

**Tipo:** minor — nueva capacidad funcional  
**Prioridad:** alta — problema visual activo en producción  
**Dependencias:** ninguna

---

## Qué hace

Agrega el campo `short_name` (nullable) al modelo `Customer` para que obras sociales y clientes con nombres muy largos puedan tener una sigla o nombre abreviado de uso interno.

**Regla de visualización:** si el cliente tiene `short_name` cargado, se usa ese en lugar del `name` en todas las vistas donde el espacio es limitado (tablas, comboboxes, encabezados de lote). Si no tiene `short_name`, se sigue usando `name` como hasta ahora.

**Accessor en modelo:** `displayName()` → `$this->short_name ?? $this->name`

Alcance: formularios de cliente, listado de admisiones lab clínico, combobox de obra social en admisión, facturación masiva, cuenta corriente de clientes, PDFs de protocolos clínicos/vet/muestras y emails.

---

## Modelo de datos

### Migración

Crear migración:

```php
$table->string('short_name')->nullable()->after('name');
```

Tabla: `customers`

### Modelo `app/Models/Customer.php`

1. Agregar `'short_name'` al array `$fillable`.
2. Agregar accessor:

```php
public function displayName(): string
{
    return $this->short_name ?? $this->name;
}
```

```html

<input type="text" name="short_name" value="{{ old('short_name', $customer->short_name ?? '') }}"
```

## Vistas — Combobox de obra social en admisión

Archivo: `resources/views/lab/admissions/create.blade.php`

El combobox Alpine.js de selección de obra social muestra `$ins->name`. Cambiar a `$ins->displayName()` en el texto visible. La búsqueda debe seguir funcionando por `name` Y por `short_name` (buscar en ambos campos).

Si la búsqueda es client-side (Alpine filtrando JSON cargado en la página), asegurarse de incluir `short_name` en el objeto JS por cliente y filtrar por ambos campos. Si es server-side, ajustar el query para buscar en ambas columnas:

```php
->where(function ($q) use ($search) {
    $q->where('name', 'like', "%{$search}%")
      ->orWhere('short_name', 'like', "%{$search}%");
})
```

Aplica también a `resources/views/lab/admissions/edit.blade.php` si tiene el mismo combobox.

## Verificación

1. Crear un cliente con nombre largo (ej: "OBRA SOCIAL DE CONDUCTORES CAMIONEROS...") sin `short_name` → se muestra el nombre largo en todos lados (sin regresión).
2. Editar ese cliente y cargar `short_name` = "OSCHOCA" → guardar → verificar que se muestra "OSCHOCA" en listado de admisiones, combobox, facturación masiva y PDFs.
3. Buscar en el combobox por "OSCHOCA" (nombre corto) → debe encontrar la obra social.
4. Buscar en el combobox por parte del nombre largo → también debe encontrarla.
5. Clientes con `short_name = null` siguen mostrando `name` sin ningún cambio visual.
