## Descripción

usa actualmente un `<select>` simple para elegir el insumo de cada ítem. Reemplazarlo
por el mismo combobox con búsqueda inteligente que se implementó en facturas de compra (v1.32.0):
búsqueda por nombre/código, Tab flow, badge del insumo vinculado.

El estado Alpine de cada fila de ítem necesita:

### 2.1 — Columna "Insumo" en la tabla de ítems

Reemplazar el `<th>` y el `<td>` con el `<select>` por:

```html
<!-- th -->
<th class="px-3 py-2 text-left text-xs font-semibold text-gray-500 uppercase tracking-wider">Insumo</th>

<!-- td -->
<td class="px-3 py-2 relative" x-data="supplySearch(item, index)">
    <input type="hidden" :name="'items[' + index + '][supply_id]'" :value="item.supply_id || ''">

    <input type="text"
           x-model="searchText"
           @input.debounce.300ms="doSearch()"
           @focus="if (searchText.length >= 2) showResults = true"
           @keydown.arrow-down.prevent="highlightNext()"
           @keydown.arrow-up.prevent="highlightPrev()"
           @keydown.enter.prevent="selectHighlighted()"
           @keydown.escape="showResults = false"
           @keydown.tab="onTab($event)"
           placeholder="Buscar insumo..."
           required
           :id="'item-supply-' + index"
           class="w-full rounded border-gray-300 text-sm focus:border-zinc-500 focus:ring-zinc-500">

    <!-- Badge del insumo vinculado -->
    <template x-if="item.supply_id">
        <div class="flex items-center gap-1 mt-1">
            <span class="inline-flex items-center px-2 py-0.5 rounded text-xs font-medium bg-teal-50 text-teal-700 border border-teal-200">
                <span x-text="item._supply_code || ''"></span>
                <button type="button" @click="unlinkSupply()" class="ml-1 text-teal-400 hover:text-teal-600">&times;</button>
            </span>
        </div>
    </template>

    <!-- Dropdown de resultados -->
    <div x-show="showResults && results.length > 0" x-cloak
         @click.outside="showResults = false"
         class="absolute z-50 left-0 right-0 mt-1 bg-white border border-gray-200 rounded-lg shadow-lg max-h-60 overflow-y-auto">

        <template x-for="(supply, i) in results" :key="supply.id">
            <button type="button"
                    @click="selectSupply(supply)"
                    @mouseenter="highlightedIndex = i"
                    :class="highlightedIndex === i ? 'bg-teal-50 text-teal-800' : 'text-gray-700'"
                    class="w-full text-left px-3 py-2 text-sm hover:bg-gray-50 flex items-center gap-2 border-b border-gray-100 last:border-0">
                <span class="font-mono text-xs text-gray-400" x-text="supply.code"></span>
                <span x-text="supply.name"></span>
            </button>
        </template>

        <div x-show="results.length === 0 && searchText.length >= 2 && !loading"
             class="px-3 py-2 text-sm text-gray-400">
            No se encontraron insumos.
        </div>
    </div>
</td>
```

### 2.2 — Función `supplySearch()` adaptada para remitos

En el remito el ítem tiene `supply_id` y `quantity` (y opcionalmente `lot_number`, `notes`).
Agregar la función `supplySearch()` al `<script>` de la vista, adaptada para el contexto de remito
(sin precio, sin `tracks_lot` visual — los remitos no muestran precio):

```javascript
function supplySearch(item, index) {
    return {
        searchText: item._supply_name || '',
        results: [],
        showResults: false,
        loading: false,
        highlightedIndex: -1,

        async doSearch() {
            if (this.searchText.length < 2) {
                this.results = [];
                this.showResults = false;
                return;
            }
            this.loading = true;
            try {
                const resp = await fetch(`{{ route('supplies.search') }}?q=${encodeURIComponent(this.searchText)}`);
                this.results = await resp.json();
                this.showResults = true;
                this.highlightedIndex = -1;
            } catch (e) {
                this.results = [];
            } finally {
                this.loading = false;
            }
        },

        selectSupply(supply) {
            item.supply_id = supply.id;
            item._supply_name = supply.name;
            item._supply_code = supply.code;
            this.searchText = supply.name;
            this.showResults = false;
            this.$nextTick(() => {
                const qtyInput = document.querySelector(`#item-qty-${index}`);
                if (qtyInput) qtyInput.focus();
            });
        },

        unlinkSupply() {
            item.supply_id = '';
            item._supply_code = '';
            item._supply_name = '';
            this.searchText = '';
        },

        highlightNext() {
            if (this.highlightedIndex < this.results.length - 1) this.highlightedIndex++;
        },

        highlightPrev() {
            if (this.highlightedIndex > 0) this.highlightedIndex--;
        },

        selectHighlighted() {
            if (this.highlightedIndex >= 0 && this.highlightedIndex < this.results.length) {
                this.selectSupply(this.results[this.highlightedIndex]);
            }
        },

        onTab(event) {
            if (this.showResults && this.highlightedIndex >= 0) {
                event.preventDefault();
                this.selectHighlighted();
            } else {
                this.showResults = false;
            }
        }
    }
}
```

**Verificación:** Tipear en el input muestra dropdown, seleccionar vincula el insumo, Tab pasa a Cantidad.

## Notas / Consideraciones

- En remitos **no se muestra precio** — el combobox solo vincula supply_id, quantity y notas.
- No se agrega opción "Crear insumo" en remitos (a diferencia de facturas de compra) porque
  los remitos registran recepción de insumos que ya deberían existir en el sistema.
- El endpoint `supplies.search` fue creado en v1.32.0 y se reutiliza sin modificación.
