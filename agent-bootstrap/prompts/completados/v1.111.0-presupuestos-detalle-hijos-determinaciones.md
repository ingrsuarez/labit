## Descripción

En el módulo de **Presupuestos** (`/quotes`), al agregar un paquete padre del nomenclador de aguas/alimentos (ej. *Fisicoquímico básico de agua potable*), hoy solo se muestra y guarda **una línea plana** con descripción, cantidad y precio. Las determinaciones hijas (pH, turbidez, temperatura, sulfatos, etc.) existen en el nomenclador (`test_parents`, `Test::getAllChildren()`) pero **no aparecen** al armar el presupuesto ni en el PDF (descarga o email).

**Objetivo:** Mostrar la jerarquía completa de hijos como sub-ítems **informativos** (sin cantidad ni precio propio), con indentación progresiva por nivel (padre → sub-padre → hoja). Al **guardar** el presupuesto, congelar un **snapshot** de esa jerarquía para que presupuestos ya enviados no cambien si se edita el nomenclador después.

**Decisiones PM acordadas:**
- Formato: hijos informativos indentados; solo el padre tiene cantidad/precio/total.
- Profundidad: todos los niveles si existen sub-padres.
- Persistencia: snapshot congelado al store/update.
- Designer: no — reutilizar patrón visual de protocolos de muestras (`sample/pdf*.blade.php`).

**Fuera de alcance:** precio por hijo; editar/quitar hijos individualmente; facturación; protocolos.

## PASO 1 — Migración y modelo: snapshot de hijos

1. Crear migración que agregue a `quote_items`:

   ```php
   $table->json('children_snapshot')->nullable();
   ```

2. Formato acordado del JSON (lista plana en orden de visualización, pre-order):

   ```json
   [
     { "test_id": 123, "name": "Grupo físico-químico", "depth": 1 },
     { "test_id": 456, "name": "pH", "depth": 2 },
     { "test_id": 457, "name": "Turbidez", "depth": 2 }
   ]
   ```

   - `depth`: 1 = hijo directo del ítem padre del presupuesto; 2+ = niveles anidados.
   - `name`: texto mostrado (código + nombre o solo nombre; ser consistente con descripción del padre).
   - `test_id`: nullable por robustez futura; incluir cuando exista.

3. En `QuoteItem`: cast `'children_snapshot' => 'array'`, agregar a `$fillable`.

4. Crear servicio `App\Services\QuoteItemChildrenSnapshotBuilder` (o nombre equivalente) que:
   - Recibe un `Test` padre.
   - Recorre la jerarquía (direct children vía `childTests` ordenados por pivot `order`, recursivo para sub-padres).
   - Devuelve el array plano con `depth` correcto.
   - Reutilizar lógica de orden ya usada en muestras/vet si existe helper compartido; si no, implementación mínima local.

**Verificación:** tinker — dado un test padre conocido del seeder de aguas, el builder devuelve lista con depth > 1 cuando hay sub-padres.

## PASO 2 — Backend: persistir snapshot en store/update/duplicate

1. Extender validación en `QuoteController@store` y `@update`:

   ```php
   'items.*.children_snapshot' => 'nullable|array',
   'items.*.children_snapshot.*.name' => 'required_with:items.*.children_snapshot|string|max:255',
   'items.*.children_snapshot.*.depth' => 'required_with:items.*.children_snapshot|integer|min:1',
   'items.*.children_snapshot.*.test_id' => 'nullable|integer|exists:tests,id',
   ```

2. Al crear/actualizar cada `QuoteItem`:
   - Si el request trae `children_snapshot`, persistirlo tal cual (sanitizado).
   - Si el ítem tiene `test_id` y **no** trae snapshot (edge case), generarlo server-side con el builder antes de guardar.

3. En `duplicate`: el `replicate()` del ítem debe copiar `children_snapshot` (verificar que no quede excluido).

4. Agregar método en `QuoteItem` o el builder:

   ```php
   public function resolvedChildren(): array
   ```

   - Si `children_snapshot` no está vacío → devolverlo.
   - Si está vacío y hay `test_id` → fallback dinámico con el builder (retrocompatibilidad presupuestos viejos).
   - Si no hay test → `[]`.

**Verificación:** crear presupuesto vía HTTP con padre; en BD `quote_items.children_snapshot` tiene JSON no nulo.

## Notas / Consideraciones

- **No persistir hijos como filas separadas en `quote_items`** — un solo registro padre con JSON evita inflar totales/subtotal.
- **Recalculate/subtotal:** solo sumar filas padre; hijos nunca entran al cálculo.
- **searchTests** hoy filtra `whereNull('parent')` — tests que solo existen en pivot `test_parents` podrían no aparecer; no cambiar ese criterio salvo bug evidente (fuera de scope).
- **Duplicar presupuesto** debe copiar snapshot para mantener paridad con el original.
- Reutilizar indentación visual existente en muestras; no invocar AGENTE_DESIGNER.
