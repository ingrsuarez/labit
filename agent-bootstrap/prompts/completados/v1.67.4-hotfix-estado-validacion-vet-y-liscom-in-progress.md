# v1.67.4 — Hotfix: estado validación vet + transición in_progress por LISCOM

**Tipo:** patch / hotfix
**Prioridad:** inmediata — bugs en flujo activo de validación y de integración LISCOM
**Dependencias:** v1.47.0 (API endpoints), v1.67.3 (último hotfix vet)

---

## Problema A — Estado "validated" no se persiste al validar determinaciones vet

Al validar una determinación veterinaria (botón Validar / Validar Todo), el campo
`VetAdmissionTest.status` no se actualiza: queda en `'completed'` o `'pending'`.
Tampoco se actualiza `VetAdmission.status` (el protocolo padre).
El accessor `getCalculatedStatusAttribute()` ya calcula el estado correcto pero
nunca persiste el resultado en la columna `status`.

## Problema B — Protocolos consultados por LISCOM no pasan a "in_progress"

Cuando LISCOM llama a:
- `GET /api/v1/protocols/by-barcode/{code}`
- `GET /api/v1/protocols/{type}/{id}`

El protocolo encontrado no transiciona de `pending` a `in_progress`.
El servicio hace el lookup pero no tiene ningún side-effect de estado.

## PASO 1 — Agregar status 'validated' al persistir validación en VetAdmissionTest

En `VetAdmissionController`:

### 1.1 `validateTest()`
Agregar `'status' => 'validated'` en el update:

```php
$vetAdmissionTest->update([
    'is_validated'  => true,
    'validated_by'  => auth()->id(),
    'validated_at'  => now(),
    'status'        => 'validated',
]);
```

Después del update, recalcular y persistir el status del protocolo padre:

```php
$vetAdmission->load('vetTests');
$vetAdmission->update(['status' => $vetAdmission->calculated_status]);
```

### 1.2 `validateAll()`
Dentro del foreach, agregar `'status' => 'validated'` al update del `$vat`:

```php
$vat->update([
    'is_validated'  => true,
    'validated_by'  => auth()->id(),
    'validated_at'  => now(),
    'status'        => 'validated',
]);
```

Después del foreach, recalcular el protocolo:

```php
$vetAdmission->load('vetTests');
$vetAdmission->update(['status' => $vetAdmission->calculated_status]);
```

### 1.3 `unvalidateTest()`
Revertir el status según si tiene resultado:

```php
$vetAdmissionTest->update([
    'is_validated'  => false,
    'validated_by'  => null,
    'validated_at'  => null,
    'status'        => $vetAdmissionTest->hasResult() ? 'completed' : 'pending',
]);
```

Después del update, recalcular el protocolo:

```php
$vetAdmission->load('vetTests');
$vetAdmission->update(['status' => $vetAdmission->calculated_status]);
```

## PASO 3 — Transición pending → in_progress al consultar por LISCOM

En `ProtocolController`, después de encontrar un protocolo y antes de retornarlo,
aplicar la transición de estado si corresponde. La transición solo se dispara
en endpoints de lookup puntual (barcode o id), **no** en el listado.

### 3.1 `showByBarcode()`

```php
$found = $this->service->findByBarcode($code, $client);

if (! $found) {
    return response()->json(['error' => 'Protocol not found'], 404);
}

if ($found->status === 'pending') {
    $found->update(['status' => 'in_progress']);
}

return new ProtocolResource($found);
```

### 3.2 `show()`

Después de encontrar `$found` con `$modelClass::query()...->first()`:

```php
if (! $found) {
    return response()->json(['error' => 'Protocol not found'], 404);
}

if ($found->status === 'pending') {
    $found->update(['status' => 'in_progress']);
}

return new ProtocolResource($found);
```

## PASO 4 — Verificación

Recorrer manualmente el flujo:

1. Abrir un protocolo veterinario con determinaciones en estado `pending` o `completed`.
2. Hacer clic en "Validar" sobre una determinación → verificar que el badge de la
   determinación muestre "Validado" y el badge del protocolo refleje el estado correcto.
3. Hacer clic en "Validar Todo" → verificar que el protocolo pase a "Validado" en su badge.
4. Hacer clic en desvalidar → verificar que revierta a "Completado" o "Pendiente".
5. Desde un cliente API (Postman/curl), hacer `GET /api/v1/protocols/by-barcode/{code}`
   con un protocolo `pending` → verificar que quede `in_progress` en la DB.
6. Repetir con `GET /api/v1/protocols/{type}/{id}`.

Si hay tests Feature en `tests/Feature/VetAdmissionTest.php` o similar, correr:

```bash
php artisan test --filter=VetAdmission
php artisan test --filter=ProtocolController
```

Corregir cualquier fallo relacionado con los cambios de este hotfix.
