## PASO 1 — Componente combobox reutilizable

Crear un componente Blade reutilizable para el combobox de búsqueda.
Puede ser inline (sin archivo separado) o un partial.

### Patrón del combobox con Alpine.js

```html
<div x-data="{
    open: false,
    search: '',
    selectedId: '',
    selectedName: '',
    
    get filtered() {
        if (!this.search) return this.items;
        const q = this.search.toLowerCase();
        return this.items.filter(i => i.name.toLowerCase().includes(q));
    },
    
    items: [/* se llena desde Blade */],
    
    select(id, name) {
        this.selectedId = id;
        this.selectedName = name;
        this.search = name;
        this.open = false;
    },
    
    clear() {
        this.selectedId = '';
        this.selectedName = '';
        this.search = '';
    }
}" @click.away="open = false" class="relative">
    
    <input type="text" 
           x-model="search" 
           @focus="open = true" 
           @input="open = true"
           placeholder="Buscar obra social..."
           class="w-full border-gray-300 rounded-lg shadow-sm focus:ring-teal-500 focus:border-teal-500">
    
    <!-- Hidden input con el valor real -->
    <input type="hidden" name="insurance_id" :value="selectedId">
    
    <!-- Botón limpiar -->
    <button type="button" x-show="selectedId" @click="clear()" 
            class="absolute right-2 top-1/2 -translate-y-1/2 text-gray-400 hover:text-gray-600">
        ✕
    </button>
    
    <!-- Dropdown de opciones -->
    <div x-show="open && filtered.length > 0" x-cloak
         class="absolute z-50 mt-1 w-full bg-white border border-gray-200 rounded-lg shadow-lg max-h-60 overflow-y-auto">
        <template x-for="item in filtered" :key="item.id">
            <div @click="select(item.id, item.name); $dispatch('insurance-changed', { id: item.id, nbu: item.nbu })"
                 class="px-3 py-2 cursor-pointer hover:bg-teal-50 text-sm"
                 :class="{ 'bg-teal-100': selectedId == item.id }">
                <span x-text="item.name"></span>
            </div>
        </template>
    </div>
    
    <!-- Sin resultados -->
    <div x-show="open && search && filtered.length === 0" x-cloak
         class="absolute z-50 mt-1 w-full bg-white border border-gray-200 rounded-lg shadow-lg p-3 text-sm text-gray-500">
        No se encontraron resultados
    </div>
</div>
```

### Datos inyectados desde Blade

El array `items` se llena con un JSON generado desde Blade:

```javascript
items: @json($insurances->map(fn($ins) => [
    'id' => $ins->id,
    'name' => strtoupper($ins->name),
    'nbu' => $ins->nbu_value ?? 0,
])),
```

## PASO 2 — Aplicar en Lab Admisión Crear

En `resources/views/lab/admissions/create.blade.php` (líneas ~125-140):

### Reemplazar el `<select>` actual:

```html
<select name="insurance_id" x-model="insuranceId" @change="onInsuranceChange" required ...>
    <option value="">Seleccionar...</option>
    @foreach($insurances as $ins)
        <option value="{{ $ins->id }}" data-nbu="{{ $ins->nbu_value }}" ...>
            {{ strtoupper($ins->name) }}
        </option>
    @endforeach
</select>
```

### Por el combobox con integración a `admissionManager()`:

**CRÍTICO:** Este formulario ya tiene un `x-data="admissionManager()"` padre que
maneja `insuranceId` y `onInsuranceChange`. El combobox debe:

1. Actualizar `insuranceId` en el componente padre cuando se selecciona
2. Disparar `onInsuranceChange` para que se recalculen los precios de las prácticas
3. Pasar el valor `data-nbu` al handler

Opciones de integración:
- **Opción A**: Usar `$dispatch('insurance-changed', { id, nbu })` y escuchar en el padre
- **Opción B**: Acceder directamente al scope padre con `$parent` o compartiendo el modelo

Evaluar cuál es más limpia según cómo está definido `admissionManager()`.
Si `onInsuranceChange` lee el `data-nbu` del `<option>` seleccionado via DOM, hay que
adaptar para que reciba el NBU directamente del item seleccionado.

### Pre-selección

Si hay un valor `old('insurance_id')` o `$patient->insurance`, el combobox debe
inicializarse con ese valor seleccionado (buscar en el array de items y setear
`selectedId` y `search` al montar).
