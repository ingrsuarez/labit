# v1.16.0 — Planillas de trabajo diario del laboratorio

**VERSIÓN:** v1.16.0
**SLUG:** planillas-trabajo-laboratorio
**DEPENDENCIAS:** ninguna
**ESTIMACIÓN:** un día
**PRIORIDAD:** alta (pedido del cliente)

---

## Descripción

Permite crear plantillas reutilizables de planillas de trabajo con un nombre, un tipo (lab clínico o muestras) y un set de determinaciones seleccionadas. Los técnicos y bioquímicos filtran por fecha, rango de protocolo y estado de resultados, y descargan un PDF tabular landscape listo para imprimir y completar a mano. Funciona para ambos módulos del laboratorio.

### 1.2 — Tabla pivot `worksheet_test`

```php
Schema::create('worksheet_test', function (Blueprint $table) {
    $table->id();
    $table->foreignId('worksheet_id')->constrained()->cascadeOnDelete();
    $table->foreignId('test_id')->constrained()->cascadeOnDelete();
    $table->unsignedInteger('sort_order')->default(0);
    $table->timestamps();
});
```

## PASO 2 — Modelo `Worksheet`

Crear `app/Models/Worksheet.php`:

```php
class Worksheet extends Model
{
    protected $fillable = ['name', 'type', 'created_by'];

    public function tests()
    {
        return $this->belongsToMany(Test::class, 'worksheet_test')
                    ->withPivot('sort_order')
                    ->orderByPivot('sort_order');
    }

    public function creator()
    {
        return $this->belongsTo(User::class, 'created_by');
    }
}
```

**Verificación:** `$worksheet->tests` devuelve los tests ordenados. `$worksheet->creator` devuelve el usuario.

### 3.7 — `generatePdf(Request $request, Worksheet $worksheet)`

Recibe los filtros, ejecuta la consulta, genera el PDF.

**Lógica de consulta — Lab clínico (type = `clinico`):**

```php
$testIds = $worksheet->tests()->pluck('tests.id');

$query = Admission::with(['patient', 'admissionTests' => function ($q) use ($testIds) {
    $q->whereIn('test_id', $testIds);
}])
->whereHas('admissionTests', function ($q) use ($testIds) {
    $q->whereIn('test_id', $testIds);
})
->whereBetween('created_at', [$dateFrom, $dateTo]); // o el campo de fecha que corresponda
```

> Investigar qué campo de fecha usar en Admission (puede ser `created_at`, `date`, `admission_date`).
> Investigar qué campo tiene el N° de protocolo en Admission (puede ser `id`, `protocol_number`).

Aplicar filtro de rango de protocolo si se proporcionó.

Aplicar filtro de resultados:
- Si solo "sin resultados": filtrar admisiones donde al menos un test del worksheet NO tenga resultado
- Si solo "con resultados": filtrar admisiones donde al menos un test del worksheet tenga resultado
- Si ambos: no filtrar por estado de resultado (mostrar todos)

**Lógica de consulta — Muestras (type = `muestras`):**

```php

    $q->whereIn('test_id', $testIds);

    $q->whereIn('test_id', $testIds);

```

> Investigar las relaciones y campos reales de Sample (puede ser `sampleDeterminations`, `determinations`, etc.).

Crear `resources/views/worksheets/pdf.blade.php`:

- **Orientación: landscape** (importante, muchas columnas)
- **Encabezado:** Nombre de la planilla + rango de fechas + fecha de generación

## Notas / Consideraciones

- Las planillas son templates reutilizables — se crean una vez y se usan diariamente cambiando el filtro de fecha.
- Cada planilla es de un solo módulo (clínico O muestras). No se mezclan tests de distintos módulos.
- El campo `code` del test se usa como abreviatura/sigla en las columnas del PDF. Si algún test no tiene código, usar las primeras letras del nombre.
- Para PDFs con muchos tests (>15 columnas), puede ser necesario reducir el font-size o rotar los encabezados. Evaluar al implementar.
- No se implementa carga de resultados desde la planilla — es solo visualización/impresión.
- No se implementa exportación Excel en esta versión.
- El sort_order de los tests en la planilla determina el orden de las columnas en el PDF.
