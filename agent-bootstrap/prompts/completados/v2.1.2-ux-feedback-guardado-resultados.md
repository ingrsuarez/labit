# v2.1.2 — UX: feedback visual de guardado en carga de resultados

**VERSIÓN:** v2.1.2  
**SLUG:** ux-feedback-guardado-resultados  
**DEPENDENCIAS:** v2.1.1  
**ESTIMACIÓN:** 30min  
**PRIORIDAD:** media

---

## Descripción

Mejora de UX en la pantalla de carga de resultados de muestras (`/sample/{id}/load-results`).
Actualmente al hacer click en "Guardar Todo" o "Aplicar a todos" (que ahora auto-guarda
después de v2.1.1), no hay feedback visual de que el formulario se está procesando.
El usuario puede hacer doble click o pensar que no funcionó.

### Mejoras

1. **Spinner en botón "Guardar Todo"** — Al hacer submit, el botón muestra "Guardando..."
   con un spinner animado y se deshabilita para evitar doble submit.
2. **Spinner en botón "Aplicar a todos"** — Mismo comportamiento al aplicar normativa
   (que ahora auto-guarda).
3. **Spinner en botón "Marcar Todo Completado"** — Feedback al cambiar estados masivamente
   (este no hace submit pero sí modifica el DOM, mostrar brevemente el cambio).
4. **Overlay semi-transparente** (opcional) — Un overlay sutil sobre el formulario durante
   el guardado para que el usuario no modifique campos mientras se procesa.

## PASO 1 — Estado de carga en "Guardar Todo"

En `resources/views/sample/load-results.blade.php`, modificar el botón "Guardar Todo"
y agregar un handler de submit al formulario.

### 1.1 — Agregar ID al botón submit

El botón "Guardar Todo" necesita un `id` para referenciarlo desde JS:

```html
<button type="submit" id="btnSaveAll"
        class="px-6 py-2 bg-teal-600 text-white rounded-lg hover:bg-teal-700 transition-colors font-medium">
    <svg class="w-5 h-5 inline mr-2" ...>...</svg>
    Guardar Todo
</button>
```

### 1.2 — Handler de submit del formulario

Agregar en el bloque `<script>`:

```javascript
document.getElementById('resultsForm').addEventListener('submit', function() {
    const btn = document.getElementById('btnSaveAll');
    if (btn) {
        btn.disabled = true;
        btn.innerHTML = `
            <svg class="w-5 h-5 inline mr-2 animate-spin" fill="none" viewBox="0 0 24 24">
                <circle class="opacity-25" cx="12" cy="12" r="10" stroke="currentColor" stroke-width="4"></circle>
                <path class="opacity-75" fill="currentColor" d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"></path>
            </svg>
            Guardando...
        `;
        btn.classList.add('opacity-75', 'cursor-not-allowed');
    }
});
```

**Verificación:** Al clickear "Guardar Todo", el botón cambia a "Guardando..." con spinner
y se deshabilita hasta que la página recarga.

## PASO 3 — Protección contra doble submit con Ctrl+S

El handler de Ctrl+S (atajo de teclado) también debe deshabilitar el botón:

```javascript
document.addEventListener('keydown', function(e) {
    if (e.ctrlKey && e.key === 's') {
        e.preventDefault();
        const btn = document.getElementById('btnSaveAll');
        if (btn && !btn.disabled) {
            document.getElementById('resultsForm').submit();
        }
    }
});
```

El `!btn.disabled` evita re-submit si ya se está guardando.

**Verificación:** Ctrl+S guarda con el mismo feedback visual. Presionar Ctrl+S dos veces
rápido solo envía una vez.

## Notas / Consideraciones

- Esta versión solo toca `load-results.blade.php` — no hay cambios de backend.
- El spinner usa la clase `animate-spin` de Tailwind que ya está disponible en el proyecto.
- No se agrega overlay porque podría ser invasivo en una pantalla de carga rápida. Si el guardado es lento (>2s), reconsiderar.
- El patrón de disable-on-submit se podría extraer a un componente o helper JS reutilizable en el futuro (para aplicar en otros formularios del sistema).
