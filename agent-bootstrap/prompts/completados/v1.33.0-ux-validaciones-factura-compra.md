# v1.33.0 — UX y validaciones en formulario de factura de compra

**VERSIÓN:** v1.33.0

## Descripción

Tres mejoras de UX y validación en el formulario de nueva/edición de factura de compra
(`purchase-invoices/create` y `purchase-invoices/edit`):

1. **Zero-padding automático** en Punto de Venta (5 dígitos) y Nº Factura (8 dígitos)
2. **Validación de duplicados** en tiempo real: al elegir proveedor + PV + Nº Factura,
   alertar si ya existe esa combinación en la base de datos
3. **Auto-cálculo de fecha de vencimiento**: al completar fecha de emisión,
   calcular automáticamente Vencimiento = Emisión + 30 días (editable)

Ningún cambio toca el modelo de datos ni la lógica de stock — solo la vista y validaciones del formulario.

## PASO 1 — Zero-padding en Punto de Venta y Nº Factura

En el formulario de factura de compra, ambos campos deben rellenarse con ceros a la izquierda
al salir del campo (evento `blur`).

- **Punto de Venta:** 5 dígitos → `"9"` → `"00009"`, `"123"` → `"00123"`
- **Nº Factura:** 8 dígitos → `"5771"` → `"00005771"`, `"1"` → `"00000001"`

### 1.1 — En el componente Alpine `invoiceForm()`

Agregar los siguientes handlers en el `return {}` del componente:

```javascript
padPointOfSale() {
    if (this.point_of_sale) {
        this.point_of_sale = String(parseInt(this.point_of_sale) || 0).padStart(5, '0');
    }
},

padInvoiceNumber() {
    if (this.invoice_number) {
        this.invoice_number = String(parseInt(this.invoice_number) || 0).padStart(8, '0');
    }
},
```

### 1.2 — En los inputs del formulario

Agregar `@blur="padPointOfSale()"` y `@blur="padInvoiceNumber()"`:

```html
<!-- Punto de Venta -->
<input type="text"
       x-model="point_of_sale"
       @blur="padPointOfSale(); checkDuplicate()"
       maxlength="5"
       placeholder="00000"
       class="...">

<!-- Nº Factura -->
<input type="text"
       x-model="invoice_number"
       @blur="padInvoiceNumber(); checkDuplicate()"
       maxlength="8"
       placeholder="00000000"
       class="...">
```

**Verificación:** Ingresar `9` en Punto de Venta, hacer Tab → muestra `00009`.
Ingresar `5771` en Nº Factura, hacer Tab → muestra `00005771`.

## PASO 2 — Validación de duplicados en tiempo real

Al completar Proveedor + Punto de Venta + Nº Factura, verificar via AJAX si ya existe
una factura de compra con esa combinación para ese proveedor.

### 2.1 — Endpoint de verificación

Agregar ruta y método en `PurchaseInvoiceController`:

```php
// routes/web.php (dentro del grupo autenticado con permisos de compras)
Route::get('purchase-invoices/check-duplicate', [PurchaseInvoiceController::class, 'checkDuplicate'])
    ->name('purchase-invoices.check-duplicate');
```

```php
// PurchaseInvoiceController.php
public function checkDuplicate(Request $request)
{
    $request->validate([
        'supplier_id'     => 'required|integer',
        'point_of_sale'   => 'required|string',
        'invoice_number'  => 'required|string',
        'exclude_id'      => 'nullable|integer', // para edición: excluir la factura actual
    ]);

    $exists = PurchaseInvoice::where('supplier_id', $request->supplier_id)
        ->where('point_of_sale', $request->point_of_sale)
        ->where('invoice_number', $request->invoice_number)
        ->when($request->exclude_id, fn($q) => $q->where('id', '!=', $request->exclude_id))
        ->exists();

    return response()->json(['duplicate' => $exists]);
}
```

### 2.2 — Función `checkDuplicate()` en Alpine

En `invoiceForm()`, agregar:

```javascript
duplicateWarning: false,

async checkDuplicate() {
    if (!this.supplier_id || !this.point_of_sale || !this.invoice_number) {
        this.duplicateWarning = false;
        return;
    }
    try {
        const params = new URLSearchParams({
            supplier_id:    this.supplier_id,
            point_of_sale:  this.point_of_sale,
            invoice_number: this.invoice_number,
            exclude_id:     this.editId || '',
        });
        const resp = await fetch(`{{ route('purchase-invoices.check-duplicate') }}?${params}`);
        const data = await resp.json();
        this.duplicateWarning = data.duplicate;
    } catch (e) {
        this.duplicateWarning = false;
    }
},
```

Llamar `checkDuplicate()` también en el evento `@change` del select de Proveedor.

### 2.3 — Alerta visual en el formulario

Debajo del grupo de campos Punto de Venta / Nº Factura / Proveedor, agregar:

```html
<div x-show="duplicateWarning" x-cloak
     class="col-span-4 flex items-center gap-2 text-sm text-amber-700 bg-amber-50 border border-amber-200 rounded-lg px-4 py-2">
    <i class="bi bi-exclamation-triangle-fill"></i>
    <span>Ya existe una factura de este proveedor con el mismo Punto de Venta y Nº de Factura.</span>
</div>
```

La alerta es informativa — **no bloquea el guardado**. El usuario puede igualmente guardar
si tiene una razón para hacerlo (ej: NC, rectificativa).

**Verificación:** Crear una FC con PV 00001, Nº 00000001, Proveedor X. Abrir una nueva FC,
seleccionar el mismo proveedor y completar los mismos datos → aparece la alerta.
Cambiar el número → desaparece.

## PASO 3 — Auto-cálculo de fecha de vencimiento

Al seleccionar la fecha de emisión, calcular automáticamente la fecha de vencimiento como
emisión + 30 días. El campo de vencimiento queda editable por si el usuario necesita cambiarlo.

### 3.1 — En `invoiceForm()`, agregar:

```javascript
autoCalcDueDate() {
    if (this.issue_date) {
        const issue = new Date(this.issue_date);
        issue.setDate(issue.getDate() + 30);
        // Formato YYYY-MM-DD para input type="date"
        this.due_date = issue.toISOString().split('T')[0];
    }
},
```

### 3.2 — En el input de Fecha de Emisión:

```html
<input type="date"
       x-model="issue_date"
       @change="autoCalcDueDate()"
       required
       class="...">
```

El campo de Fecha de Vencimiento no requiere cambios — ya es editable y el usuario puede
sobreescribir el valor calculado.

**Verificación:** Seleccionar fecha de emisión `2026-04-04` → Fecha de Vencimiento se completa
automáticamente con `2026-05-04`. El usuario puede cambiar el valor de vencimiento manualmente.

```javascript
function invoiceForm(params = {}) {
```

**Verificación:** En edición, la factura actual no dispara la alerta de duplicado.
Cambiar el número a uno que sí existe en otra factura → sí dispara la alerta.

## Notas / Consideraciones

- El padding usa `padStart(5, '0')` y `padStart(8, '0')` que es nativo de JS, sin dependencias.
- La alerta de duplicados es advertencia, no bloqueo — el sistema no puede saber si es una
  nota de crédito, rectificativa u otro motivo válido para repetir numeración.
- El cálculo de +30 días es la regla general. El usuario siempre puede editar el campo.
  En versiones futuras se podría configurar el plazo por proveedor.
