## PASO 2 — Configuración de entorno

Agregar variables de entorno al `.env` y al `.env.example`:

```
# AFIP Web Services
AFIP_CUIT=
AFIP_CERT_PATH=storage/app/afip/cert.pem
AFIP_KEY_PATH=storage/app/afip/key.pem
AFIP_PRODUCTION=false
```

Crear archivo de configuración `config/afip.php`:

```php
<?php

return [
    'cuit' => env('AFIP_CUIT'),
    'cert_path' => env('AFIP_CERT_PATH', storage_path('app/afip/cert.pem')),
    'key_path' => env('AFIP_KEY_PATH', storage_path('app/afip/key.pem')),
    'production' => env('AFIP_PRODUCTION', false),
];
```

Crear el directorio para certificados:
```bash
mkdir -p storage/app/afip
```

Agregar `storage/app/afip/*.pem` al `.gitignore` para no commitear certificados.

**Verificación:** `config('afip.cuit')` devuelve el valor del .env. El directorio `storage/app/afip/` existe.

## PASO 3 — Crear AfipService

Crear `app/Services/AfipService.php`. Este servicio encapsula toda la comunicación con AFIP.

El servicio debe:

1. **Instanciar la librería Afip** con CUIT, certificado, clave privada y ambiente (homologación/producción) desde config
2. **Método `getServerStatus()`** — llama a `FEDummy` para verificar conectividad con AFIP (retorna estado de AppServer, DbServer, AuthServer)
3. **Método `getLastVoucher(int $pointOfSale, int $voucherType): int`** — llama a `FECompUltimoAutorizado` para obtener el último número de comprobante autorizado
4. **Método `getVoucherTypeId(string $letter, string $type = 'factura'): int`** — mapea tipo de comprobante:
   - Factura A = 1, Factura B = 6, Factura C = 11
   - NC A = 3, NC B = 8, NC C = 13
5. **Manejo de errores**: capturar excepciones de la librería y lanzar excepciones propias con mensajes claros

Registrar el servicio en el container si es necesario (o usar inyección directa).

Usar la documentación de AfipSDK: la clase `Afip` se instancia con un array de configuración:
```php
$afip = new Afip([
    'CUIT' => config('afip.cuit'),
    'cert' => config('afip.cert_path'),
    'key' => config('afip.key_path'),
    'production' => config('afip.production'),
]);
```

Para WSFEv1, usar `$afip->ElectronicBilling->...`

**Verificación:** La clase existe, se puede instanciar, y `getServerStatus()` no tira excepción (puede fallar si no hay certificados todavía, eso es OK).

## Notas / Consideraciones

- Los certificados de homologación se generan desde WSASS de AFIP — el usuario ya tiene acceso
- La librería AfipSDK maneja internamente el Token y Sign del WSAA (con cache)
- En homologación, el CUIT puede ser cualquiera habilitado en el entorno de testing de AFIP
- No comprometer certificados reales al repositorio
- Leer el documento de diseño para las clases Tailwind exactas a usar
