# v1.6.1 — Filtrar nomencladores de dropdowns y crear "Particular"

**VERSIÓN:** v1.6.1  
**SLUG:** filtrar-nomencladores-dropdowns-particular  
**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 30min  
**PRIORIDAD:** alta (bloquea flujo de admisiones)

---

## Descripción

Los dropdowns de "Obra Social" en la admisión de pacientes y del lab clínico
muestran nomencladores base (type = 'nomenclador') que no deberían aparecer.
Solo deben mostrarse obras sociales reales y "Particular".

Además, crear el registro "Particular" en la tabla `insurances` con un seeder,
ya que no existe en producción. "Particular" funciona como una obra social
con NBU propio para pacientes que pagan directamente.

---

Leer sin ejecutar nada:

- `app/Http/Controllers/PatientController.php` — métodos que cargan insurances
- `app/Http/Controllers/LabAdmissionController.php` — métodos index, create, edit
- `app/Models/Insurance.php` — modelo, campo `type`
- `resources/views/patient/index.blade.php` — dropdown de obra social en pacientes
- `resources/views/patient/edit.blade.php` — dropdown de obra social en edición
- `resources/views/lab/admissions/create.blade.php` — dropdown en admisión lab
- `resources/views/lab/admissions/index.blade.php` — filtro de obra social en listado

## PASO 1 — Crear el seeder ParticularInsuranceSeeder

Crear `database/seeders/ParticularInsuranceSeeder.php`.

### Lógica:

1. Verificar si ya existe un Insurance con `type = 'particular'`
2. Si no existe, crear:

```php
Insurance::firstOrCreate(
    ['type' => 'particular'],
    [
        'name' => 'Particular',
        'tax_id' => null,
        'tax' => null,
        'address' => null,
        'phone' => null,
        'email' => null,
        'state' => null,
        'country' => 'Argentina',
        'price' => 0,
        'nbu' => 1,
        'nbu_value' => 0, // Se configura después desde el nomenclador
        'nomenclator_id' => null,
        'group' => null,
        'type' => 'particular',
        'instructions' => 'Paciente sin cobertura médica. Pago directo.',
    ]
);
```

3. Loguear si se creó o ya existía.

**Verificación:** El seeder es idempotente. Ejecutar dos veces no duplica.

## PASO 2 — Filtrar nomencladores en PatientController

En `app/Http/Controllers/PatientController.php`:

### index() — línea ~13:

**Antes:**
```php
$insurances = Insurance::all();
```

**Después:**
```php
$insurances = Insurance::where('type', '!=', 'nomenclador')
    ->orderByRaw("CASE WHEN type = 'particular' THEN 0 ELSE 1 END")
    ->orderBy('name')
    ->get();
```

### edit() — línea ~65:

**Antes:**
```php
$insurances = Insurance::all();
```

**Después:**
```php
$insurances = Insurance::where('type', '!=', 'nomenclador')
    ->orderByRaw("CASE WHEN type = 'particular' THEN 0 ELSE 1 END")
    ->orderBy('name')
    ->get();
```

Esto filtra los nomencladores base y pone "Particular" primero en la lista.

## PASO 3 — Filtrar nomencladores en LabAdmissionController

En `app/Http/Controllers/LabAdmissionController.php`, hay 3 métodos que cargan insurances.
Todos ya tienen el `orderByRaw` pero les falta el filtro.

### index() — línea ~52:

**Antes:**
```php
$insurances = Insurance::orderByRaw("CASE WHEN type = 'particular' THEN 0 ELSE 1 END")
    ->orderBy('type')
    ->orderBy('name')
    ->get();
```

**Después:**
```php
$insurances = Insurance::where('type', '!=', 'nomenclador')
    ->orderByRaw("CASE WHEN type = 'particular' THEN 0 ELSE 1 END")
    ->orderBy('name')
    ->get();
```

### create() — línea ~71:

Mismo cambio: agregar `->where('type', '!=', 'nomenclador')`.

### edit() — línea ~218:

Mismo cambio: agregar `->where('type', '!=', 'nomenclador')`.
