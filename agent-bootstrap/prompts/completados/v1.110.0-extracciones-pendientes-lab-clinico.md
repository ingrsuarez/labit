## Descripción

Implementar el flujo de **toma de muestra (extracción)** en **laboratorio clínico** únicamente.

### Reglas de negocio (acordadas en PM)

1. **¿Requiere extracción?** Admisión con al menos una práctica **padre** (no hija de otra práctica ya cargada en el protocolo) cuyo `Test` tenga `material` asignado (`materialRelation`). Reutilizar la misma noción que `LabAdmissionController::groupByMaterial()` (ver PASO 2).
2. **Pendiente:** requiere extracción **y** `sample_drawn_by` es `null`.
3. **Cierre de extracción:** al asignar `sample_drawn_by` (y `sample_drawn_at`), la admisión deja de estar pendiente.
4. **Tomador obligatorio:** no se puede cerrar extracción sin `sample_drawn_by` apuntando a un usuario con rol `tecnico-lab` o `bioquimico`.
5. **Roles que operan el flujo:** `tecnico-lab`, `bioquimico`, `recepcion-lab`, `admin`.
   - Técnico / bioquímico: al registrar desde el modal, `sample_drawn_by` = usuario autenticado.
   - Recepción / admin: deben elegir tomador en un `<select>` (solo usuarios técnico o bioquímico).
6. **Dos caminos de registro:**
   - **A — Modal en header:** botón llamativo con contador por sede activa → drawer/modal con lista de pendientes → confirmar extracción.
   - **B — Formulario admisión (create/edit):** selector de tomador **siempre visible** cuando hay prácticas con material (Alpine: mostrar/ocultar según `selectedTests`). Vacío → queda pendiente; con valor → extracción cerrada al guardar (caso carga masiva post-extracción en sedes con 20+ pacientes).
7. **Planillas de trabajo (`type === clinico`):** excluir admisiones que requieren extracción y aún no tienen `sample_drawn_by`. Mostrar aviso en preview/PDF si se omitieron protocolos por este motivo.
8. **Auditoría:** acción `sample_drawn` en el `Admission` auditable. Mensaje con protocolo, paciente y **nombre del tomador** (`sample_drawn_by`), no el usuario de recepción que registró.
9. **KPIs:** en `EmployeeProductivityService`, métrica `samples_drawn` para filas con rol `tecnico-lab` o `bioquimico`, contando por `admissions.sample_drawn_by` en el día (filtro sede). Agregar columna **Extracciones** en `resources/views/rrhh/productividad.blade.php` y export CSV.
10. **Fuera de alcance:** veterinario, muestras/aguas, impresión automática de etiquetas, backfill de admisiones históricas.

## PASO 7 — KPIs y vista RRHH

```php
private function sampleDrawMetrics(int $userId, Carbon $start, Carbon $end, ?int $labBranchId): array
{
    $q = Admission::query()
        ->where('sample_drawn_by', $userId)
        ->whereBetween('sample_drawn_at', [$start, $end]);
    if ($labBranchId) {
        $q->where('lab_branch_id', $labBranchId);

    return ['samples_drawn' => $q->count()];
}
```

## Notas / Consideraciones

- **Detección material:** alinear con etiquetas (prácticas padre con `material`, no hijos duplicados).
- **Cambio de prácticas en edit:** si se elimina última práctica con material, limpiar extracción registrada o mantener — preferencia: **limpiar** si ya no requiere.
- **Re-extracción / corrección:** fuera de scope; admin podría editar tomador en `edit` admisión si se implementa el selector ahí.
- **Sede “Todas”:** contador suma pendientes de todas las sedes; modal lista con columna sede si hay mezcla.
- Sin Designer: usar tokens Tailwind existentes (teal lab, rose para urgencia extracción).
