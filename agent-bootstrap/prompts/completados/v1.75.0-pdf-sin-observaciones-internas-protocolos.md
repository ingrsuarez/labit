# v1.75.0 — PDF de protocolos sin observaciones internas

**Tipo:** patch / fix de producto  
**Prioridad:** media-alta — evita fugas de información operativa en informes al cliente  
**Dependencias:** ninguna

---

## Problema

En los PDF de resultados (lab clínico, muestras y veterinario) se imprimen las
**observaciones del protocolo** (y en muestras también observaciones por determinación).
Ese contenido se usa como nota interna del laboratorio (urgencias, indicaciones entre
personal) y **no debe figurar en el informe** que ve el cliente.

Las observaciones deben seguir apareciendo en las pantallas web de administración para el equipo.

---

## Contexto — Archivos donde hoy se imprimen observaciones

```
resources/views/lab/admissions/pdf-mpdf.blade.php      ← bloque $admission->observations
resources/views/vet/admissions/pdf-mpdf.blade.php     ← bloque $vetAdmission->observations
resources/views/sample/pdf-mpdf.blade.php              ← bloque $sample->observations + filas $det->observations
resources/views/sample/pdf.blade.php                 ← igual (vista legacy si aún se usa)
```

Confirmar en `routes` o controladores si `sample/pdf.blade.php` sigue referenciado; si no,
documentar en el PR y aplicar igualmente por consistencia.

---

## PASO 1 — Lab clínico

En `lab/admissions/pdf-mpdf.blade.php`, eliminar por completo el bloque:

```blade
@if($admission->observations)
<div class="observations-block">
    ...
</div>
@endif
```

Opcional: si las clases CSS `.observations-block` / `.observations-title` quedan sin uso en
este archivo, eliminarlas del `<style>` del mismo blade para no dejar código muerto.

## PASO 2 — Veterinario

En `vet/admissions/pdf-mpdf.blade.php`, igual que PASO 1: quitar el bloque
`$vetAdmission->observations`. Limpiar CSS huérfano si aplica.
