# v1.67.1 — Hotfix: billing batch al cambiar empresa (MethodNotAllowed)

**Tipo:** patch / hotfix
**Prioridad:** inmediata — bug en producción
**Dependencias:** ninguna (v1.67.0 ya completada)

---

## Problema

Tras hacer batch-preview de facturación de determinaciones seleccionadas (`POST /billing/batch-preview`), la URL del navegador queda en `/billing/batch-preview`. Al cambiar de empresa en la barra superior, `CompanyController@switchCompany` ejecuta `redirect()->back()`, que genera un **GET** a esa URL; la ruta solo acepta **POST** → `MethodNotAllowedHttpException` (error 405).

Mismo patrón que v1.41.1 (Libro IVA), que ya anticipaba este caso en su nota PM.

---

## Contexto — Archivos a leer

```
app/Http/Controllers/CompanyController.php        ← switchCompany() con lista de paths POST-only
routes/web.php                                      ← rutas billing/batch-preview y billing/batch-invoice (solo POST)
agent-bootstrap/prompts/completados/HOTFIX-v1.41.1-libro-iva-cambio-empresa.md  ← patrón de referencia
```

---

## PASO 1 — Extender lista de paths POST-only en switchCompany

En `CompanyController@switchCompany`, agregar `/billing/batch-preview` y `/billing/batch-invoice` a la lógica de detección de paths POST-only. Redirigir a `route('billing.uninvoiced')` con mensaje flash descriptivo.

Patrón existente (Libro IVA):
```php
if (str_ends_with($path, '/libro-iva/preview') || str_ends_with($path, '/libro-iva/download')) {
    return redirect()->route('libro-iva.index')
        ->with('success', 'Empresa actualizada. Volvé a generar el resumen del Libro IVA para la nueva empresa.');
}
```

Agregar bloque análogo para billing:
```php
if (str_ends_with($path, '/billing/batch-preview') || str_ends_with($path, '/billing/batch-invoice')) {
    return redirect()->route('billing.uninvoiced')
        ->with('success', 'Empresa actualizada. Seleccioná nuevamente las determinaciones a facturar.');
}
```
