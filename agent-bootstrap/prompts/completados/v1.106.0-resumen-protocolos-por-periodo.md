## Descripción

Reemplazar el reporte mensual clínico actual (`lab/reports/monthly`), que genera **una fila por práctica** (paciente repetido N veces), por un resumen de **una fila por protocolo** con códigos de determinaciones concatenados (`-`) y precio total del protocolo.

Además, agregar el mismo tipo de resumen para **Aguas y Alimentos** y **Veterinario**, cada uno con card propia en su hub de sección, filtros por **cliente** o **obra social** + **rango de fechas**, y exportación **PDF + Excel**.

Incluye **todos** los protocolos del período (facturados y sin facturar). Regla de códigos acordada en planificación PM:

- Incluir determinaciones sueltas, títulos y padres.
- **No** incluir hijos si el padre está cargado en el mismo protocolo.
- **Sí** incluir hijos huérfanos (padre no presente en el protocolo).

**Diseño obligatorio:** `docs/designs/DISEÑO_v1.106.0-resumen-protocolos-por-periodo.md`

## PASO 1 — Servicio compartido: códigos y filas de resumen

Crear `app/Services/BillingSummaryCodeResolver.php`:

```php
/**
 * Dado un protocolo con líneas que tienen test_id + test (eager parentTests/childTests),
 * devuelve las líneas "visibles" en el resumen y el string de códigos.
 *
 * @param  Collection<int, object>  $lines  AdmissionTest|SampleDetermination|VetAdmissionTest
 * @param  callable(object): ?Test  $getTest
 * @param  callable(object): int  $getTestId
 * @param  callable(object): bool  $includeLine  Filtro de negocio (ej. excluir rechazadas / paid_by_patient)
 */
public function resolve(Collection $lines, callable $getTest, callable $getTestId, callable $includeLine): array
```

**Algoritmo de inclusión:**

1. Filtrar líneas con `$includeLine`.
2. Construir `$presentTestIds` = todos los `test_id` del protocolo.
3. Para cada línea restante, obtener padres vía `$test->parentTests` (eager).
4. **Excluir** la línea si existe algún `parent_test_id` en `$presentTestIds`.
5. **Incluir** el resto (sueltas, padres/títulos, huérfanos).
6. Ordenar por `sort_order` de la línea o por `test.code` ascendente (definir y documentar; preferir orden de carga del protocolo).
7. `codes_string` = `collect($included)->pluck('test.code')->filter()->implode('-')`.

Crear `app/Services/BillingSummaryService.php` con métodos:

- `buildClinicalRows(Insurance $insurance, Carbon $from, Carbon $to): array{rows: Collection, totals: array}`
- `buildSampleRows(Customer $customer, Carbon $from, Carbon $to): array`
- `buildVetRows(Customer $customer, Carbon $from, Carbon $to): array`

**Fila clínico** (por `Admission`):

| Campo | Origen |
|-------|--------|
| `date` | `admission.date` |
| `name` | `patient.full_name` |
| `dni` | `patient.patientId` o documento habitual del modelo |
| `affiliate` | `affiliate_number` o `N/A` |
| `codes` | resolver |
| `price` | suma de `(price - copago)` de líneas incluidas; excluir `paid_by_patient` y `authorization_status === 'rejected'` del resolver |

**Fila muestras** (por `Sample`):

| Campo | Origen |
|-------|--------|
| `date` | `entry_date` |
| `name` | `location` ?: `product_name` ?: `protocol_number` |
| `codes` | resolver sobre `determinations` |
| `price` | suma `price` de líneas incluidas |

**Fila vet** (por `VetAdmission`):

| Campo | Origen |
|-------|--------|
| `date` | `date` |
| `name` | `animal_name` |
| `codes` | resolver sobre `vetTests` |
| `price` | suma `price` de líneas incluidas (o `total_price` si coincide con suma filtrada) |

**Totales** (los tres módulos):

```php
[
    'protocol_count' => $rows->count(),
    'total_amount' => $rows->sum('price'),
]
```

**Tests unitarios** `tests/Unit/BillingSummaryCodeResolverTest.php`:

- Padre + hijos en protocolo → solo código del padre (y montos del padre si aplica).
- Hijo sin padre en protocolo → código del hijo incluido.
- Dos sueltas → `A-B`.
- Clínico: línea `paid_by_patient` excluida del string y del precio.

**Verificación:** `php artisan test --filter=BillingSummaryCodeResolver`
