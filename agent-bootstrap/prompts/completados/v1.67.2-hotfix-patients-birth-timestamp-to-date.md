# v1.67.2 — Hotfix: columna birth de pacientes (timestamp → date)

**Tipo:** patch / hotfix
**Prioridad:** inmediata — bug en producción
**Dependencias:** ninguna (v1.67.1 ya completada)

---

## Problema

Al crear un paciente nacido antes de 1970 (ej: 1949-09-06), MySQL rechaza el valor con:

```
SQLSTATE[22007]: Formato de fecha y hora no válido: 1292 Valor de fecha y hora
incorrecto: '1949-09-06 00:00:00' para la columna `patients`.`birth`
```

**Causa raíz:** La migración original define `birth` como `TIMESTAMP`, cuyo rango válido
en MySQL es 1970-01-01 a 2038-01-19. Una fecha de nacimiento necesita tipo `DATE`
(rango 1000-01-01 a 9999-12-31).

**Bug adicional:** El controlador tiene la línea `$patient->type = 'active';` duplicada
en `store()` y `save_changes()`.

---

## PASO 1 — Migración: cambiar birth de timestamp a date

Crear migración `database/migrations/YYYY_MM_DD_HHMMSS_change_patients_birth_to_date.php`:

```php
Schema::table('patients', function (Blueprint $table) {
    $table->date('birth')->nullable()->change();
});
```

**Importante:** Requiere `doctrine/dbal` para `->change()`. Si no está instalado,
usar statement raw:

```php
DB::statement('ALTER TABLE patients MODIFY birth DATE NULL');
```

Verificar que los datos existentes no se corrompan — los registros con `birth` en rango
válido de timestamp se convierten automáticamente a DATE sin pérdida.
