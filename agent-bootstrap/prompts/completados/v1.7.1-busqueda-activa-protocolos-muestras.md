# v1.7.1 — Búsqueda activa en listado de protocolos de muestras

**VERSIÓN:** v1.7.1  
**SLUG:** busqueda-activa-protocolos-muestras  
**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 30min  
**PRIORIDAD:** media

---

## Descripción

Reemplazar el buscador actual del listado de protocolos de muestras (que requiere
presionar "Filtrar") por una búsqueda en tiempo real que filtra mientras se escribe.
Los filtros de tipo y estado también deben aplicarse instantáneamente al cambiar valor.

### Comportamiento deseado

- Un campo de búsqueda de texto que filtra por: protocolo, cliente, lugar
- Selects de tipo y estado que filtran al cambiar
- Todo filtrado client-side con Alpine.js (sin recargar página)
- Eliminar el botón "Filtrar"
- El filtrado es instantáneo al escribir/seleccionar

### Consideración: paginación

Actualmente el listado usa `paginate(15)`. Para que el filtrado client-side funcione,
hay dos opciones:

**Opción A (recomendada para <500 registros):** Cargar todos los registros sin paginación
y filtrar con Alpine.js. Más simple, funciona inmediato.

**Opción B (si hay muchos registros):** Mantener paginación server-side y usar búsqueda
con debounce que envía request AJAX. Más complejo pero escala mejor.

Evaluar cuántos protocolos hay y elegir la opción apropiada. Si son menos de 500,
ir con la opción A. Si son más, ir con la B.

```html

```

### 3.2 — Barra de filtros

Reemplazar el buscador actual + botón "Filtrar" por:

```html
<div class="flex items-center gap-4 mb-6">
    <!-- Búsqueda de texto -->
    <div class="flex-1 relative">
        <svg class="absolute left-3 top-1/2 -translate-y-1/2 w-4 h-4 text-gray-400" fill="none" stroke="currentColor" viewBox="0 0 24 24">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M21 21l-6-6m2-5a7 7 0 11-14 0 7 7 0 0114 0z"/>
        </svg>
        <input type="text" x-model="search" placeholder="Buscar por protocolo, cliente o lugar..."

    </div>

    <select x-model="filterType" class="border-gray-300 rounded-lg shadow-sm text-sm focus:ring-teal-500 focus:border-teal-500">

        <option value="Validado">Validado</option>
```

### 3.3 — Filas de tabla con x-show

Cada `<tr>` de la tabla debe tener un `x-show` que evalúa los filtros.
Usar data-attributes para los datos de cada fila:

```html
@foreach($samples as $sample)
<tr x-show="matchesFilter('{{ strtolower($sample->protocol_number) }}', '{{ strtolower($sample->customer?->name ?? '') }}', '{{ strtolower($sample->sampling_point ?? $sample->description ?? '') }}', '{{ strtolower($sample->sample_type ?? '') }}', '{{ strtolower($sample->status ?? '') }}')"
    class="hover:bg-gray-50">
    ...
</tr>
@endforeach
```

### 3.4 — Lógica Alpine.js

```javascript
function sampleFilter() {
    return {
        search: '',
        filterType: '',
        filterStatus: '',
        
        matchesFilter(protocol, customer, place, type, status) {
            const q = this.search.toLowerCase();
            const matchesSearch = !q || 
                protocol.includes(q) || 
                customer.includes(q) || 
                place.includes(q);
            const matchesType = !this.filterType || type === this.filterType.toLowerCase();
            const matchesStatus = !this.filterStatus || status === this.filterStatus.toLowerCase();
            return matchesSearch && matchesType && matchesStatus;
        }
    }
}
```

### 3.5 — Contador de resultados

Agregar un contador de resultados visibles:

```html
<p class="text-sm text-gray-500 mb-2">
    Mostrando <span x-text="document.querySelectorAll('tbody tr:not([style*=display\\:none])').length"></span>
    de {{ $samples->count() }} protocolos
</p>
```

Alternativa más robusta: contar en Alpine.js con un computed.

### 3.6 — Eliminar paginación

Si se usa opción A, eliminar el `{{ $samples->links() }}` del pie de tabla.
