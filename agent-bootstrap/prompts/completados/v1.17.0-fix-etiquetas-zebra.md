## Descripción

La impresión directa de etiquetas a la impresora Zebra GK420T via Zebra Browser Print no funciona.
El error es: "No se pudo enviar la etiqueta a la impresora: Error al enviar a la impresora".

**Causa raíz:** El módulo JS `zebra-label-print.js` parsea la respuesta de `GET /available` como texto
separado por tabs, pero la versión instalada de Zebra Browser Print devuelve JSON. Esto causa que
el objeto `device` enviado a `POST /write` tenga datos malformados (el JSON crudo como nombre),
y la impresora no se encuentra → error.

**Problemas secundarios:**
- El dropdown de impresoras muestra JSON crudo en vez de nombres legibles
- `Content-Type: text/plain` en el POST pero el body es JSON (debería ser `application/json`)
- La URL de la API (`127.0.0.1:9100`) está hardcodeada; si el sitio se accede por HTTPS
  (sistema.ipac.com.ar) se necesita el puerto HTTPS (9101) para evitar mixed content
- El método `labelData()` del controlador no tiene `authorize()`

## PASO 1 — Reescribir `public/js/zebra-label-print.js`

Reescribir el módulo completo. El archivo tiene dos partes: el objeto `ZebraLabelPrint` y la función
Alpine `zebraPrintModal()`. Ambas necesitan cambios.

### 1.1 — Detección automática de puerto y protocolo

La API de Zebra Browser Print escucha en:
- `http://localhost:9100` para sitios HTTP
- `https://localhost:9101` para sitios HTTPS

Detectar automáticamente según `window.location.protocol`:

```javascript
const API_URL = window.location.protocol === 'https:'
    ? 'https://localhost:9101'
    : 'http://localhost:9100';
```

IMPORTANTE: Usar `localhost` en vez de `127.0.0.1` porque el certificado de Zebra Browser Print
está emitido para `localhost`.

### 1.2 — Parseo correcto de impresoras (`getAvailablePrinters`)

La API de Zebra Browser Print puede devolver **JSON o texto plano** según la versión.
Implementar detección dual:

```javascript
async getAvailablePrinters() {
    const response = await fetch(`${this.API_URL}/available?type=printer`);
    if (!response.ok) throw new Error('No se pudo conectar con Zebra Browser Print');

    const text = await response.text();
    let printers;

    try {
        // Intentar parsear como JSON (versiones modernas)
        const parsed = JSON.parse(text);
        printers = Array.isArray(parsed) ? parsed : [parsed];
        printers = printers.map(p => ({
            name: p.name || 'Impresora Zebra',
            uid: p.uid || '',
            connection: p.connection || 'unknown',
            deviceType: p.deviceType || 'printer',
        }));
    } catch {
        // Fallback: parsear como texto tab-separated (versiones viejas)
        printers = this._parsePrintersFromText(text);
    }

    this.printers = printers;
    // ... resto de lógica de selección guardada en localStorage ...
}
```

### 1.3 — Envío correcto a la impresora (`printLabel`)

Cambiar el `POST /write`:
- `Content-Type` debe ser `application/json`
- El body es un JSON con `device` (objeto completo de la impresora) y `data` (string ZPL)

```javascript
async printLabel(zplContent) {
    if (!this.selectedPrinter) throw new Error('No hay impresora seleccionada');

    const response = await fetch(`${this.API_URL}/write`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            device: {
                name: this.selectedPrinter.name,
                uid: this.selectedPrinter.uid,
                connection: this.selectedPrinter.connection,
                deviceType: this.selectedPrinter.deviceType,
            },
            data: zplContent,
        }),
    });

    if (!response.ok) {
        const errorText = await response.text().catch(() => 'Error desconocido');
        throw new Error('Error al enviar a la impresora: ' + errorText);
    }

    return true;
}
```

- El ZPL actual genera etiquetas de ~60×30mm (PW480, LL240 a 203 DPI). No cambiar las dimensiones.
