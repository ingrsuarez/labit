## Qué se acordó

Feature para prácticas cuyo resultado es una **fórmula aritmética** entre otras determinaciones del nomenclador (ej. Índice de Castelli = Colesterol total ÷ HDL). Autocompletado **en vivo** en protocolo (clínico, vet, muestras) cuando los operandos del pedido tienen valor numérico. Campo editable pero **el recálculo siempre pisa** al cambiar un operando.

## Decisiones de scope

| Decisión | Valor |
|----------|--------|
| Recálculo vs manual | Cálculo manda siempre |
| Módulos | Clínico + vet + muestras |
| Operandos inválidos | Vacío, sin bloquear |
| Decimales | `test.decimals` o 2 default |
| Alta en protocolo | No auto-agregar |
| LISCOM/API | Ignorar ingesta en tests con fórmula |

## Edge cases detectados

- División por cero → vacío, no error UI
- Padres-título sin resultado no son operandos válidos
- No confundir con práctica “formula leucocitaria” (`empty_result_exempt`)
- Cero como operando es válido (v1.19.2)
- Evitar `eval()` sobre strings de usuario; evaluador seguro por tokens
