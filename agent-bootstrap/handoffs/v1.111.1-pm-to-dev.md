## Resumen ejecutivo

## Decisiones de scope (acordadas con el usuario)

| Tema | Decisión |
|------|----------|
| Visibilidad | Botón oculto si `count === 0` |
| Sede concreta | Solo pendientes de esa sede |
| Todas las sedes | Todas las pendientes, sumadas en contador |
| Orden modal | `protocol_number` ascendente (cambiar desde `created_at`) |
| Mobile | Fuera de alcance |

## Edge cases

- Primer paint: `count` inicia en 0 → botón oculto hasta fetch (evitar flash con `x-cloak`).
- Usuario registra última extracción en modal → `refreshCount()` → botón desaparece.
- Cambio de sede vía form POST → reload → `init()` recalcula.
- Post-filter `admissionRequiresSampleDraw()` puede desordenar → re-sort collection si hace falta.
