## Resumen ejecutivo

Cola operativa de **extracción de sangre/muestra** post-admisión en lab **clínico**, filtrada por **sede activa**, con registro obligatorio del **tomador** (técnico o bioquímico). Bloquea **planillas clínicas** hasta registrar extracción. Alimenta **auditoría** y **KPIs de productividad**.

---

## Decisiones de scope (acordadas con el usuario)

| Tema | Decisión |
|------|----------|
| Módulos | Solo lab clínico (`Admission`) |
| ¿Cuándo hay pendiente? | ≥1 práctica **padre** con `material` (misma lógica que etiquetas) |
| UI cola | Modal/drawer desde **header** (opción 1 PM) |
| UI admisión | Selector tomador **siempre visible** si hay material; vacío = pendiente (opción 2 PM) |
| Roles operadores | `tecnico-lab`, `bioquimico`, `recepcion-lab`, `admin` |
| Tomador | Obligatorio; técnico/bioquímico = self; recepción/admin eligen en lista |
| Planillas | Excluir protocolos sin extracción; aviso en preview |
| KPI | Contar por `sample_drawn_by`, no por quien clickeó en recepción |
| Backfill histórico | No |
| Vet / muestras | No |

## Edge cases

- Admisión editada: quitar todas las prácticas con material → limpiar `sample_drawn_*`.
- Carga masiva recepción: completar tomador en el mismo formulario evita cola.
- Sede “Todas las sedes” en header: sumar pendientes globales; considerar columna sede en modal.
- No confundir con `results_loaded` ni entregas PDF/email.
