## Contexto del pedido

Hoy los informes de lab clínico generados en Labit se suben **manualmente** al portal Space10 (SPACE4CLINIC, repo hermano en `c:\wamp64\www\space10`). Space10 ya tiene API `POST /api/upload/lab` con Sanctum y un script Python de referencia.

El usuario pidió automatizar desde Labit:

1. **Al enviar email** del informe (individual o masivo) → subir PDF a Space10 por DNI del paciente.
2. **Batch desde listado** → subir varios protocolos sin enviar email.
3. **No re-subir** si ya fue subido (ahorro de tiempo).

**Solo lab clínico.** Vet y muestras fuera de alcance.

## Tensiones y advertencias para el Dev

### 1. Orden de operaciones en email

Subir a Space10 **después** de confirmar envío de email exitoso. Si Space10 falla, el email ya salió — mostrar warning, no rollback.

### 2. Paciente inexistente en Space10

API responde 404. Tratar como error recuperable; usuario puede crear paciente en Space10 y reintentar vía batch.

### 3. Límite 8MB en Space10

`UploadLabController` valida `max:8000` KB. Informes muy grandes pueden fallar — loguear error claro.

### 4. Token Sanctum

El usuario de Space10 que posee el token debe tener `currentInstitution` (UploadLabController usa `$user->currentInstitution`). Verificar en staging que el token de servicio tenga institución activa.

### 5. URL local WAMP

Ejemplo dev: `http://localhost/space10/public/api/upload/lab` — ajustar según virtual host del entorno.

### 6. Batch email + Space10

En `batchEmail`, tras un solo `Mail::send` exitoso, iterar uploads por admisión. Agregar clave `space10` al JSON de respuesta para feedback en UI (opcional en v1.114.0 si batch email no muestra detalle Space10 — mínimo: audit log).
