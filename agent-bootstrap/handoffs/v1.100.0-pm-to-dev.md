## Contexto del pedido

Tras v1.66.0, el sidebar admin tiene **cuatro entradas** relacionadas con RRHH:
«Recursos Humanos» (panel analítico en `/rrhh`) más **Personal**, **Ausencias** y
**Liquidaciones** (cada una lleva a una página intermedia con cards en `admin.section`).

El usuario pidió **unificar**: un solo ítem en el sidebar y, dentro de `/rrhh`, bloques
con título y cards debajo (mismos íconos y destinos que hoy).

---

## Decisiones tomadas en la sesión PM

| Decisión | Valor | Razón |
|----------|--------|--------|
| Contenido de `/rrhh` | Solo hub de navegación (3 bloques + cards) | Opción 2 elegida por el usuario |
| Panel analítico (KPIs actuales) | Mover a `/rrhh/resumen` | No perder funcionalidad; separar navegación vs.analytics |
| Sidebar | Un ítem «Recursos Humanos»; quitar Personal, Ausencias, Liquidaciones | Objetivo explícito del pedido |
| Diseño previo | Sí — documento Designer antes del Dev | Aprobado por el usuario |
| Rutas `admin.section.*` | Redirect 302 al hub con `#personal` / `#ausencias` / `#liquidaciones` | Bookmarks y links viejos siguen funcionando |
| Acceso al hub | Cualquier permiso `*.section` de RRHH + admin/contador | Hoy RRHH sidebar solo admin/contador pero las 3 secciones usaban `@can` — corregir inconsistencia |
| Acceso a resumen | Solo admin y contador | Igual que el panel analítico actual |
| Designer | Completado en `docs/design/v1.100.0-rrhh-hub-navegacion.md` | — |
