---

## Contexto del pedido

Al presionar **«Administración»** desde el lab, el usuario llega a `/manage/index`
(lista de empleados) en lugar de un punto de entrada útil. Esto es un vestigio de
**v1.66.0**, cuando RRHH vivía en `/dashboard` y empleados en `manage.index`.

El usuario pidió que `/dashboard` sea el **home personalizado** con cards de acceso
rápido, y que el sistema **aprenda** qué pantallas usa cada persona para ordenar
esas cards por frecuencia.

---

## Decisiones tomadas en la sesión PM

| Decisión | Valor | Razón |
|----------|--------|--------|
| Home principal | `/dashboard` → cards personalizadas | Punto de entrada unificado |
| Dashboard financiero | Mover a `/dashboard/financiero` | No perder KPIs de v1.66.0 |
| Tracking scope | **Todo el sistema** (admin + lab + portal) | Pedido explícito del usuario |
| Ordenamiento | `hit_count DESC` + defaults por rol si sin historial | Aprendizaje progresivo |
| Cantidad de cards | Máximo 12 | Legible en grid existente |
| Empleados (`manage.index`) | Sin cambios | Accesible desde RRHH |
| Diseño UI | Reutilizar `section-cards` | Sin Designer |
| Redirects actuales en `DashboardController` | Eliminar | El hub reemplaza compras/ventas/lab redirect |
| Portal-only users | Siguen en `portal.dashboard` | `CheckSystemAccess` no cambia; tracking sí corre en portal |

## Advertencias para el Dev

1. **`FinancialDashboardTest`** asume KPIs en `/dashboard` — mover assertions a `/dashboard/financiero`.
2. **Tests de redirect** compras/ventas/lab desde `/dashboard` deben **eliminarse o invertirse** (ahora todos ven el home).
3. El middleware debe registrarse **después** de auth; rutas GET anónimas no se trackean.
4. Excluir `route('dashboard')` del tracking para no inflar hits al refrescar home.
5. Card «Resumen financiero» solo visible para admin/contador — no usar solo `compras.section`.
6. Shortcut `portal.team` requiere lógica de supervisor (`$employee->isSupervisor()`) — no mostrar si no aplica.
7. **`route_names`**: mapear `index/show/create/edit` al mismo `shortcut_key` para no fragmentar stats.
