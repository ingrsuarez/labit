## Contexto del pedido

Pantalla de **tesorería** para ordenar el flujo de caja: vista **mensual** (default) y
**semanal** con todos los pagos a realizar, agrupados por categoría, con montos y links
al origen.

El usuario (admin/contador) necesita anticiparse a salidas de caja sin revisar módulos
dispersos (FC, DDJJ, liquidaciones, obligaciones manuales).

**Handoff de base:** esta versión implementa lo acordado en la sesión PM del 2026-05-30.

## Advertencias para el Dev

### 1. IVA — prioridad del monto

Para cada mes futuro/proyectado:

1. Si existe `TaxReturn` confirmada para impuesto IVA + período → usar `balance` (saldo a pagar; si negativo, no mostrar evento o mostrar $0 con nota "saldo a favor").
2. Si no → sumar IVA débito (FV − NC venta) − IVA crédito (FC − NC compra) del período anterior al vencimiento (documentar regla en servicio).

Identificar impuesto IVA por nombre/code en catálogo `Tax` o flag `is_iva` — definir criterio robusto (seeder o convención documentada).

### 2. Gastos fijos — fecha de proyección

Si la última FC de alquiler vence el 10/05, proyectar el 10 de cada mes siguiente con
monto = total de esa FC. Badge **estimado**. Si entra FC nueva, actualizar proyección.

### 3. Performance

Agregar eventos en PHP para el rango visible (mes o semana + buffer). No N+1: eager load
suppliers, tax returns, etc.

### 4. Feriados

Reutilizar modelo/tabla de feriados existente (`HolidaysSeeder`). Si no hay servicio de
día hábil, crear helper mínimo `BusinessDayCalculator::nthBusinessDayOfMonth($year, $month, $n)`.
